Handout: Typische Anwendungsfälle von Enums

1. Enums in switch-Statements

Enums eignen sich hervorragend für den Einsatz in switch-Anweisungen, da sie eine festgelegte, überschaubare Menge an Werten haben. Das macht den Code übersichtlich und typischer als bei der Verwendung von Strings oder Zahlen.

Beispiel:

public enum Wochentag {
    MONTAG, DIENSTAG, MITTWOCH, DONNERSTAG, FREITAG, SAMSTAG, SONNTAG
}

public class EnumBeispiel {
    public static void zeigeTagesinfo(Wochentag tag) {
        switch (tag) {
            case MONTAG:
                System.out.println("Wochenstart!");
                break;
            case FREITAG:
                System.out.println("Bald Wochenende!");
                break;
            case SAMSTAG:
            case SONNTAG:
                System.out.println("Wochenende!");
                break;
            default:
                System.out.println("Ein normaler Wochentag.");
        }
    }
}

Vorteile:

  • Tippfehler werden vom Compiler erkannt.
  • Der Compiler warnt, wenn ein Wert nicht behandelt wird.
  • Umstellung (z. B. Reihenfolge der Werte) ist einfach und sicher.

2. Enums als sichere Alternative zu magischen Zahlen/Strings

Magische Zahlen/Strings sind nicht selbsterklärend, können zu Tippfehlern führen und machen den Code schwer wartbar.

Schlecht (magische Zahlen/Strings):

int status = 2; // Was bedeutet 2?
if (status == 2) { ... }

String phase = "ROOOT"; // Tippfehler bleibt unentdeckt
if (phase.equals("ROT")) { ... }

Gut (Enums):

public enum Status {
    AKTIV, INAKTIV, GESPERRT
}

Status status = Status.AKTIV;
if (status == Status.AKTIV) { ... }

Vorteile:

  • Typsicherheit: Es kann kein ungültiger Wert zugewiesen werden.
  • Fehlervermeidung: Tippfehler werden beim Kompilieren entdeckt.
  • Bessere Lesbarkeit: Jeder Wert ist klar und selbsterklärend.
  • Leicht erweiterbar: Neue Werte können einfach ergänzt werden.

3. Enums in Collections (EnumSet, EnumMap)

Die Java-Standardbibliothek bietet zwei spezielle Collection-Typen, die speziell für Enums optimiert sind:

a) EnumSet

  • Sehr effiziente Set-Implementierung für Enum-Typen.
  • Schneller als HashSet und mit geringerem Speicherverbrauch.

Beispiel:

import java.util.EnumSet;

public class EnumSetBeispiel {
    public enum Richtung { NORD, SUED, OST, WEST }

    public static void main(String[] args) {
        EnumSet<Richtung> erlaubteRichtungen = EnumSet.of(Richtung.NORD, Richtung.OST);

        for (Richtung r : erlaubteRichtungen) {
            System.out.println(r);
        }
    }
}
  • EnumSet.allOf(Richtung.class) erzeugt ein Set mit allen Enum-Werten.
  • EnumSet.noneOf(Richtung.class) erzeugt ein leeres EnumSet.

b) EnumMap

  • Eine Map, deren Schlüssel vom Typ Enum sind.
  • Sehr schnell und speichersparend, da intern ein Array verwendet wird.

Beispiel:

import java.util.EnumMap;

public class EnumMapBeispiel {
    public enum Status { AKTIV, INAKTIV, GESPERRT }

    public static void main(String[] args) {
        EnumMap<Status, String> statusTexte = new EnumMap<>(Status.class);

        statusTexte.put(Status.AKTIV, "Benutzer ist aktiv");
        statusTexte.put(Status.INAKTIV, "Benutzer ist inaktiv");
        statusTexte.put(Status.GESPERRT, "Benutzer ist gesperrt");

        for (Status status : Status.values()) {
            System.out.println(status + ": " + statusTexte.get(status));
        }
    }
}

4. Übungen zum Selbststudium

  1. Enum & switch: Schreibe ein Enum AmpelPhase mit den Werten ROT, GELB, GRUEN. Implementiere eine Methode, die je nach Phase eine passende Anweisung ausgibt (z. B. „Stop“, „Achtung“, „Fahren“).

  2. Magische Strings ersetzen: Ersetze im folgenden Code die Farbe als String durch ein Enum und passe die Vergleiche entsprechend an:

    String farbe = "Rot";
if (farbe.equals("Rot")) { ... }

  3. EnumSet/EnumMap: Lege ein EnumSet mit allen Wochenendtagen an. Lege eine EnumMap an, die für jeden Wochentag einen kurzen Info-Text speichert.

5. Merksätze

  • Enums sind der sichere, moderne Weg für Zustände, Optionen und Kategorien im Code.
  • Für Mengen und Zuordnungen von Enum-Werten verwende EnumSet und EnumMap.
  • Setze Enums gezielt ein, um Fehler durch magische Zahlen/Strings zu vermeiden und den Code klarer zu machen.
