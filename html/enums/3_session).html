Handout: Erweiterte Möglichkeiten mit Enums in Java

1. Methoden in Enums

Enums können – wie Klassen – eigene Methoden und sogar Überschreibungen von Standardmethoden enthalten. Das macht sie sehr mächtig und vielseitig.

a) Eigene Methoden im Enum

Du kannst in einem Enum beliebige Methoden definieren, um das Verhalten der einzelnen Enum-Werte zu beschreiben.

Beispiel:

public enum Richtung {
    NORD, SUED, OST, WEST;

    // Eigene Methode
    public Richtung gegenrichtung() {
        switch(this) {
            case NORD: return SUED;
            case SUED: return NORD;
            case OST:  return WEST;
            case WEST: return OST;
            default:   return null;
        }
    }
}

Verwendung:

Richtung r = Richtung.OST;
System.out.println(r.gegenrichtung()); // WEST

b) Überschreiben der toString()-Methode

Wie bei Klassen kann auch in Enums die Methode toString() überschrieben werden, um eine lesbare oder angepasste String-Repräsentation zu liefern.

Beispiel:

public enum Status {
    AKTIV, INAKTIV, GESPERRT;

    @Override
    public String toString() {
        switch(this) {
            case AKTIV:    return "Aktiv";
            case INAKTIV:  return "Inaktiv";
            case GESPERRT: return "Gesperrt";
            default:       return super.toString();
        }
    }
}

Verwendung:

Status s = Status.AKTIV;
System.out.println(s); // Gibt "Aktiv" aus

2. Attribute (Felder) und Konstruktoren im Enum

Enums können Felder (Attribute) besitzen und Konstruktoren definieren, um jedem Wert zusätzliche Eigenschaften zu geben.

a) Enum mit zusätzlichen Eigenschaften

Beispiel: Ein Enum für Planeten mit Masse und Radius:

public enum Planet {
    MERKUR(3.30e23, 2_439_700),
    VENUS(4.87e24, 6_051_800),
    ERDE(5.97e24, 6_371_000);

    private final double masse;  // in Kilogramm
    private final double radius; // in Metern

    // Konstruktor
    Planet(double masse, double radius) {
        this.masse = masse;
        this.radius = radius;
    }

    // Getter
    public double getMasse() { return masse; }
    public double getRadius() { return radius; }
}

Verwendung:

Planet p = Planet.ERDE;
System.out.println(p + ": Masse = " + p.getMasse() + ", Radius = " + p.getRadius());

b) Konstruktoren und Getter

  • Der Konstruktor im Enum ist immer privat (bzw. package-private), wird aber ohne Zugriffsmodifizierer geschrieben.
  • Du kannst für jedes Enum-Element beim Anlegen Eigenschaften mitgeben.

Beispiel:

public enum AmpelPhase {
    ROT("Stop"),
    GELB("Achtung"),
    GRUEN("Go");

    private final String bedeutung;

    AmpelPhase(String bedeutung) {
        this.bedeutung = bedeutung;
    }

    public String getBedeutung() {
        return bedeutung;
    }
}

Verwendung:

for (AmpelPhase phase : AmpelPhase.values()) {
    System.out.println(phase + ": " + phase.getBedeutung());
}

Ausgabe:

ROT: Stop
GELB: Achtung
GRUEN: Go

3. Übungen zum Selbststudium

  1. Definiere ein Enum Schulnote mit den Werten EINS, ZWEI, DREI, VIER, FUENF, SECHS. Gib jeder Note die Eigenschaft „Punkte“ (z. B. EINS = 15, ZWEI = 12, …). Implementiere eine Methode, die für jede Note die Punkte zurückliefert.

  2. Schreibe ein Enum Wetterlage mit den Werten SONNIG, BEWÖLKT, REGNERISCH, SCHNEE. Überschreibe die toString()-Methode so, dass eine Wetterbeschreibung wie „Die Sonne scheint“ ausgegeben wird.

  3. Füge in ein vorhandenes Enum eine eigene Methode ein (z. B. istWochenende() bei Wochentag).

4. Merksätze

  • Enums können Methoden, Attribute und Konstruktoren besitzen.
  • Jeder Enum-Wert kann eigene Eigenschaften und individuelles Verhalten haben.
  • Mit toString() kannst du die Textdarstellung deiner Enum-Werte anpassen.
