Handout: Einführung in Enums (Java)

1. Was sind Enums?

Enums (kurz für Enumerationen oder Aufzählungstypen) sind ein spezieller Datentyp in Java, mit dem eine feste Menge benannter, konstanter Werte definiert werden kann.

  • Ein Enum repräsentiert alle zulässigen Werte für eine bestimmte Variable, z. B. die Wochentage, die Himmelsrichtungen oder Statusangaben wie „AKTIV“ oder „INAKTIV“.
  • Jeder Wert eines Enums ist ein eigenständiges Objekt, vom Compiler automatisch als public static final angelegt.

Beispiel:

public enum Wochentag {
    MONTAG, DIENSTAG, MITTWOCH, DONNERSTAG, FREITAG, SAMSTAG, SONNTAG
}
  • Hier können Variablen vom Typ Wochentag nur einen der sieben Wochentage annehmen.
  • Die Werte werden typischerweise in Großbuchstaben geschrieben.

2. Unterschiede zu klassischen Konstanten (final static Felder)

Vor Java 5 wurden oft Konstanten mit public static final deklariert, um feste Werte zu definieren.

Beispiel (alte Variante):

public class Farbe {
    public static final int ROT = 0;
    public static final int GRUEN = 1;
    public static final int BLAU = 2;
}

Nachteile:

  • Keine Typprüfung: Jeder beliebige int-Wert kann gesetzt werden (z. B. 17).
  • Keine Gruppierung im Typensystem: Es bleibt beim primitiven Datentyp.
  • Keine Erweiterungen möglich (z. B. Methoden, Attribute).

Mit Enum:

public enum Farbe {
    ROT, GRUEN, BLAU
}

Vorteile:

  • Variablen können nur einen gültigen Wert annehmen, der im Enum definiert ist.
  • Der Compiler prüft, ob der Wert gültig ist.
  • Zusätzliche Methoden und Attribute können dem Enum hinzugefügt werden.

Vergleichstabelle:

Klassische Konstante (static final) Enum
Typprüfung Nein Ja
Nur erlaubte Werte Nein Ja
Eigene Methoden Nein Ja
Sicher im Switch Nein Ja

3. Motivation: Warum Enums anstelle von int- oder String-Konstanten?

a) Typsicherheit

Mit Enums kannst du Variablen nur mit erlaubten Werten belegen. Beispiel:

Farbe f = Farbe.ROT; // Gültig
// f = 5; // Compilerfehler: Falscher Typ!

b) Klarheit und Lesbarkeit

Enums machen den Code viel verständlicher als Zahlen oder Strings:

// Weniger lesbar
int status = 2; // Was bedeutet 2?

// Klar und deutlich
Status s = Status.AKTIV;

c) Fehlervermeidung

Mit Enums kann niemand „magische Zahlen“ oder „Tippfehler-Strings“ einschleusen:

// Mit String
String wochentag = "Monntag"; // Tippfehler, aber Compiler merkt es nicht!

// Mit Enum
Wochentag tag = Wochentag.MONTAG; // Tippfehler? -> Compilerfehler!

d) Erweiterbarkeit und Funktionalität

Enums können Methoden und Eigenschaften besitzen:

public enum Planet {
    ERDE(5.97E24), MARS(6.42E23);

    private double masse;

    Planet(double masse) {
        this.masse = masse;
    }

    public double getMasse() {
        return masse;
    }
}

e) Nutzung in switch-Statements

Enums sind für die Verwendung in switch-Anweisungen optimiert:

Wochentag tag = Wochentag.MONTAG;
switch(tag) {
    case MONTAG: System.out.println("Wochenstart!"); break;
    case FREITAG: System.out.println("Bald Wochenende!"); break;
    // ...
}

4. Zusammenfassung

  • Enums sind der moderne und sichere Weg, eine feste Menge von Werten zu definieren.
  • Sie bieten Typsicherheit, Lesbarkeit und viele zusätzliche Möglichkeiten.
  • Enums vermeiden Fehler, die bei der Nutzung von int- oder String-Konstanten häufig auftreten.
  • Sie sind einfach zu nutzen und können erweitert werden (z. B. Methoden, Attribute).

  2. Ersetze im folgenden Code alle Farbangaben durch ein Enum:
