Handout: Syntax und Grundlagen von Enums

1. Definition eines Enums

a) Grundaufbau (enum Wochentag { ... })

Ein Enum wird mit dem Schlüsselwort enum definiert und enthält eine durch Kommas getrennte Liste möglicher Werte.

Beispiel:

public enum Wochentag {
    MONTAG, DIENSTAG, MITTWOCH, DONNERSTAG, FREITAG, SAMSTAG, SONNTAG
}
  • Jeder Wert (MONTAG, DIENSTAG …) ist ein Enum-Konstantenwert.
  • Du kannst so viele Werte angeben, wie für deinen Anwendungsfall nötig sind.
  • Die Definition erfolgt typischerweise außerhalb oder als eigene Datei (wie bei einer Klasse), kann aber auch als static innerhalb einer anderen Klasse stehen.

b) Werte und Großschreibung

  • Konvention: Enum-Konstanten werden in Großbuchstaben geschrieben, einzelne Wörter werden mit _ getrennt (z. B. NICHT_VERFUEGBAR).
  • Das ist keine Pflicht, aber gängige Praxis und erhöht die Lesbarkeit.

Beispiele:

public enum Status {
    AKTIV, INAKTIV, GESPERRT
}

public enum Richtung {
    NORD, SUED, OST, WEST
}

2. Verwendung eines Enums im Code

Enums können wie jeder andere Typ in Variablen, Methodenparametern und Rückgabewerten verwendet werden.

a) Zuweisung

Um einer Variable einen Enum-Wert zuzuweisen, nutzt du den Enum-Namen und den gewünschten Wert mit Punktnotation:

Wochentag heute = Wochentag.DIENSTAG;
Status benutzerStatus = Status.AKTIV;

b) Vergleich

Enums werden immer mit == oder equals() verglichen – beide Möglichkeiten funktionieren gleich sicher, da es garantiert immer nur eine Instanz jedes Enum-Wertes gibt.

Beispiel:

if (heute == Wochentag.DIENSTAG) {
    System.out.println("Heute ist Dienstag!");
}
  • Vorteil gegenüber Strings: Tippfehler beim Wert werden sofort vom Compiler erkannt.

c) Ausgabe

Enums können direkt ausgegeben werden, z. B. mit System.out.println():

System.out.println(heute); // Gibt "DIENSTAG" aus

  • Standardmäßig wird der Name der Enum-Konstanten als String ausgegeben.

  • Mit .name() kannst du den Namen als String bekommen:

    String tagName = heute.name(); // "DIENSTAG"

d) Enum als Typ für Variablen und Parameter

Enums eignen sich hervorragend, um Methoden klar zu typisieren und Fehler zu vermeiden:

Beispiel – Methode mit Enum-Parameter:

public void druckeFeiertagsgruß(Wochentag tag) {
    if (tag == Wochentag.SONNTAG) {
        System.out.println("Schönen Sonntag!");
    }
}

Beispiel – Enum als Rückgabewert:

public Wochentag naechsterTag(Wochentag tag) {
    switch (tag) {
        case MONTAG: return Wochentag.DIENSTAG;
        case DIENSTAG: return Wochentag.MITTWOCH;
        // ...
        case SONNTAG: return Wochentag.MONTAG;
    }
    return null; // Sollte eigentlich nicht passieren!
}

3. Übungen zum Selbststudium

  1. Definiere ein Enum AmpelPhase mit den Werten ROT, GELB, GRUEN. Lege eine Variable an und weise ihr einen Wert zu.
  2. Schreibe eine Methode, die einen Gruß passend zur aktuellen AmpelPhase ausgibt.
  3. Probiere aus, wie der Compiler reagiert, wenn du einen Wert verwendest, der im Enum nicht existiert.

4. Merksätze

  • Enums werden mit enum definiert und Werte stehen in Großbuchstaben.
  • Zuweisung, Vergleich und Ausgabe sind einfach und sicher.
  • Enums machen Code klarer, typischer und weniger fehleranfällig als Zahlen oder Strings.
