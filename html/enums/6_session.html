Handout: Praktische Übungen und Beispiele zu Enums

1. Eigenes Enum definieren und verwenden

Aufgabe 1

Definiere ein Enum AmpelPhase mit den Werten ROT, GELB, GRUEN. Verwende es in einer Methode, die einen Hinweis ausgibt.

Musterlösung

public enum AmpelPhase {
    ROT, GELB, GRUEN
}

public class AmpelTest {
    public static void main(String[] args) {
        AmpelPhase phase = AmpelPhase.ROT;
        printHinweis(phase);
    }

    public static void printHinweis(AmpelPhase phase) {
        if (phase == AmpelPhase.ROT) {
            System.out.println("Stehen bleiben!");
        } else if (phase == AmpelPhase.GELB) {
            System.out.println("Gleich geht's weiter.");
        } else if (phase == AmpelPhase.GRUEN) {
            System.out.println("Losfahren!");
        }
    }
}

2. Enum mit Attributen und Methoden implementieren

Aufgabe 2

Erweitere das Enum AmpelPhase um eine Zeichenkettenbeschreibung (z. B. „Stop“, „Achtung“, „Fahren“) und eine Methode, die diese Beschreibung zurückgibt.

Musterlösung

public enum AmpelPhase {
    ROT("Stop"),
    GELB("Achtung"),
    GRUEN("Fahren");

    private final String beschreibung;

    AmpelPhase(String beschreibung) {
        this.beschreibung = beschreibung;
    }

    public String getBeschreibung() {
        return beschreibung;
    }
}

Verwendung:

public class AmpelTest {
    public static void main(String[] args) {
        for (AmpelPhase phase : AmpelPhase.values()) {
            System.out.println(phase + ": " + phase.getBeschreibung());
        }
    }
}

Ausgabe:

ROT: Stop
GELB: Achtung
GRUEN: Fahren

3. Anwendung von Enums in einem switch

Aufgabe 3

Schreibe eine Methode, die für jeden Wochentag eine kurze Nachricht im switch-Statement ausgibt.

Musterlösung

public enum Wochentag {
    MONTAG, DIENSTAG, MITTWOCH, DONNERSTAG, FREITAG, SAMSTAG, SONNTAG
}

public class WochentagTest {
    public static void printMessage(Wochentag tag) {
        switch (tag) {
            case MONTAG:
                System.out.println("Wochenstart!");
                break;
            case FREITAG:
                System.out.println("Bald Wochenende!");
                break;
            case SAMSTAG:
            case SONNTAG:
                System.out.println("Wochenende!");
                break;
            default:
                System.out.println("Ein gewöhnlicher Tag.");
        }
    }

    public static void main(String[] args) {
        for (Wochentag tag : Wochentag.values()) {
            printMessage(tag);
        }
    }
}

4. Refactoring: Ersetze klassische Konstanten durch ein Enum

Aufgabe 4

Gegeben ist folgender (veralteter) Code:

public class FarbenAlt {
    public static final int ROT = 0;
    public static final int BLAU = 1;
    public static final int GRUEN = 2;
}

Refactore das Beispiel so, dass ein Enum verwendet wird.

Musterlösung

public enum Farbe {
    ROT, BLAU, GRUEN
}

// Verwendung:
Farbe farbe = Farbe.ROT;
if (farbe == Farbe.BLAU) {
    System.out.println("Die Farbe ist Blau!");
}

5. Reflexionsfragen (Selbststudium)

  1. Welche Vorteile ergeben sich durch den Einsatz von Enums im Vergleich zu klassischen Konstanten?
  2. Wie kannst du einem Enum zusätzliche Eigenschaften geben? Wo ist das sinnvoll?
  3. Wann ist die Verwendung eines Enums im Code besser als die Arbeit mit Zahlen oder Strings?

6. Zusammenfassung

  • Enums bieten eine saubere, sichere und erweiterbare Möglichkeit, feste Wertemengen zu repräsentieren.
  • Du kannst Enums mit Attributen, Methoden und Konstruktoren anreichern.
  • Die Verwendung von Enums im switch-Statement sorgt für klaren, wartbaren Code.
  • Das Refactoring von klassischen Konstanten zu Enums steigert die Typsicherheit und Lesbarkeit deines Programms.
