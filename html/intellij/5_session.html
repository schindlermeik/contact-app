Handout: Individualisierung und Erweiterungen in IntelliJ IDEA

1. Plugins in IntelliJ IDEA

1.1. Was sind Plugins?

Plugins sind Zusatzmodule, mit denen du den Funktionsumfang von IntelliJ IDEA nach deinen Bedürfnissen erweitern kannst. Es gibt Plugins für Programmiersprachen, Frameworks, Produktivität, Design, Tools und vieles mehr.

1.2. Plugin-Marktplatz entdecken

So öffnest du den Plugin-Marktplatz:

  1. Menü:

    • „File“ → „Settings“ (oder Strg + Alt + S)
    • Navigiere zu „Plugins“

  2. Alternativ:

    • „Preferences“ → „Plugins“ (macOS: ⌘ + ,)

Im Fenster siehst du:

  • Marketplace: Tausende frei verfügbare Plugins durchsuchen.
  • Installed: Übersicht deiner bereits installierten Plugins.
  • Updates: Updates für vorhandene Plugins.

Suche: Im Marketplace kannst du nach Stichworten suchen (z. B. „Lombok“ oder „Theme“).

1.3. Plugins installieren

Beispiel: Installation von Lombok

  1. Gehe im Plugin-Tab auf „Marketplace“.
  2. Suche nach Lombok.
  3. Klicke „Install“.
  4. Nach der Installation: Restart IDE, damit das Plugin aktiviert wird.

Weitere empfohlene Plugins:

  • Key Promoter X: Zeigt zu häufig per Maus genutzten Aktionen den passenden Shortcut an – optimal zum Shortcuts-Lernen.
  • Rainbow Brackets: Hebt verschachtelte Klammern farbig hervor.
  • Material Theme UI: Für individuelle, moderne Oberflächen.
  • GitToolBox: Erweiterte Git-Features.

Tipp: Weniger ist oft mehr – installiere nur Plugins, die du wirklich brauchst, um die Performance der IDE nicht zu beeinträchtigen.

1.4. Plugins verwalten und entfernen

  • Im Tab „Installed“ siehst du alle aktiven Plugins.
  • Mit „Disable“ kannst du ein Plugin deaktivieren (z. B. zum Testen, ob ein Fehler dadurch verursacht wird).
  • Mit „Uninstall“ entfernst du das Plugin komplett.

2. Hilfe & Troubleshooting

2.1. „Invalidate Caches / Restart“

Manchmal „spinnt“ IntelliJ (falsche Fehleranzeigen, merkwürdiges Verhalten). Lösung: „Invalidate Caches / Restart“ – das leert interne Speicher und startet die IDE neu.

So geht’s:

  1. Menü: File → Invalidate Caches / Restart…
  2. Im Dialog „Invalidate and Restart“ wählen.

Wirkung:

  • IntelliJ räumt temporäre Dateien auf, läd Indizes und Projekte neu.
  • Keine Angst: Dein Code bleibt erhalten!

Tipp: Sollte ein Projekt sich plötzlich seltsam verhalten, hilft diese Funktion oft Wunder.

2.2. Hilfe & Support in IntelliJ

Schnelle Hilfen:

  • Quick Documentation: Strg + Q (macOS: F1)
  • Find Action: Strg + Shift + A (macOS: ⌘ + Shift + A) – hier auch nach Hilfe suchen!
  • Kontextbezogene Hilfe: Rechtsklick auf Menüpunkt → „Help“
  • Online-Hilfe: Menü Help → „Help“ oder IntelliJ IDEA Online-Dokumentation

Weitere Anlaufstellen:

  • JetBrains Support Center
  • Stack Overflow (Fragen zu „intellij-idea“ taggen)
  • JetBrains Community-Foren

3. (Optional) Arbeitsorganisation & Zusammenarbeit

3.1. Aufgaben/Tasks (Kontextwechsel)

IntelliJ bietet eine Task-Funktion (Aufgabenverwaltung), z. B. zur Integration mit Issue-Trackern wie Jira, YouTrack oder GitHub Issues.

Vorteile:

  • Wechsel zwischen Aufgaben legt automatisch neue Branches an und speichert offenen Stand.
  • Du kannst schnell zwischen Aufgaben/Branch-Kontexten wechseln.

So nutzt du Tasks:

  1. Menü: View → Tool Windows → Tasks (oder Strg + Shift + F4)
  2. Neue Aufgabe starten: „Create New Task“
  3. Bei verknüpften Issue-Trackern: Aufgaben können automatisch angelegt und synchronisiert werden.

Tipp: Für Einsteiger reicht meist die Git-Branch-Verwaltung – Tasks werden vor allem in größeren Teams genutzt.

3.2. (Optional) Code-Review-Features (Ultimate Edition)

Mit der Ultimate Edition bietet IntelliJ eingebaute Tools für Code-Reviews und Pull Requests, z. B. für GitHub oder GitLab.

Features:

  • Anzeigen und Kommentieren von Pull Requests direkt in der IDE.
  • Inline-Diskussionen und Reviews, ohne IntelliJ zu verlassen.
  • Anzeigen von Review-Kommentaren im Editor.

So findest du die Review-Ansicht:

  • View → Tool Windows → Git (oder VCS → Git → Pull Requests)
  • Dort kannst du Pull Requests durchsuchen, ansehen und kommentieren.

Tipp: Für Open-Source-Projekte und Teamarbeit extrem praktisch – für private oder Einzelprojekte meist nicht nötig.

Übungsaufgaben (zum Selbststudium)

  1. Installiere das Plugin „Key Promoter X“ und lerne die ersten Shortcuts, die dir angezeigt werden.
  2. Probiere „Invalidate Caches / Restart“ aus, nachdem du eine kleine Änderung gemacht hast.
  3. Installiere und deaktiviere ein Plugin – prüfe, wie sich das auf deine IDE auswirkt.
  4. (Optional) Lege eine Aufgabe im Task-Tool an oder öffne die Pull-Request-Ansicht (wenn du Zugriff auf ein GitHub/GitLab-Repo hast).
  5. Nutze die Hilfe-Funktion (Strg + Q oder Strg + Shift + A), um eine Erklärung zu einem Menüpunkt zu finden.

Tipp: Nimm dir regelmäßig Zeit, um Plugins, Themes oder neue Tools zu entdecken und zu bewerten. So bleibt deine IDE optimal für deine Arbeitsweise eingestellt!

Mit diesen Kenntnissen kannst du IntelliJ IDEA optimal auf dich zuschneiden und Probleme gezielt beheben. Viel Spaß beim Ausprobieren!
