Handout: Arbeiten mit Ressourcen, Tools und Konfigurationen in IntelliJ IDEA

1. Arbeiten mit Ressourcen-Dateien

Ressourcendateien sind Konfigurationsdateien, die dein Projekt steuern – etwa .properties, .yml (YAML) oder .xml-Dateien. Besonders in Java- und Spring-Projekten sind sie sehr wichtig.

1.1. Properties-Dateien

Beispiel: application.properties für Spring Boot-Projekte.

Typische Inhalte:

server.port=8081
spring.datasource.url=jdbc:mysql://localhost:3306/meinedatenbank

Funktionen in IntelliJ:

  • Syntax-Highlighting: IntelliJ erkennt Properties-Dateien und hebt Schlüssel/Werte farbig hervor.
  • Autovervollständigung: Viele bekannte Properties werden vorgeschlagen.
  • Fehlermeldungen: Fehlerhafte Zeilen werden markiert.

Tipp: Mit Strg + F kannst du in Properties-Dateien gezielt nach Einstellungen suchen.

1.2. YAML-Dateien

Beispiel: application.yml für Spring Boot-Projekte.

Typische Inhalte:

server:
  port: 8082
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/meinedatenbank

Funktionen in IntelliJ:

  • Einrückungen werden geprüft und farblich hervorgehoben.
  • Faltbare Bereiche (mit - und Einrückung).
  • Autovervollständigung für viele Frameworks (z. B. Spring).

Tipp: Achte bei YAML immer auf korrekte Einrückung (z. B. 2 Leerzeichen pro Ebene).

1.3. XML-Dateien

Beispiel: pom.xml (Maven) oder Spring-Konfiguration.

Typische Inhalte:

<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.example</groupId>
  <artifactId>meine-anwendung</artifactId>
  <version>1.0.0</version>
</project>

Funktionen in IntelliJ:

  • Code-Vervollständigung für XML-Tags und Attribute.
  • Fehleranzeige (z. B. vergessene schließende Tags).
  • Automatische Formatierung (Alt + Shift + L / ⌘ + Alt + L).
  • Verlinkung zu Dokumentation (Ctrl + Click auf XML-Tags).

1.4. Strukturansicht auch für Ressourcen

Shortcut:

  • Alt + 7 (Windows/Linux), ⌘ + 7 (macOS) Öffnet das Strukturfenster für die aktuell geöffnete Datei.

Was bringt das?

  • Zeigt alle Schlüssel (Properties), Knoten (YAML/XML), Tags und Werte übersichtlich an.
  • Sehr praktisch für große Konfigurationsdateien!

Tipp: Auch nutzbar für HTML, JSON und viele andere Dateitypen.

2. Tools und zusätzliche Features

2.1. Integriertes Terminal

IntelliJ bringt ein vollwertiges Terminal mit, das in dein Projektverzeichnis eingebettet ist.

Shortcut:

  • Alt + F12 (Windows/Linux)
  • ⌘ + Option + T (macOS)

Typische Anwendungen:

  • Maven/Gradle-Befehle (z. B. mvn clean install)
  • Git-Befehle (git status, git commit)
  • Eigene Scripte ausführen

Vorteile:

  • Kein Wechsel zu einem externen Terminal nötig
  • Mehrere Tabs möglich (über + Symbol)
  • Terminal startet automatisch im Projektordner

2.2. Kurzer Ausblick: Datenbank-Tools (optional, für Interessierte)

IntelliJ (besonders die Ultimate-Version) bietet eingebaute Datenbank-Tools.

Features:

  • Verbindung zu Datenbanken (z. B. MySQL, PostgreSQL, H2)
  • Tabellen anzeigen, abfragen, Daten editieren
  • SQL direkt aus IntelliJ ausführen

Wie einrichten?

  1. Ansicht „Database“ öffnen (rechts oder über View → Tool Windows → Database).
  2. „+“ → Datenbanktyp wählen → Zugangsdaten eingeben.
  3. Verbindung testen, dann Datenbank wie gewohnt nutzen.

Tipp: So kannst du z. B. SQL-Skripte direkt im Projekt speichern und testen.

2.3. Build-Tools: Maven und Gradle

Die wichtigsten Java-Build-Tools sind Maven und Gradle. Sie automatisieren das Bauen, Testen und Verwalten von Abhängigkeiten deines Projekts.

Typische Aufgaben:

  • Projekt bauen/kompilieren
  • Tests ausführen
  • Abhängigkeiten aktualisieren
  • Pakete (JAR, WAR) erzeugen

2.3.1. Maven- und Gradle-Tasks direkt aus IntelliJ starten

  • IntelliJ erkennt automatisch pom.xml (Maven) und build.gradle (Gradle).
  • Links im „Maven“- oder „Gradle“-Fenster werden alle Tasks und Plugins angezeigt.
  • Doppelklick auf einen Task, z. B. clean, install oder test, um ihn auszuführen.

Tipp: Mit Rechtsklick auf pom.xml oder build.gradle kannst du das Projekt neu laden oder bestimmte Tasks ausführen.

2.3.2. "Reload Project" bei Änderungen

Immer wenn du Abhängigkeiten oder Einstellungen in pom.xml bzw. build.gradle änderst:

  • Klicke auf das kleine „Refresh“-Symbol im Maven/Gradle-Fenster ODER
  • Rechtsklick auf die Datei → „Reload Project“

Warum? So werden neue Abhängigkeiten oder Änderungen korrekt eingelesen.

Übungsaufgaben (zum Selbststudium)

  1. Öffne und bearbeite eine application.properties- und eine application.yml-Datei. Nutze Autovervollständigung und finde gezielt Einstellungen per Suche.
  2. Öffne das Strukturfenster (Alt + 7) in einer .properties, .yml oder .xml-Datei. Navigiere zu bestimmten Einstellungen oder Tags.
  3. Starte das integrierte Terminal und führe einen einfachen Befehl (z. B. dir, ls oder einen Maven/Gradle-Befehl) aus.
  4. Führe über das Maven-/Gradle-Fenster einen Build oder Test aus.
  5. (Optional) Erstelle eine Datenbankverbindung, öffne eine Tabelle und sieh dir die Daten an.

Zusätzliche Tipps

  • Nutze das integrierte Terminal, um nahtlos zwischen IDE und Kommandozeile zu arbeiten.
  • Mit der Strukturansicht findest du auch in langen Konfigurationsdateien schnell jede Einstellung.
  • Für größere Projekte ist es sinnvoll, Build- und Konfigurationsdateien in Git zu versionieren.

Mit diesen Funktionen hast du wichtige Werkzeuge für professionelle Java-Projekte an der Hand!
