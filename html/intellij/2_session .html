Handout: Code-Bearbeitung und Automatisierung in IntelliJ IDEA

1. Code-Vervollständigung und intelligente Hilfe

1.1. Code-Vervollständigung (Auto-Completion)

IntelliJ IDEA bietet eine leistungsstarke Codevervollständigung, die dir Methoden, Variablen, Klassennamen usw. passend zum Kontext vorschlägt.

Wie nutze ich das?

  • Windows/Linux: Strg + Leertaste
  • macOS: ⌃ + Leertaste
  • Schreibe z. B. den Anfang eines Klassennamens oder einer Methode und drücke den Shortcut.
  • IntelliJ zeigt eine Auswahl möglicher Vorschläge an, oft mit zusätzlichen Hinweisen (z. B. Typ oder aus welcher Bibliothek).

Beispiel: Du tippst System.o und drückst Strg + Leertaste – IDEA schlägt out vor, du kannst direkt System.out.println() verwenden.

Tipp:

  • Durch mehrmaliges Drücken von Strg + Leertaste bekommst du noch spezifischere Vorschläge.
  • Tippfehler werden oft erkannt und „intelligente“ Korrekturvorschläge gemacht.

1.2. Kontextabhängige Aktionen und Fehlerbehebung (Intention Actions)

Shortcut:

  • Windows/Linux: Alt + Enter
  • macOS: ⌥ + Enter

Was macht das?

  • Zeigt kontextabhängige Aktionen an: Fehler beheben, Quick-Fixes vorschlagen, Methoden implementieren, Imports ergänzen etc.
  • Immer wenn ein kleines Glühbirnchen oder roter Fehler angezeigt wird, einfach Alt + Enter drücken!

Beispiel: Du hast eine Klasse verwendet, aber keinen Import. Drücke Alt + Enter und wähle „Import class“.

1.3. Quick Documentation

Shortcut:

  • Windows/Linux: Strg + Q
  • macOS: F1

Nutzen: Zeigt die Dokumentation (Javadoc, Beschreibung, Parameter etc.) zum markierten Symbol direkt im Editor an – ohne Google oder externe Hilfe.

2. Codegenerierung

IntelliJ hilft dir, oft gebrauchte Code-Teile automatisch zu erzeugen.

Shortcut für Codegenerierung:

  • Windows/Linux: Alt + Einfügen
  • macOS: ⌘ + N

Was kann generiert werden?

  • Konstruktoren
  • Getter und Setter
  • toString(), equals(), hashCode()
  • Testklassen und Testmethoden

So geht’s:

  1. Cursor in die Klasse setzen.
  2. Shortcut drücken.
  3. Im Menü auswählen, was generiert werden soll.
  4. Felder/Methoden auswählen, Enter drücken – fertig!

3. Live Templates und Postfix-Operatoren

3.1. Live Templates

Live Templates sind Kürzel, die zu häufigen Code-Snippets erweitert werden.

Beispiele:

  • psvm → public static void main(String[] args) { … }
  • sout → System.out.println();

So geht’s:

  • Schreibe z. B. sout, drücke Tab – schon steht der komplette Code da.
  • Viele Templates sind schon voreingestellt; eigene lassen sich über die Einstellungen anlegen.

Tipp: Mit Strg + J (⌘ + J auf Mac) siehst du eine Liste aller verfügbaren Live Templates.

3.2. Postfix-Operatoren

Mit Postfix-Operatoren kannst du den Code nach dem Schreiben eines Ausdrucks schnell umwandeln.

Beispiele:

  • Schreibe list.for, drücke Tab: IntelliJ erstellt automatisch eine for-Schleife über die Liste.
  • Schreibe name.if, Tab: IntelliJ baut ein if-Statement um name.

Tipp: Liste aller Postfix-Templates findest du in den Einstellungen.

4. Refactoring

Refactoring bedeutet, den Code umzubauen, ohne die Funktion zu verändern. IntelliJ bietet viele Automatisierungen dafür!

Aktion Windows/Linux macOS Beschreibung
Umbenennen Shift + F6 Shift + F6 Variablen, Methoden, Klassen sicher umbenennen
Methode extrahieren Strg + Alt + M ⌘ + Alt + M Markierten Codeblock als neue Methode auslagern
Variable extrahieren Strg + Alt + V ⌘ + Alt + V Ausdruck als Variable definieren
Konstante extrahieren Strg + Alt + C ⌘ + Alt + C Wert als Konstante definieren

Ablauf:

  • Markiere den gewünschten Code.
  • Drücke den passenden Shortcut.
  • Folge dem Dialog, um Namen und Sichtbarkeit zu wählen.

5. Code-Struktur

5.1. Code formatieren

  • Windows/Linux: Strg + Alt + L
  • macOS: ⌘ + Alt + L Richtet Einrückungen, Abstände, Zeilenumbrüche nach dem Standard sauber aus.

5.2. Strukturansicht & Hierarchie

  • Strukturansicht (Alt + 7 / ⌘ + 7): Zeigt alle Klassen, Methoden, Felder der aktuellen Datei.

  • Hierarchie (Strg + H / ^ + H): Zeigt Vererbungs- oder Interface-Hierarchien an.

6. Produktivitätstipps

6.1. Mehrfach-Cursor/Multicaret

  • Alt + Klick: Setzt mehrere Cursors.
  • Strg + Alt + Shift + J (⌘ + Ctrl + Shift + G): Wählt alle Vorkommen des aktuellen Worts im Dokument aus.

6.2. Find Action

  • Strg + Shift + A (⌘ + Shift + A): Damit kannst du jede Funktion oder Einstellung über ein Suchfeld aufrufen.

6.3. Shortcut-Hilfe

  • Strg + J (⌘ + J): Zeigt die Liste der verfügbaren Live Templates.

Übungsaufgaben (zum Selbststudium)

  1. Nutze die Codevervollständigung: Schreibe eine neue Klasse und lasse IntelliJ Methoden und Felder automatisch vorschlagen.

  2. Generiere Getter/Setter: Erzeuge eine Klasse mit mehreren Feldern, lasse Getter und Setter automatisch erstellen.

  3. Wende ein Live Template an: Schreibe sout und prüfe, wie IntelliJ daraus System.out.println() macht.

  4. Benutze Refactoring: Extrahiere aus einer längeren Methode einen Teil als eigene Methode.

  5. Formatieren: Schreibe absichtlich „krummen“ Code und lasse ihn per Shortcut formatieren.

  6. Arbeite mit mehreren Cursors: Ändere denselben Variablennamen an mehreren Stellen gleichzeitig.

  7. Finde eine Aktion: Suche mit Strg + Shift + A nach einer Funktion (z. B. „Reformat Code“).

Weitere Tipps

  • Probiere die Postfix-Operatoren an eigenen Beispielen aus!
  • Experimentiere mit verschiedenen Live Templates – sie sparen viel Zeit.
  • Mach dich mit den Refactoring-Dialogen vertraut, um Änderungen sicher vorzunehmen.
  • Passe bei Bedarf die Keymap in den Einstellungen an dein System an (z. B. Mac vs. Windows).

Du bist jetzt bereit, effektiv und effizient Code mit IntelliJ IDEA zu schreiben und zu pflegen!
