Handout: Methodenreferenzen (Method References) in Java

1. Was sind Methodenreferenzen?

Methodenreferenzen sind eine besonders kompakte und lesbare Kurzschreibweise für Lambda-Ausdrücke, die einfach eine bestehende Methode aufrufen.

Statt

x -> Methode(x)

kannst du schreiben:

Klasse::Methode

Sie können überall dort verwendet werden, wo ein Lambda-Ausdruck eine bereits existierende Methode ohne zusätzliche Logik ausführt.

2. Unterschiedliche Formen von Methodenreferenzen

Es gibt vier Formen von Methodenreferenzen in Java:

a) Statische Methodenreferenz

Verweis auf eine statische Methode einer Klasse.

Syntax:

Klasse::statischeMethode

Beispiel:

Function<String, Integer> parser = Integer::parseInt;
System.out.println(parser.apply("123")); // 123

Entspricht dem Lambda:

s -> Integer.parseInt(s)

b) Instanzmethodenreferenz eines beliebigen Objekts

Verweis auf eine Instanzmethode einer Klasse – das Objekt wird als Argument übergeben.

Syntax:

Klasse::instanzMethode

Beispiel:

List<String> namen = Arrays.asList("anna", "bob", "clara");
namen.forEach(String::toUpperCase); // Funktioniert so nicht direkt, aber:
namen.forEach(s -> System.out.println(s.toUpperCase()));
namen.forEach(System.out::println); // Gültige Methodenreferenz

Ein weiteres Beispiel für Vergleich:

Comparator<String> cmp = String::compareToIgnoreCase;
int ergebnis = cmp.compare("Anna", "anna"); // 0

Entspricht dem Lambda:

(a, b) -> a.compareToIgnoreCase(b)

c) Instanzmethodenreferenz eines bestimmten Objekts

Verweis auf eine Methode eines bestimmten Objekts.

Syntax:

objekt::instanzMethode

Beispiel:

Consumer<String> drucker = System.out::println;
drucker.accept("Hallo Welt!"); // Gibt "Hallo Welt!" aus

Entspricht dem Lambda:

s -> System.out.println(s)

d) Konstruktorreferenz

Verweis auf einen Konstruktor – besonders praktisch bei Streams und Factory-Methoden.

Syntax:

Klasse::new

Beispiel:

Supplier<List<String>> listSupplier = ArrayList::new;
List<String> meineListe = listSupplier.get();

Entspricht dem Lambda:

() -> new ArrayList<>()

3. Beispiele und Einsatzmöglichkeiten

a) Mit Collections und Streams

List<String> namen = Arrays.asList("Anna", "Bernd", "Clara");

// Statt:
namen.forEach(s -> System.out.println(s));
// Besser:
namen.forEach(System.out::println);

// Umwandlung in Großbuchstaben:
List<String> upper = namen.stream()
    .map(String::toUpperCase)
    .collect(Collectors.toList());

b) Mit Comparator

List<String> namen = Arrays.asList("Anna", "Bernd", "Clara");

// Statt:
namen.sort((a, b) -> a.compareToIgnoreCase(b));
// Besser:
namen.sort(String::compareToIgnoreCase);

c) Mit eigenen Klassen

class Mathe {
    public static int verdoppeln(int x) { return x * 2; }
}

Function<Integer, Integer> func = Mathe::verdoppeln;
System.out.println(func.apply(5)); // 10

d) Konstruktorreferenz in Streams

Stream<String> stream = Stream.of("A", "B", "C");
List<String> liste = stream.collect(Collectors.toCollection(ArrayList::new));

4. Übungen zum Selbststudium

  1. Schreibe einen Lambda-Ausdruck, dann wandle ihn in eine Methodenreferenz um:

    • Beispiel: s -> System.out.println(s) ⇒ System.out::println

  2. Sortiere eine Liste von Strings nach natürlicher Reihenfolge mit Methodenreferenz.

  3. Erzeuge eine neue Liste mit der Konstruktorreferenz ArrayList::new.

  4. Schreibe eine eigene statische Hilfsmethode und verwende sie als Methodenreferenz in einer Stream-Operation.

5. Merksätze

  • Methodenreferenzen sind noch kompakter als Lambda-Ausdrücke und erhöhen die Lesbarkeit.
  • Es gibt vier Formen: statisch, Instanz beliebigen Objekts, Instanz eines bestimmten Objekts, Konstruktor.
  • Immer dann verwenden, wenn der Lambda-Ausdruck nur eine bestehende Methode aufruft.
