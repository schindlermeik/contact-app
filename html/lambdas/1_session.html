Handout: Einführung in funktionale Programmierung in Java

1. Was ist funktionale Programmierung?

Funktionale Programmierung ist ein Programmierparadigma, bei dem das Arbeiten mit Funktionen im Mittelpunkt steht – Funktionen gelten hier als „erste Bürger“ („First-Class Citizens“), das heißt, sie können wie Werte behandelt, übergeben, zurückgegeben und in Variablen gespeichert werden.

Merkmale funktionaler Programmierung:

  • Funktionen als Werte: Funktionen können an Methoden übergeben und von Methoden zurückgegeben werden.
  • Immutabilität: Daten werden möglichst nicht verändert, sondern neue erzeugt.
  • Keine Seiteneffekte: Funktionen sollen (nach Möglichkeit) nur Eingaben in Ausgaben verwandeln, ohne den Zustand außerhalb zu beeinflussen.
  • Höhere Ordnung: Es gibt sogenannte „Higher-Order Functions“, also Funktionen, die andere Funktionen als Argumente oder Rückgabewert nutzen.

Beispiel (pseudocode):

Ergebnis = berechne(x -> x * 2)

Hier wird eine Funktion als Argument übergeben.

2. Unterschiede zur objektorientierten Programmierung

Java ist traditionell objektorientiert – dort stehen Objekte im Mittelpunkt, also Daten + Verhalten in einer Einheit.

Vergleich:

Aspekt Objektorientiert (OOP) Funktional
Zentrale Bausteine Objekte (Klassen, Instanzen) Funktionen
Fokus Zustände, Methoden, Attribute Datenfluss, Funktionskomposition
Typische Aktion Methode auf Objekt ausführen: obj.methode() Funktion auf Wert anwenden: funktion(wert)
Datenänderung Oft: Objektzustand ändern (mutabel) Möglichst keine Zustandsänderung (immutable)
Vererbung/Polymorphie Häufig Seltener, Funktionen im Vordergrund

OOP Beispiel:

Auto auto = new Auto();
auto.fahren(100);

Funktional Beispiel (mit Lambda):

List<Integer> zahlen = Arrays.asList(1, 2, 3);
zahlen.forEach(x -> System.out.println(x * 2));

Hier wird eine Funktion als Argument übergeben.

3. Motivation: Warum funktionale Konzepte in Java?

  • Moderne Anwendungen: Immer mehr Aufgaben verlangen nach kompaktem, flexiblem, nebenläufigem und leicht testbarem Code.
  • Lesbarkeit & Wartbarkeit: Viele Aufgaben lassen sich als Funktionsfolgen (Transformationen) viel kürzer und verständlicher ausdrücken.
  • Parallelisierung: Funktionaler Stil (ohne Seiteneffekte) lässt sich leichter parallelisieren – z. B. mit der Streams-API.
  • Java 8 und neuer: Seit Java 8 sind Lambda-Ausdrücke und funktionale Interfaces Teil der Sprache. Damit können Entwickler viele Vorteile funktionaler Programmierung nutzen, ohne Java zu verlassen.
  • Trends aus anderen Sprachen: Viele populäre Sprachen (JavaScript, Scala, Python, Haskell) haben funktionale Konzepte – Entwickler sind diese Arbeitsweise gewohnt.

Praxisbeispiel:

  • Eine Liste filtern und verarbeiten – klassisch (OOP):

    List<String> ergebnis = new ArrayList<>();
for (String s : namen) {
    if (s.length() > 4) {
        ergebnis.add(s.toUpperCase());
    }
}

  • Funktional (ab Java 8):

    List<String> ergebnis = namen.stream()
    .filter(s -> s.length() > 4)
    .map(String::toUpperCase)
    .collect(Collectors.toList());

Vorteile:

  • Kürzerer, prägnanterer Code
  • Besser testbar
  • Weniger Fehler durch „nebenläufigen“ Zustand

5. Merksätze

  • Funktionale Programmierung setzt auf Funktionen als Werte, Immutabilität und Datenfluss.
  • In Java ergänzt sie die Objektorientierung und ist seit Java 8 fest integriert.
  • Sie ermöglicht prägnanten, modularen und oft weniger fehleranfälligen Code.
