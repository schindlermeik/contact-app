Handout: Praktische Anwendungsfälle von Lambdas

1. Lambdas im Zusammenspiel mit Collections

Mit Lambda-Ausdrücken lassen sich Operationen auf Collections wie Sortieren, Filtern oder Mapping besonders kompakt und ausdrucksstark schreiben.

a) Sortieren

Beispiel: Sortiere eine Liste von Strings nach Länge.

List<String> namen = Arrays.asList("Anna", "Bernd", "Christoph");

// Mit Lambda-Ausdruck:
namen.sort((a, b) -> a.length() - b.length());
System.out.println(namen); // [Anna, Bernd, Christoph]

// Mit Methodenreferenz (Vergleich ignoriert Groß-/Kleinschreibung):
namen.sort(String::compareToIgnoreCase);
System.out.println(namen); // [Anna, Bernd, Christoph]

b) Filtern

Beispiel: Finde alle Namen, die mit „A“ beginnen.

List<String> namen = Arrays.asList("Anna", "Bernd", "Anja", "Clara");

// Mit Streams und Lambda:
List<String> gefiltert = namen.stream()
    .filter(n -> n.startsWith("A"))
    .collect(Collectors.toList());

System.out.println(gefiltert); // [Anna, Anja]

c) Mapping

Beispiel: Wandle alle Namen in Großbuchstaben um.

List<String> namen = Arrays.asList("Anna", "Bernd", "Clara");

List<String> upper = namen.stream()
    .map(String::toUpperCase)
    .collect(Collectors.toList());

System.out.println(upper); // [ANNA, BERND, CLARA]

2. Lambdas und Streams-API (z. B. map, filter, reduce)

Die Streams-API (seit Java 8) kombiniert die funktionale Arbeitsweise mit leistungsfähigen Datenoperationen.

a) map – Transformation jedes Elements

List<Integer> zahlen = Arrays.asList(1, 2, 3);

List<Integer> quadriert = zahlen.stream()
    .map(x -> x * x)
    .collect(Collectors.toList());

System.out.println(quadriert); // [1, 4, 9]

b) filter – Auswahl bestimmter Elemente

List<Integer> zahlen = Arrays.asList(1, 2, 3, 4, 5);

List<Integer> gerade = zahlen.stream()
    .filter(x -> x % 2 == 0)
    .collect(Collectors.toList());

System.out.println(gerade); // [2, 4]

c) reduce – Aggregation/Reduktion der Elemente

Beispiel: Summiere alle Zahlen in einer Liste.

List<Integer> zahlen = Arrays.asList(1, 2, 3, 4);

int summe = zahlen.stream()
    .reduce(0, (a, b) -> a + b); // Startwert: 0

System.out.println(summe); // 10

3. Eigene Methoden, die Functional Interfaces als Parameter nutzen

Du kannst Methoden schreiben, die ein Functional Interface als Argument akzeptieren – zum Beispiel, um flexibles Verhalten zu ermöglichen.

Beispiel: Schreibe eine Methode, die eine Liste von Strings und ein Predicate<String> erhält und alle passenden Strings ausgibt.

public static void filterUndAusgeben(List<String> daten, Predicate<String> kriterium) {
    for (String s : daten) {
        if (kriterium.test(s)) {
            System.out.println(s);
        }
    }
}

// Verwendung:
List<String> namen = Arrays.asList("Anna", "Bernd", "Anja", "Clara");
filterUndAusgeben(namen, s -> s.length() > 4); // Gibt nur "Bernd" und "Clara" aus

Noch kompakter mit Streams:

public static void filterUndAusgeben(List<String> daten, Predicate<String> kriterium) {
    daten.stream().filter(kriterium).forEach(System.out::println);
}

4. Übungen zum Selbststudium

  1. Sortiere eine Liste von Zahlen absteigend mit einem Lambda-Ausdruck.
  2. Filtere aus einer Liste von Strings alle, die länger als fünf Zeichen sind.
  3. Verwende map, um die Längen aller Wörter in einer Liste als neue Liste zu erzeugen.
  4. Schreibe eine eigene Methode, die als Parameter ein Function<Integer, Integer>-Interface bekommt und eine Liste von Zahlen damit transformiert.

5. Merksätze

  • Lambdas machen typische Operationen auf Collections und Streams kurz und lesbar.
  • Streams ermöglichen das Verketteln von Operationen wie filter, map, reduce.
  • Mit eigenen Methoden, die Functional Interfaces akzeptieren, wird dein Code modular und flexibel.
