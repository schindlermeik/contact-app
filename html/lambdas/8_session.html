Handout: Praktische Übungen und Beispiele zu Lambdas und Functional Interfaces

1. Schreibe eigene Functional Interfaces und nutze sie mit Lambdas

Aufgabe 1

Definiere ein Functional Interface Rechner, das zwei Zahlen als Argumente nimmt und einen Wert zurückgibt. Implementiere es mit einem Lambda für Addition, Subtraktion und Multiplikation.

Musterlösung:

@FunctionalInterface
public interface Rechner {
    int berechne(int a, int b);
}

// Lambdas für verschiedene Operationen:
Rechner addition = (a, b) -> a + b;
Rechner subtraktion = (a, b) -> a - b;
Rechner multiplikation = (a, b) -> a * b;

// Verwendung:
System.out.println(addition.berechne(4, 2));        // 6
System.out.println(subtraktion.berechne(4, 2));     // 2
System.out.println(multiplikation.berechne(4, 2));  // 8

2. Refactoring: Ersetze anonyme Klassen durch Lambda-Ausdrücke

Aufgabe 2

Der folgende Code nutzt eine anonyme Klasse. Ersetze sie durch einen Lambda-Ausdruck.

Comparator<String> cmp = new Comparator<String>() {
    @Override
    public int compare(String s1, String s2) {
        return s1.length() - s2.length();
    }
};

Lösung mit Lambda:

Comparator<String> cmp = (s1, s2) -> s1.length() - s2.length();

Verwendung:

List<String> namen = Arrays.asList("Anna", "Bernd", "Clara");
namen.sort(cmp);
System.out.println(namen); // Sortiert nach Länge

3. Anwenden von Methodenreferenzen

Aufgabe 3

Wandle den folgenden Lambda-Ausdruck in eine Methodenreferenz um:

List<String> namen = Arrays.asList("Anna", "Bernd", "Clara");
namen.forEach(s -> System.out.println(s));

Lösung mit Methodenreferenz:

namen.forEach(System.out::println);

Weitere Beispiele:

  • Umwandeln in Großbuchstaben mit Methodenreferenz:

    List<String> upper = namen.stream()
    .map(String::toUpperCase)
    .collect(Collectors.toList());

  • Vergleich mit Methodenreferenz:

    namen.sort(String::compareToIgnoreCase);

4. Eigenes Beispiel für Methoden als Parameter

Aufgabe 4

Schreibe eine Methode, die ein Functional Interface als Parameter nimmt und nutze einen Lambda-Ausdruck und eine Methodenreferenz.

Beispiel:

@FunctionalInterface
interface TextVerarbeiter {
    String verarbeite(String text);
}

public static void wendeAnUndGibAus(String eingabe, TextVerarbeiter tv) {
    System.out.println(tv.verarbeite(eingabe));
}

// Verwendung mit Lambda:
wendeAnUndGibAus("Hallo", s -> s.toUpperCase());    // HALLO

// Verwendung mit Methodenreferenz:
wendeAnUndGibAus("Hallo", String::toLowerCase);     // hallo

5. Reflexionsfragen (Selbststudium)

  1. Wo sind Lambda-Ausdrücke für dich besonders nützlich? Wo sind Methodenreferenzen besser?
  2. Wie würde ein Codeabschnitt ohne Lambdas/anonyme Klassen im Vergleich aussehen?
  3. Schreibe ein eigenes Beispiel mit einem Functional Interface, das einen Wert prüft (Predicate<T>).

6. Merksätze

  • Eigene Functional Interfaces helfen dir, Funktionen flexibel als Parameter zu nutzen.
  • Lambdas machen den Code kürzer und übersichtlicher als anonyme Klassen.
  • Methodenreferenzen sind noch kompakter, wenn sie nur eine bestehende Methode aufrufen.
  • Nutze Methoden als Parameter für wiederverwendbaren, flexiblen und modernen Java-Code.
