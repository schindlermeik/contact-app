Handout: Einführung in Lambda-Ausdrücke

1. Syntax und Grundstruktur eines Lambda-Ausdrucks

Lambda-Ausdrücke sind eine kompakte Möglichkeit, anonyme Funktionen (ohne Namen) zu schreiben, die ein Functional Interface implementieren. Sie bestehen aus:

  • Parameterliste (optional mit Typangabe)
  • Pfeil-Operator ->
  • Rumpf (ein Ausdruck oder ein Block mit Anweisungen)

Grundform:

(Parameter) -> Ausdruck
(Parameter) -> { Anweisungen }

Beispiele:

a) Ohne Typangabe (der Compiler erkennt den Typ automatisch):

x -> x * x

b) Mit mehreren Parametern:

(a, b) -> a + b

c) Mit expliziten Typen:

(int a, int b) -> a * b

d) Mit Block:

(a, b) -> {
    int summe = a + b;
    return summe * 2;
}

e) Kein Parameter (z. B. für Runnable):

() -> System.out.println("Hallo Welt!")

2. Vergleich: Anonyme Klasse vs. Lambda-Ausdruck

Vor Lambda-Ausdrücken musste man oft eine „anonyme Klasse“ schreiben, um ein Functional Interface schnell zu implementieren. Das war sehr umständlich.

a) Anonyme Klasse:

Runnable r1 = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hallo aus der anonymen Klasse!");
    }
};
r1.run();

b) Lambda-Ausdruck:

Runnable r2 = () -> System.out.println("Hallo aus dem Lambda!");
r2.run();

Vorteile von Lambdas:

  • Kürzer
  • Klarer
  • Lesbarer
  • Kein Overhead von Klassendefinitionen

Vergleichstabelle:

Anonyme Klasse Lambda-Ausdruck
Syntax Viel Boilerplate-Code Sehr kompakt
Lesbarkeit Oft verschachtelt, schwer lesbar Direkt und übersichtlich
Typ-Inferenz Nein, Typ muss genannt werden Ja, meist keine Typangabe nötig

3. Vorteile und Grenzen von Lambda-Ausdrücken

Vorteile

  • Kompakter Code: Deutlich weniger Schreibaufwand.
  • Bessere Lesbarkeit: Fokus auf das Wesentliche – das Verhalten.
  • Typinferenz: Der Compiler erkennt meist die Typen automatisch.
  • Einfachere Übergabe von Verhalten: Funktionen können einfach an Methoden weitergegeben werden (z. B. für Sortierung, Filter, Mapping, Event-Handling).

Grenzen

  • Nur für Functional Interfaces: Lambdas funktionieren nur dort, wo genau ein abstrakte Methode benötigt wird.
  • Kein Zugriff auf eigene Methoden oder Felder: Im Lambda gibt es keine eigene Klasse, daher keine eigenen Member.
  • Nicht für komplexe Logik: Für sehr lange oder komplexe Methoden ist eine klassische Methode meist übersichtlicher.
  • Kein Zustand: Lambdas sind am besten für zustandslose oder kleine, zustandsarme Operationen geeignet.
  • Eingeschränkte Ausnahmebehandlung: Lambda-Ausdrücke können Ausnahmen nur behandeln, wenn das Functional Interface sie auch zulässt.

4. Übungen zum Selbststudium

  1. Schreibe einen Lambda-Ausdruck, der zwei Zahlen multipliziert und das Ergebnis zurückgibt ((a, b) -> a * b).
  2. Erstelle eine Runnable-Variable mit einem Lambda, das eine Meldung ausgibt.
  3. Vergleiche eine anonyme Klasse und einen Lambda-Ausdruck, die beide ein Comparator<String> sind, und sortiere eine Liste von Strings nach Länge.
  4. Überlege, wo im eigenen Code Lambdas sinnvoll eingesetzt werden könnten – und wo lieber nicht.

5. Merksätze

  • Lambda-Ausdrücke machen den Code kürzer, klarer und oft moderner.
  • Sie sind ein zentrales Element funktionaler Programmierung in Java.
  • Lambdas ersetzen anonyme Klassen, wo immer ein Functional Interface gebraucht wird.
