Handout: Verwendung von Lambda-Ausdrücken mit Functional Interfaces

1. Lambda-Ausdrücke als Implementierung für Functional Interfaces

Ein Lambda-Ausdruck kann immer dann verwendet werden, wenn ein Functional Interface gefragt ist, also ein Interface mit genau einer abstrakten Methode. Der Lambda-Ausdruck „implementiert“ diese Methode und kann an jeder Stelle eingesetzt werden, an der eine Instanz dieses Interfaces erwartet wird.

Beispiele:

a) Eigenes Functional Interface

@FunctionalInterface
public interface Rechner {
    int berechne(int a, int b);
}

// Lambda als Implementierung:
Rechner summe = (a, b) -> a + b;
System.out.println(summe.berechne(5, 3)); // 8

b) Java-Standardinterfaces

Runnable r = () -> System.out.println("Task läuft");
r.run();

Comparator<String> laengenVergleich = (s1, s2) -> s1.length() - s2.length();
System.out.println(laengenVergleich.compare("Hund", "Katze")); // -1

2. Typinferenz bei Lambda-Ausdrücken

Typinferenz bedeutet, dass der Java-Compiler die Typen der Parameter in Lambda-Ausdrücken automatisch erkennt. Du musst sie also nicht explizit angeben, solange der Kontext klar ist.

Beispiel:

Comparator<Integer> cmp1 = (a, b) -> a - b;          // Typen werden abgeleitet
Comparator<Integer> cmp2 = (Integer a, Integer b) -> a - b; // Typen explizit angegeben
  • Beide Varianten sind möglich – die Kurzform ist aber meist übersichtlicher.

Achtung: Wenn der Kontext nicht eindeutig ist (z. B. bei mehreren möglichen Interfaces), muss der Typ explizit angegeben werden.

3. Übergabe von Lambda-Ausdrücken an Methoden

Viele Methoden der Java-API erwarten Functional Interfaces als Argumente. Dort kannst du Lambda-Ausdrücke direkt übergeben.

a) forEach (Consumer)

List<String> namen = Arrays.asList("Anna", "Bob", "Chris");
namen.forEach(s -> System.out.println(s.toUpperCase()));
  • Das Lambda (s -> ...) ist die Implementierung des Consumer<String>-Interfaces.

b) sort (Comparator)

List<String> namen = Arrays.asList("Anna", "Bernd", "Clara");
namen.sort((a, b) -> b.compareTo(a)); // Sortierung absteigend
System.out.println(namen);
  • Das Lambda ist ein Comparator<String>.

c) filter (Predicate) – im Stream-API-Kontext

List<String> namen = Arrays.asList("Max", "Moni", "Chris");
List<String> gefiltert = namen.stream()
    .filter(name -> name.startsWith("M"))
    .collect(Collectors.toList());
System.out.println(gefiltert); // [Max, Moni]
  • Das Lambda (name -> ...) implementiert Predicate<String>.

4. Übungen zum Selbststudium

  1. Schreibe ein eigenes Functional Interface TextTest, das prüft, ob ein String ein bestimmtes Kriterium erfüllt. Implementiere es mit einem Lambda, das prüft, ob der Text länger als 8 Zeichen ist.

  2. Erstelle eine Liste von Zahlen. Nutze forEach, um alle Werte verdoppelt auszugeben.

  3. Sortiere eine Liste von Strings nach ihrer Länge mit einem Lambda-Ausdruck.

  4. Filtere mit Streams alle Namen, die mit einem bestimmten Buchstaben beginnen.

5. Merksätze

  • Lambda-Ausdrücke sind immer eine Implementierung für ein Functional Interface.
  • Der Java-Compiler erkennt die Typen der Parameter in Lambdas meist automatisch (Typinferenz).
  • Viele Standardmethoden in Java (wie forEach, sort, filter) erwarten Functional Interfaces und sind für Lambdas gemacht.
