Handout: Was sind Functional Interfaces?

1. Definition und Eigenschaften eines Functional Interface

Ein Functional Interface (deutsch: funktionales Interface) ist ein Interface, das genau eine abstrakte Methode deklariert. Es bildet das Bindeglied zwischen Java und der funktionalen Programmierung, da es damit möglich ist, Funktionen als Werte zu behandeln – z. B. durch Lambda-Ausdrücke oder Methodenreferenzen.

Wichtig:

  • Ein Functional Interface kann beliebig viele default- oder static-Methoden enthalten, aber nur eine „abstrakte“ (d. h. zu implementierende) Methode.
  • Diese eine abstrakte Methode heißt auch SAM (Single Abstract Method).

Beispiel:

@FunctionalInterface
public interface Rechner {
    int berechne(int a, int b);
}
  • Das Interface Rechner ist ein Functional Interface, da es nur eine abstrakte Methode (berechne) besitzt.

2. Die Annotation @FunctionalInterface

Mit der Annotation @FunctionalInterface kann (und sollte) explizit markiert werden, dass ein Interface funktional gemeint ist. Das ist keine Pflicht, aber gute Praxis, weil der Compiler dann einen Fehler meldet, falls versehentlich mehr als eine abstrakte Methode vorhanden ist.

Beispiel:

@FunctionalInterface
public interface Printer {
    void print(String text);

    // default-Methoden sind erlaubt
    default void printTwice(String text) {
        print(text);
        print(text);
    }
}
  • Der Compiler akzeptiert mehrere default- und static-Methoden, aber maximal eine abstrakte Methode.

Vorteil: Mehr Klarheit, weniger Fehler und bessere Wartbarkeit im Team.

3. Beispiele aus der Java-Standardbibliothek

Viele Interfaces in der Java-Standardbibliothek sind funktionale Interfaces. Sie sind die Grundlage für Lambda-Ausdrücke und Methodenreferenzen.

Hier einige der wichtigsten und häufigsten Functional Interfaces aus java.lang und java.util.function:

a) Runnable

  • Deklariert: void run()
  • Wird z. B. für Threads verwendet.
Runnable r = () -> System.out.println("Läuft im Thread!");
new Thread(r).start();

b) Comparator<T>

  • Deklariert: int compare(T o1, T o2)
  • Zum Vergleichen und Sortieren von Objekten.
Comparator<String> cmp = (a, b) -> a.length() - b.length();
List<String> namen = Arrays.asList("Anna", "Bob", "Christina");
namen.sort(cmp);

c) Consumer<T>

  • Deklariert: void accept(T t)
  • „Verbraucher“ – nimmt ein Argument und tut etwas damit.
Consumer<String> printer = s -> System.out.println(s.toUpperCase());
printer.accept("Hallo");

d) Function<T, R>

  • Deklariert: R apply(T t)
  • Wandelt einen Wert von Typ T in einen Wert vom Typ R um.
Function<String, Integer> lengthFunc = s -> s.length();
int len = lengthFunc.apply("Lambda");

e) Predicate<T>

  • Deklariert: boolean test(T t)
  • Prüft eine Bedingung für ein Objekt, liefert true oder false.
Predicate<String> lang = s -> s.length() > 5;
System.out.println(lang.test("OpenAI")); // true
System.out.println(lang.test("Java"));   // false

4. Übungen zum Selbststudium

  1. Schreibe ein eigenes Functional Interface MatheAktion, das zwei Zahlen als Argumente nimmt und einen Wert zurückliefert.
  2. Erstelle einen Lambda-Ausdruck, der dieses Interface für eine Addition implementiert.
  3. Verwende ein Consumer<Integer>, um alle Werte einer Liste zu verdoppeln und auszugeben.
  4. Finde in der Java-API-Dokumentation weitere Functional Interfaces, z. B. Supplier<T> und UnaryOperator<T>. Wofür könnte man sie nutzen?

5. Merksätze

  • Functional Interfaces sind die Voraussetzung für Lambda-Ausdrücke in Java.
  • Sie müssen genau eine abstrakte Methode besitzen.
  • Die Annotation @FunctionalInterface hilft, Fehler frühzeitig zu erkennen.
  • Die Java-Standardbibliothek enthält viele vordefinierte Functional Interfaces – für nahezu alle Anwendungsfälle.
