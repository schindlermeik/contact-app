Handout: Praxis – Logging mit SLF4J & Logback

3.1 Kurzes Live-Coding/Demo

A. Beispielprojekt aufsetzen

Voraussetzung:

  • Java & Maven (oder Gradle) installiert

Projektstruktur (Empfehlung):

logging-demo/
├── src/
│   └── main/
│       ├── java/
│       │   └── de/
│       │       └── beispiel/
│       │           └── LoggingDemo.java
│       └── resources/
│           └── logback.xml
└── pom.xml

B. Abhängigkeiten (Maven)

In die pom.xml einfügen:

<dependencies>
    <!-- SLF4J-API -->
    <dependency>
        <groupId>org.slf4j</groupId>
        <artifactId>slf4j-api</artifactId>
        <version>2.0.9</version>
    </dependency>
    <!-- Logback (Implementierung) -->
    <dependency>
        <groupId>ch.qos.logback</groupId>
        <artifactId>logback-classic</artifactId>
        <version>1.4.14</version>
    </dependency>
</dependencies>

C. Logger deklarieren und nutzen

Beispielcode (LoggingDemo.java):

package de.beispiel;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class LoggingDemo {
    private static final Logger logger = LoggerFactory.getLogger(LoggingDemo.class);

    public static void main(String[] args) {
        logger.trace("Dies ist eine TRACE-Nachricht (sehr fein).");
        logger.debug("Dies ist eine DEBUG-Nachricht (Detail für Entwickler).");
        logger.info("Dies ist eine INFO-Nachricht (allgemeine Infos).");
        logger.warn("Achtung! Dies ist eine WARN-Nachricht.");
        logger.error("Fehler! Dies ist eine ERROR-Nachricht.");

        int zahl = 42;
        logger.info("Beispiel für Variablen im Log: zahl={}", zahl);

        try {
            wirfFehler();
        } catch (Exception e) {
            logger.error("Ein Fehler ist aufgetreten!", e);
        }
    }

    private static void wirfFehler() throws Exception {
        throw new Exception("Demo-Exception für Logging.");
    }
}

Erklärung:

  • Pro Klasse ein Logger (LoggerFactory.getLogger(…)).
  • Verschiedene Log-Levels werden genutzt.
  • Variablen können mit {} eingebunden werden.
  • Fehler werden inkl. Stacktrace geloggt.

D. Beispiel-Konfiguration (logback.xml):

<configuration>
    <appender name="STDOUT" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>[%d{HH:mm:ss}] %-5level %logger{36} - %msg%n</pattern>
        </encoder>
    </appender>
    <root level="DEBUG">
        <appender-ref ref="STDOUT" />
    </root>
</configuration>

Erklärung:

  • Log-Ausgabe auf Konsole.
  • Zeigt Zeit, Level, Logger-Name, Nachricht.
  • root level="DEBUG" gibt alles ab DEBUG-Level aus (DEBUG, INFO, WARN, ERROR).

E. Anwendung starten

  • Mit IDE oder
  • mvn clean compile exec:java -Dexec.mainClass=de.beispiel.LoggingDemo

Ergebnis: Du siehst alle Log-Ausgaben in der Konsole.

3.2 Konfigurationsänderungen live zeigen

A. Log-Level ändern und Ergebnis beobachten

  • Ändere in logback.xml das Root-Level:

    <root level="INFO">
    <appender-ref ref="STDOUT" />
</root>

  • Erwartung: TRACE- und DEBUG-Nachrichten werden NICHT mehr ausgegeben, nur noch INFO, WARN und ERROR.

Teste das Verhalten:

  • Anwendung starten → weniger Log-Ausgaben.

B. Appender & Log-Format anpassen

1. Log-Format verändern:

  • In <pattern> ergänzen, z.B. um Thread und Methode:

    <pattern>[%d{HH:mm:ss}] [%thread] %-5level %logger{36} %M - %msg%n</pattern>

  • Anwendung starten → Du siehst Thread und Methodennamen im Log.

2. Einen FileAppender ergänzen:

<appender name="FILE" class="ch.qos.logback.core.FileAppender">
    <file>logs/demo.log</file>
    <encoder>
        <pattern>[%d{yyyy-MM-dd HH:mm:ss.SSS}] %-5level %logger{36} - %msg%n</pattern>
    </encoder>
</appender>

  • In <root> zusätzlich:

    <appender-ref ref="FILE" />

Ergebnis: Log-Ausgaben erscheinen jetzt zusätzlich in der Datei logs/demo.log.

Zusammenfassung:

  • SLF4J + Logback lassen sich schnell einrichten.
  • Das Log-Level steuert, was ausgegeben wird.
  • Mit Appender und Pattern gestaltest du Ziel und Aussehen der Logs flexibel.

Erweiterungen:

  • Probiere weitere Log-Levels.
  • Ergänze einen RollingFileAppender für produktiven Einsatz.
  • Passe das Pattern an eigene Wünsche an.
