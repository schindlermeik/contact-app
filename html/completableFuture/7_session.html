Handout: Praxis – Hands-On und Übungen mit CompletableFuture

7.1 Implementiere einen einfachen asynchronen Workflow mit CompletableFuture

Ziel: Eine Aufgabe wird asynchron gestartet, und das Ergebnis wird weiterverarbeitet – ohne dass der Haupt-Thread blockiert.

Beispiel:

import java.util.concurrent.CompletableFuture;

public class SimpleAsyncWorkflow {
    public static void main(String[] args) {
        System.out.println("Start Hauptprogramm");

        CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> {
            // Simuliere eine aufwändige Berechnung
            try { Thread.sleep(1000); } catch (InterruptedException ignored) {}
            return 21;
        });

        future.thenAccept(result -> System.out.println("Ergebnis (21*2): " + (result * 2)));

        System.out.println("Hauptprogramm läuft weiter ...");
        try { Thread.sleep(2000); } catch (InterruptedException ignored) {} // Damit das Programm nicht sofort endet
    }
}
  • Erklärung: Das Hauptprogramm läuft weiter, während die Berechnung im Hintergrund läuft. Sobald das Ergebnis da ist, wird es verarbeitet.

7.2 Verkette mehrere Futures und kombiniere ihre Ergebnisse

a) Verkettung abhängiger Aufgaben mit thenCompose

Beispiel:

CompletableFuture<String> userIdFuture = CompletableFuture.supplyAsync(() -> "user-123");

CompletableFuture<String> userDetailsFuture = userIdFuture.thenCompose(userId ->
    CompletableFuture.supplyAsync(() -> "Details für " + userId)
);

userDetailsFuture.thenAccept(details -> System.out.println(details));
  • Erklärung: Die zweite Aufgabe hängt vom Ergebnis der ersten ab und startet erst, wenn die erste fertig ist.

b) Parallele Futures kombinieren mit thenCombine

Beispiel:

CompletableFuture<Integer> f1 = CompletableFuture.supplyAsync(() -> 10);
CompletableFuture<Integer> f2 = CompletableFuture.supplyAsync(() -> 20);

CompletableFuture<Integer> sumFuture = f1.thenCombine(f2, (a, b) -> a + b);

sumFuture.thenAccept(sum -> System.out.println("Summe: " + sum));
  • Erklärung: Beide Aufgaben laufen parallel, das Ergebnis wird kombiniert, sobald beide fertig sind.

7.3 Fehlerfälle simulieren und abfangen

a) Fehler simulieren mit exceptionally

Beispiel:

CompletableFuture<Integer> errorFuture = CompletableFuture.supplyAsync(() -> {
    if (true) throw new RuntimeException("Simulierter Fehler");
    return 1;
});

errorFuture
    .exceptionally(ex -> {
        System.out.println("Fehler aufgetreten: " + ex.getMessage());
        return 0; // Ersatzwert
    })
    .thenAccept(result -> System.out.println("Ergebnis trotz Fehler: " + result));

b) Fehlerbehandlung mit handle

Beispiel:

CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10 / 0);

future.handle((result, ex) -> {
    if (ex != null) {
        System.out.println("Division durch null, Fehler: " + ex.getMessage());
        return -1;
    }
    return result;
}).thenAccept(result -> System.out.println("Ergebnis: " + result));

7.4 (Optional) Integration in eine bestehende Anwendung oder API-Aufruf

a) Asynchroner HTTP-Request mit Java 11 HTTP-Client

Beispiel:

import java.net.URI;
import java.net.http.*;
import java.util.concurrent.*;

public class HttpAsyncExample {
    public static void main(String[] args) throws Exception {
        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://jsonplaceholder.typicode.com/posts/1"))
            .build();

        CompletableFuture<HttpResponse<String>> responseFuture =
            client.sendAsync(request, HttpResponse.BodyHandlers.ofString());

        responseFuture
            .thenAccept(response -> System.out.println("Antwort: " + response.body()))
            .join(); // Warten, bis fertig (nur im Demo-Code!)
    }
}
  • Erklärung: Der HTTP-Request wird asynchron ausgeführt, das Ergebnis per Callback verarbeitet.

b) Integration in bestehende (blocking) APIs

  • Bestehende Methoden, die blockierend sind, können per supplyAsync() in den asynchronen Workflow integriert werden:

    public String blockingDatabaseCall() {
    // ... blockierender Code
}

CompletableFuture<String> dbFuture = CompletableFuture.supplyAsync(() -> blockingDatabaseCall());
dbFuture.thenAccept(result -> System.out.println("DB-Ergebnis: " + result));

Zusammenfassung

  • Mit CompletableFuture lassen sich asynchrone Aufgaben einfach und übersichtlich implementieren.
  • Verkettungen (thenCompose) und Kombinationen (thenCombine) erlauben komplexe Workflows und parallele Verarbeitung.
  • Fehlerbehandlung erfolgt gezielt mit Methoden wie exceptionally und handle.
  • Bestehende APIs (auch blocking) lassen sich einfach in den asynchronen Workflow einbinden.
