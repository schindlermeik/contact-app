Handout: Verkettung und Kombination von asynchronen Aufgaben mit CompletableFuture

4.1 Methoden zur Verkettung: thenApply, thenAccept, thenRun

Was ist Verkettung („Chaining“)?

  • Mit CompletableFuture kannst du die Verarbeitung von Ergebnissen aneinanderreihen („chainen“), sodass nach Abschluss einer Aufgabe weitere Schritte automatisch ausgeführt werden.
  • Dadurch entstehen übersichtliche, nicht-blockierende Workflows – ähnlich wie bei async/await in anderen Sprachen.

Wichtige Methoden für das Chaining

thenApply(Function)

  • Nutzt das Ergebnis eines CompletableFuture, transformiert es und liefert einen neuen Wert zurück.

  • Beispiel:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10);
CompletableFuture<String> result = future.thenApply(val -> "Ergebnis: " + (val * 2));
result.thenAccept(System.out::println); // Gibt "Ergebnis: 20" aus

thenAccept(Consumer)

  • Nutzt das Ergebnis, aber liefert selbst keinen weiteren Wert zurück.

  • Praktisch für „Endpunkte“ wie Ausgaben, Logging, Speichern.

  • Beispiel:

    CompletableFuture.supplyAsync(() -> "Test")
    .thenAccept(s -> System.out.println("Fertig: " + s));

thenRun(Runnable)

  • Wird ausgeführt, sobald der vorherige Task fertig ist, interessiert sich aber nicht für das Ergebnis.

  • Nutze dies für einfache Folgeaktionen ohne Parameter.

  • Beispiel:

    CompletableFuture.supplyAsync(() -> 5)
    .thenRun(() -> System.out.println("Alles erledigt!"));

4.2 Asynchrone Verarbeitungsketten (thenCompose, thenCombine, allOf, anyOf)

thenCompose(Function)

  • Verkettet zwei asynchrone Operationen, die voneinander abhängen.

  • Die zweite Operation hängt vom Ergebnis der ersten ab, beide sind asynchron.

  • Beispiel:

    CompletableFuture<String> first = CompletableFuture.supplyAsync(() -> "UserId");
CompletableFuture<String> workflow = first.thenCompose(id ->
    CompletableFuture.supplyAsync(() -> "UserDetails for " + id)
);
workflow.thenAccept(System.out::println);

  • Vergleichbar mit „await“ im Await-Pattern anderer Sprachen.

thenCombine(CompletionStage, BiFunction)

  • Kombiniert die Ergebnisse zweier unabhängiger Futures, sobald beide abgeschlossen sind.

  • Beispiel:

    CompletableFuture<Integer> future1 = CompletableFuture.supplyAsync(() -> 2);
CompletableFuture<Integer> future2 = CompletableFuture.supplyAsync(() -> 3);

CompletableFuture<Integer> combined = future1.thenCombine(future2, (a, b) -> a * b);
combined.thenAccept(result -> System.out.println("Produkt: " + result)); // 6

allOf und anyOf

  • allOf: Wartet, bis alle angegebenen Futures fertig sind.

    CompletableFuture<Void> all = CompletableFuture.allOf(future1, future2, ...);
all.thenRun(() -> System.out.println("Alle Aufgaben fertig!"));
    • Die Ergebnisse der einzelnen Futures können dann mit get()/join() abgerufen werden.

  • anyOf: Wartet, bis mindestens einer fertig ist.

    CompletableFuture<Object> any = CompletableFuture.anyOf(future1, future2, ...);
any.thenAccept(result -> System.out.println("Erster fertig: " + result));

4.3 Fehlerbehandlung (exceptionally, handle, whenComplete)

exceptionally(Function<Throwable, ? extends T>)

  • Wird aufgerufen, falls im asynchronen Ablauf ein Fehler (Exception) auftritt. Liefert einen Ersatzwert.

  • Beispiel:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> {
    if (true) throw new RuntimeException("Fehler!");
    return 10;
});
future.exceptionally(ex -> {
    System.out.println("Fehler aufgetreten: " + ex.getMessage());
    return 0; // Ersatzwert
}).thenAccept(System.out::println);

handle(BiFunction<T, Throwable, R>)

  • Wird immer aufgerufen: egal, ob das Future erfolgreich oder mit Fehler abgeschlossen wurde.

  • Du bekommst sowohl das Ergebnis (ggf. null) als auch die Exception (ggf. null).

  • Beispiel:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10 / 0);
future.handle((result, ex) -> {
    if (ex != null) {
        System.out.println("Exception: " + ex.getMessage());
        return -1;
    }
    return result;
}).thenAccept(System.out::println);

whenComplete(BiConsumer<T, Throwable>)

  • Wird am Ende aufgerufen, kann aber das Ergebnis nicht ersetzen oder verändern (nur für Logging etc.).

  • Beispiel:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10 / 2);
future.whenComplete((result, ex) -> {
    if (ex != null) {
        System.out.println("Fehler: " + ex.getMessage());
    } else {
        System.out.println("Erfolg: " + result);
    }
});

4.4 Beispiele für komplexe Workflows: Parallele und abhängige Aufgaben

a) Parallele Aufgaben kombinieren

  • Mehrere Datenquellen gleichzeitig abfragen und die Ergebnisse kombinieren:

    CompletableFuture<String> db = CompletableFuture.supplyAsync(() -> "DB-Ergebnis");
CompletableFuture<String> rest = CompletableFuture.supplyAsync(() -> "REST-Ergebnis");

CompletableFuture<Void> combined = CompletableFuture.allOf(db, rest);
combined.thenRun(() -> {
    try {
        System.out.println("DB: " + db.get());
        System.out.println("REST: " + rest.get());
    } catch (Exception e) { e.printStackTrace(); }
});

b) Abhängige Aufgaben (Workflow)

  • Das Ergebnis eines Tasks ist Input für den nächsten:

    CompletableFuture<String> step1 = CompletableFuture.supplyAsync(() -> "userId-123");
CompletableFuture<String> workflow = step1
    .thenCompose(userId -> CompletableFuture.supplyAsync(() -> "Details von " + userId))
    .thenApply(details -> details.toUpperCase());

workflow.thenAccept(System.out::println); // Gibt die Details in Großbuchstaben aus

c) Fehlerfälle abfangen und alternative Abläufe definieren

  • Fehler in einem Schritt abfangen und „Fallback“ nutzen:

    CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 10 / 0)
    .exceptionally(ex -> {
        System.out.println("Division durch null! Nutze Ersatzwert.");
        return 1;
    });
future.thenAccept(System.out::println); // Gibt 1 aus

Zusammenfassung

  • Mit CompletableFuture lassen sich komplexe, nicht-blockierende Workflows durch Verkettung und Kombination realisieren.
  • Wichtige Methoden sind thenApply, thenAccept, thenRun (einfache Ketten), thenCompose und thenCombine (für abhängige und parallele Tasks), sowie allOf/anyOf (für viele Futures).
  • Fehler lassen sich elegant asynchron abfangen, ohne das Programm zu blockieren.
  • Parallele und abhängige Aufgaben lassen sich kompakt, übersichtlich und sicher modellieren – ein großer Fortschritt gegenüber klassischem Thread- oder Future-Code.
