Handout: Einführung in CompletableFuture (Java 8+)

3.1 Was ist CompletableFuture? – Konzept und Motivation (Java 8+)

Konzept

  • CompletableFuture<V> ist eine Klasse aus dem Paket java.util.concurrent, eingeführt mit Java 8.
  • Sie steht für ein „zukünftiges“ Ergebnis einer asynchronen Berechnung – ähnlich wie ein verbessertes Future.
  • Sie unterstützt funktionale, nicht-blockierende Programmierung mit modernen Methoden (Chaining, Fehlerbehandlung, Kombinieren von Tasks).

Warum CompletableFuture?

Begrenzungen der klassischen Future-API:

  • Blockierendes Warten (get()).
  • Kein „Chaining“ von Aufgaben möglich.
  • Schlechte Fehlerbehandlung.
  • Keine einfache Kombination mehrerer Futures.

Vorteile von CompletableFuture:

  • Nicht-blockierende Verarbeitung: Reaktionen auf Ergebnisse, sobald sie vorliegen.
  • Verkettung: Weiterverarbeitung von Ergebnissen (z. B. „wenn fertig, dann...“).
  • Parallele und abhängige Aufgaben: Mehrere Aufgaben lassen sich kombinieren und koordinieren.
  • Fehlerbehandlung: Moderne, flexible Exception-Strategien.
  • Integration in funktionale Programmierung: Lambda-Ausdrücke und Streams.

Vergleich

Future CompletableFuture
Asynchron Ja Ja
Nicht-blockierend Nein Ja
Chaining Nein Ja
Fehlerbehandlung Umständlich Modern, flexibel
Kombinieren Nein Ja (z. B. allOf, anyOf)

3.2 Erstellen und Starten eines CompletableFuture

3.2.1 Mit supplyAsync()

  • Nutzt einen Thread Pool (Standard: ForkJoinPool.commonPool()), um eine asynchrone Berechnung mit Rückgabewert zu starten.

Beispiel:

import java.util.concurrent.*;

CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> {
    // aufwändige Berechnung
    try { Thread.sleep(1000); } catch (InterruptedException ignored) {}
    return 42;
});

future.thenAccept(result -> System.out.println("Ergebnis: " + result));
  • Der Hauptthread läuft weiter, während im Hintergrund die Aufgabe ausgeführt wird.
  • Mit thenAccept() wird eine Aktion ausgeführt, sobald das Ergebnis verfügbar ist (ohne blockierendes get()).

3.2.2 Mit runAsync()

  • Für Aufgaben, die keinen Rückgabewert liefern (entspricht Runnable).

Beispiel:

CompletableFuture<Void> future = CompletableFuture.runAsync(() -> {
    try { Thread.sleep(500); } catch (InterruptedException ignored) {}
    System.out.println("Aufgabe abgeschlossen!");
});

future.thenRun(() -> System.out.println("Weiter mit nächster Aufgabe."));
  • Nach Abschluss des Tasks wird die nächste Aktion ausgeführt.

3.2.3 Eigene CompletableFutures manuell vervollständigen

  • Du kannst ein CompletableFuture-Objekt auch von Hand erzeugen und das Ergebnis später „setzen“ (nützlich für Brücken zu Callback-basierten APIs).

Beispiel:

CompletableFuture<String> future = new CompletableFuture<>();

// Im Hintergrund, z.B. Callback von API-Aufruf:
new Thread(() -> {
    try {
        Thread.sleep(700);
        future.complete("Antwort vom Service");
    } catch (InterruptedException e) {
        future.completeExceptionally(e);
    }
}).start();

future.thenAccept(result -> System.out.println("Ergebnis: " + result));
  • Mit complete() wird das Ergebnis gesetzt, mit completeExceptionally() ein Fehler signalisiert.
  • Der aufrufende Code kann sofort mit .then...-Methoden auf das Ergebnis reagieren.

Zusammenfassung:

  • CompletableFuture ist das zentrale Werkzeug für moderne asynchrone Programmierung in Java.
  • Aufgaben können mit supplyAsync() (mit Rückgabewert) oder runAsync() (ohne Rückgabewert) einfach gestartet werden.
  • Eigene Futures lassen sich manuell vervollständigen – ideal für Integration in komplexe Architekturen oder bei asynchronen Ereignissen.
