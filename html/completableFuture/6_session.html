Handout: Best Practices und typische Fallstricke bei asynchronem Code (CompletableFuture)

6.1 Richtige Fehlerbehandlung und Exception-Handling bei asynchronem Code

Herausforderung:

  • Fehler (Exceptions) passieren oft „im Hintergrund“ und können im Hauptthread leicht übersehen werden.
  • Klassische try/catch-Blöcke funktionieren bei asynchronem Code nicht wie gewohnt.

Best Practices:

  • Sichere Fehlerbehandlung mit CompletableFuture-Methoden:

    • exceptionally(): Liefert einen Ersatzwert oder führt eine Aktion bei Fehlern aus.

      CompletableFuture.supplyAsync(() -> {
    if (true) throw new RuntimeException("Fehler!");
    return 10;
}).exceptionally(ex -> {
    System.out.println("Fehler: " + ex.getMessage());
    return 0;
});

    • handle(): Wird bei Erfolg und Fehler aufgerufen – flexibles Fehler- und Ergebnis-Handling.

      .handle((result, ex) -> ex != null ? -1 : result)

    • whenComplete(): Für Logging, unabhängig vom Erfolg oder Fehler (ändert das Ergebnis nicht).

      .whenComplete((result, ex) -> {
    if (ex != null) System.out.println("Fehler: " + ex.getMessage());
});

  • Fehler weitergeben statt verschlucken: Lasse Fehler nicht einfach „verschwinden“, sondern behandle sie sichtbar (Logging, Monitoring, Weiterreichen).

  • Kombinierte Fehlerbehandlung in Verkettungen: Auch in verketteten Chains sollte jeder Schritt Fehler korrekt weitergeben oder abfangen.

6.2 Ressourcen-Management und Umgang mit Thread Pools

Herausforderung:

  • Jede asynchrone Aufgabe braucht Threads. Ohne Kontrolle können zu viele Threads entstehen, die die Anwendung verlangsamen oder abstürzen lassen.

Best Practices:

  • Eigene, begrenzte Thread Pools verwenden:

    ExecutorService pool = Executors.newFixedThreadPool(4);
CompletableFuture.supplyAsync(() -> rechenlastig(), pool);

  • Thread Pools korrekt herunterfahren: Vergiss niemals, nach der Verarbeitung shutdown() aufzurufen:

    pool.shutdown();

  • Keine „Fire-and-Forget“-Tasks ohne Kontrolle: Sammle alle gestarteten Futures ein und überprüfe deren Status – so vermeidest du „Thread-Leaks“.

  • Pool-Größe anpassen: Je nach Anwendung (CPU- oder IO-lastig) die Pool-Größe sinnvoll wählen.

6.3 Vermeidung von Callback-Höllen (Callback Hell) durch methodisches Chaining

Problem:

  • Verschachtelte Callbacks („Callback-Hölle“) führen zu unleserlichem, schwer wartbarem Code.

Beispiel (schlecht):

api1(result1 -> {
    api2(result1, result2 -> {
        api3(result2, result3 -> {
            // Weiterverarbeitung...
        });
    });
});

Best Practices:

  • Verkettung (Chaining) von CompletableFutures nutzen: Nutze Methoden wie thenCompose, thenApply, um asynchrone Abläufe linear darzustellen:

    CompletableFuture.supplyAsync(() -> api1())
    .thenCompose(result1 -> CompletableFuture.supplyAsync(() -> api2(result1)))
    .thenCompose(result2 -> CompletableFuture.supplyAsync(() -> api3(result2)))
    .thenAccept(result3 -> System.out.println("Finales Ergebnis: " + result3));

  • Jede Stufe hat ihren eigenen klaren Aufgabenbereich.

  • Kombiniere parallele Aufgaben mit thenCombine, allOf usw., statt sie manuell zu synchronisieren.

6.4 Debugging und Testen asynchroner Logik

Herausforderungen:

  • Fehler treten oft nicht sofort auf, sondern erst „irgendwann“ im Hintergrund.
  • Stacktraces sind häufig schwer nachvollziehbar.

Best Practices:

  • Systematisches Logging: Logge Start, Ende und Fehler jeder asynchronen Aufgabe. Ergänze Logs mit Thread-Namen und Context-Infos.

  • Timeouts verwenden: Verhindere „hängende“ Futures durch sinnvolle Zeitbegrenzungen:

    future.get(2, TimeUnit.SECONDS);

  • Unit-Tests für asynchrone Logik:

    • Verwende Test-Utilities wie Awaitility, um auf Ergebnisse zu warten.
    • Prüfe sowohl Erfolg als auch Fehlerfälle.

Beispiel (JUnit5):

CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> 42);
Assertions.assertEquals(42, future.join());
  • Verwende .join() im Testcode: Im Unterschied zu get() wirft .join() eine CompletionException zur Laufzeit und muss nicht mit checked exceptions umgehen.

Zusammenfassung

  • Moderne Fehlerbehandlung bei asynchronem Code nutzt exceptionally, handle und whenComplete.
  • Kontrollierter Umgang mit Thread Pools ist essenziell für stabile und skalierbare Anwendungen.
  • Methodisches Chaining macht asynchronen Code lesbar und wartbar – vermeide Verschachtelung!
  • Teste asynchrone Logik gezielt, setze Timeouts und nutze Logging und geeignete Test-Tools.
