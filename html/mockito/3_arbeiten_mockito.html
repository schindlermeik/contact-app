Handout: Arbeiten mit Mockito (als Standard-Framework)

1. Grundlegende Konzepte von Mockito

  • Mockito ist ein populäres Java-Framework, mit dem sich sogenannte Mocks für Unit-Tests einfach und flexibel erstellen und verwenden lassen.
  • Ziel: Abhängigkeiten ersetzen, Verhalten kontrollieren und Interaktionen testen, ohne echten Code auszuführen.
  • Mockito lässt sich am besten mit JUnit 5 verwenden.

2. Erstellung von Mocks

a) Manuelle Erstellung mit Mockito.mock()

  • Direkt im Test eine Mock-Instanz erzeugen:

    MyService mockService = Mockito.mock(MyService.class);

b) Nutzung der Annotation @Mock (mit JUnit 5)

  • Eleganter und übersichtlicher, besonders bei mehreren Mocks.

  • Benötigt das Hinzufügen der Mockito-Erweiterung zur Testklasse:

    @ExtendWith(MockitoExtension.class)
class MyTest {
    @Mock
    MyService mockService;

    @Test
    void testSomething() {
        // mockService ist schon ein Mock!
    }
}

3. Verhalten von Mocks definieren

a) Rückgabewerte festlegen: when(...).thenReturn(...)

  • So gibst du vor, was der Mock auf bestimmte Aufrufe zurückgeben soll:

    when(mockService.getData()).thenReturn("Hello World");

b) Mehrere Rückgabewerte:

  • Unterschiedliche Werte bei mehreren Aufrufen:

    when(mockService.getData())
    .thenReturn("A")
    .thenReturn("B");
// 1. Aufruf: "A", 2. Aufruf: "B"

c) Exceptions simulieren: doThrow() oder thenThrow()

  • Wenn eine Methode eine Exception werfen soll:

    when(mockService.getData()).thenThrow(new RuntimeException("Fehler"));
// oder
doThrow(new IllegalStateException()).when(mockService).save(any());

4. Verifikation von Methodenaufrufen

  • Prüfe, ob und wie oft eine Methode tatsächlich aufgerufen wurde:

    mockService.getData();
verify(mockService).getData();             // wurde aufgerufen?
verify(mockService, times(1)).getData();   // exakt 1x aufgerufen?
verify(mockService, never()).delete();     // wurde NIE aufgerufen?
verify(mockService, atLeastOnce()).getData(); // mind. 1x?

  • Überprüfen von Aufrufen mit bestimmten Parametern:

    mockService.save("Hallo");
verify(mockService).save("Hallo");

5. Nutzung von Argument Matchern (any(), eq(), etc.)

  • Argument Matcher erlauben flexiblere Verifikation und Verhalten:

    when(mockService.save(any())).thenReturn(true);
when(mockService.find(eq("X"))).thenReturn(new Data("X"));
verify(mockService).save(anyString());
verify(mockService).find(eq("Hallo"));

  • Wichtige Matcher:

    • any(), anyString(), anyInt(), etc. (beliebiger Wert)
    • eq(value) (genau dieser Wert)
    • isNull(), notNull()
    • Eigene Matcher sind möglich mit argThat(...)

Beispiel für eigene Bedingungen:

verify(mockService).save(argThat(str -> str.startsWith("Test")));

6. Praxisbeispiel: Einfacher Mockito-Test

import static org.mockito.Mockito.*;
import static org.junit.jupiter.api.Assertions.*;
import org.junit.jupiter.api.*;
import org.mockito.*;

@ExtendWith(MockitoExtension.class)
class UserServiceTest {
    @Mock
    EmailService emailService;

    @InjectMocks
    UserService userService;

    @Test
    void shouldSendWelcomeMail() {
        // Verhalten definieren
        when(emailService.sendWelcomeMail(anyString())).thenReturn(true);

        // Methode testen
        boolean result = userService.registerUser("test@mail.com");

        // Ergebnis prüfen
        assertTrue(result);

        // Verifizieren, dass E-Mail-Service wirklich aufgerufen wurde
        verify(emailService).sendWelcomeMail(eq("test@mail.com"));
    }
}

7. Zusammenfassung & Tipps

  • Mocks ersetzen Abhängigkeiten im Test und machen das Verhalten kontrollierbar.
  • Erzeuge Mocks per Mockito.mock() oder Annotationen (@Mock).
  • Definiere Rückgabewerte, Exceptions und prüfe Interaktionen.
  • Nutze Argument Matcher, um flexibel zu bleiben.
  • Verifiziere, ob Methoden wirklich und korrekt aufgerufen wurden.
  • Halte deine Tests einfach, klar und unabhängig von echter Infrastruktur.
