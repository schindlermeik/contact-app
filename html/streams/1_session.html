Handout: Einführung in Streams (Java)

1. Was sind Streams?

Streams sind ein zentrales Konzept in Java seit Version 8. Sie ermöglichen eine deklarative, funktionale und oft sehr kompakte Verarbeitung von Daten – insbesondere von Sammlungen wie Listen oder Arrays.

  • Ein Stream ist eine Folge von Elementen, die verschiedene Operationen (wie Filtern, Umwandeln, Zusammenfassen) ermöglichen – ähnlich wie bei Fließbändern in der Industrie.
  • Streams verändern die zugrunde liegenden Datenstrukturen nicht, sondern liefern neue Ergebnisse.
  • Streams unterstützen Pipelines: Mehrere Operationen können hintereinander verkettet werden.

Beispiel (Kurzform):

List<String> namen = Arrays.asList("Anna", "Bernd", "Clara");

// Alle Namen, die mit "A" beginnen, in Großbuchstaben umwandeln:
List<String> gefiltert = namen.stream()
    .filter(n -> n.startsWith("A"))
    .map(String::toUpperCase)
    .collect(Collectors.toList());

System.out.println(gefiltert); // [ANNA]

2. Unterschiede zwischen Collections und Streams

Collections Streams
Speichern tatsächlich die Daten (persistente Daten) Verarbeiten Daten im „Fluss“ (flüchtig)
Können direkt geändert werden (hinzufügen, entfernen) Werden meist einmal durchlaufen, nicht veränderbar
Zugriff auf einzelne Elemente möglich (z. B. per Index) Kein direkter Zugriff auf einzelne Elemente
Iteration mit Schleifen oder Iterator Deklarative Verkettung von Operationen
Eher imperativer Stil Funktional/deklarativ, Pipeline-Stil

Beispiel Collection:

List<String> namen = new ArrayList<>();
namen.add("Anna");
namen.add("Bernd");
namen.add("Clara");
for (String name : namen) {
    System.out.println(name);
}

Beispiel Stream:

namen.stream()
    .filter(n -> n.length() > 4)
    .forEach(System.out::println);

3. Motivation: Warum Streams nutzen?

  • Kürzerer, übersichtlicher Code: Viele Arbeitsschritte können in einer einzigen „Pipeline“ ausgedrückt werden.
  • Funktionaler Stil: Das „Was“ rückt in den Vordergrund, nicht das „Wie“ (weniger Schleifen, mehr Logik).
  • Keine Veränderung der Originaldaten: Streams liefern neue Werte und sind „seiteneffektarm“.
  • Leicht parallelisierbar: Streams können auf Wunsch auch mit mehreren Kernen gleichzeitig arbeiten (.parallelStream()).
  • Einfaches Kombinieren von Operationen: Filtern, Umwandeln, Gruppieren und vieles mehr – alles in einer Kette.
  • Bessere Lesbarkeit: Wer Streams beherrscht, versteht auf einen Blick die Datenverarbeitung.

Vergleich:

  • Vor Streams: Um eine Liste zu filtern und zu transformieren, sind oft mehrere verschachtelte Schleifen nötig.
  • Mit Streams: Die gleiche Logik ist in einer klaren, lesbaren Befehlsfolge möglich.

5. Merksätze

  • Streams sind Werkzeuge zur Datenverarbeitung, keine Datenspeicher!
  • Streams ermöglichen Pipeline-Verarbeitung und funktionalen Stil.
  • Sie machen Code oft kürzer, klarer und einfacher zu warten.
