Handout: Arbeiten mit komplexen Datenstrukturen in Streams

1. Streams von Objekten (z. B. Listen von Klassen)

Die Stärke der Streams-API zeigt sich besonders bei komplexeren Datenstrukturen, etwa Listen von Objekten.

Beispielklasse:

public class Person {
    private String name;
    private int alter;

    public Person(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }
    public String getName() { return name; }
    public int getAlter() { return alter; }
}

Beispiel:

List<Person> personen = Arrays.asList(
    new Person("Anna", 20),
    new Person("Bernd", 35),
    new Person("Clara", 35),
    new Person("David", 17)
);

// Alle Namen der Personen ab 18 in Großbuchstaben:
List<String> namenErwachsener = personen.stream()
    .filter(p -> p.getAlter() >= 18)
    .map(p -> p.getName().toUpperCase())
    .collect(Collectors.toList());
System.out.println(namenErwachsener); // [ANNA, BERND, CLARA]

2. Gruppieren und Partitionieren

Mit Collectors kannst du Streams in Gruppen (nach Kriterium) oder in zwei Gruppen (Partition) aufteilen.

a) Gruppieren mit Collectors.groupingBy()

Gruppiere z. B. Personen nach Alter:

Map<Integer, List<Person>> gruppiertNachAlter = personen.stream()
    .collect(Collectors.groupingBy(Person::getAlter));
System.out.println(gruppiertNachAlter);
// Beispielausgabe: {35=[Person@... , Person@...], 20=[Person@...], 17=[Person@...]}
  • Jeder Schlüssel ist ein Alter, der Wert die Liste der Personen mit diesem Alter.

Gruppieren nach abgeleitetem Kriterium:

Map<String, List<Person>> nachErwachsen = personen.stream()
    .collect(Collectors.groupingBy(p -> p.getAlter() >= 18 ? "Erwachsen" : "Kind"));

b) Partitionieren mit Collectors.partitioningBy()

Teilt die Daten nach einem boolean-Kriterium in zwei Gruppen (true/false).

Map<Boolean, List<Person>> partition = personen.stream()
    .collect(Collectors.partitioningBy(p -> p.getAlter() >= 18));
// partition.get(true): alle Erwachsenen
// partition.get(false): alle Minderjährigen

3. Mapping und Reduktion auf komplexe Typen

Du kannst mit Streams nicht nur filtern und gruppieren, sondern auch bestimmte Werte aus verschachtelten Strukturen extrahieren oder aggregieren.

a) Mapping auf bestimmte Eigenschaften

List<Integer> alleAlter = personen.stream()
    .map(Person::getAlter)
    .collect(Collectors.toList());
System.out.println(alleAlter); // [20, 35, 35, 17]

b) Reduktion auf komplexe Typen

Zum Beispiel: Finde die älteste Person.

Optional<Person> aelteste = personen.stream()
    .max(Comparator.comparing(Person::getAlter));
aelteste.ifPresent(p -> System.out.println(p.getName()));

Summe aller Alter:

int summeAlter = personen.stream()
    .mapToInt(Person::getAlter)
    .sum();

4. Übungen zum Selbststudium

  1. Erstelle eine Liste von Person-Objekten und gib alle Namen der Personen, deren Name mit „A“ beginnt, in Kleinbuchstaben aus.
  2. Gruppiere die Personen nach „erwachsen“ und „kind“ mit partitioningBy().
  3. Ermittle das Durchschnittsalter aller Personen in der Liste.
  4. Finde mit reduce() die längste Namenskette in einer Liste von Personen.

5. Merksätze

  • Streams können mit Objekten aller Art arbeiten – besonders mächtig bei komplexen Datenstrukturen.
  • Mit groupingBy() gruppierst du nach beliebigen Kriterien, mit partitioningBy() nach einer boolean-Bedingung.
  • Mapping und Reduktion machen es leicht, einzelne Eigenschaften zu extrahieren oder Zusammenfassungen zu berechnen.
