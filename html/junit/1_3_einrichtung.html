Handout: Einrichtung und Integration von JUnit ins Projekt

1. Maven-Integration von JUnit

a) Was ist Maven?

  • Maven ist ein weit verbreitetes Build- und Projektmanagement-Tool für Java-Projekte.
  • Es ermöglicht das einfache Hinzufügen von Bibliotheken (wie JUnit) durch sogenannte Dependencies in der pom.xml.

b) JUnit als Dependency einbinden

  1. Öffne die Datei pom.xml im Hauptverzeichnis deines Projekts.
  2. Füge folgenden Abschnitt im <dependencies>-Block hinzu (Beispiel für JUnit 5):
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.10.2</version>
    <scope>test</scope>
</dependency>
  • Das Attribut <scope>test</scope> sorgt dafür, dass JUnit nur für Tests geladen wird und nicht im fertigen Programm enthalten ist.
  • Nach dem Hinzufügen: Speichern und ggf. das Projekt „reimportieren“ (oftmals automatisch in modernen IDEs).

Tipp: Immer auf die offizielle JUnit-Seite schauen, um die aktuellste Version zu nutzen!

2. Projektstruktur mit Test-Ordner

a) Standardmäßige Maven-Projektstruktur

Ein typisches Maven-Projekt ist folgendermaßen aufgebaut:

projekt-root/
 ├─ src/
 │   ├─ main/
 │   │   └─ java/              # Produktivcode
 │   │       └─ ...klassen...
 │   └─ test/
 │       └─ java/              # Testcode (JUnit-Tests!)
 │           └─ ...testklassen...
 ├─ pom.xml
  • Produktivcode kommt in src/main/java.
  • Testcode (z.B. CalculatorTest.java) kommt in src/test/java.
  • Beide Ordner können dieselbe Package-Struktur haben.

Beispiel:

src/
 ├─ main/java/com/beispiel/Calculator.java
 └─ test/java/com/beispiel/CalculatorTest.java

Vorteil dieser Trennung: Produktiv- und Testcode sind sauber voneinander getrennt und werden beim Build-Prozess unterschiedlich behandelt.

3. IDE-Unterstützung

a) Allgemeines

  • Moderne Java-IDEs wie IntelliJ IDEA und Eclipse bieten umfassende Unterstützung für JUnit.

  • Vorteile:

    • Tests lassen sich bequem direkt aus der IDE heraus starten.
    • Testresultate werden übersichtlich angezeigt (grün/rot, Stacktraces).
    • Fehlgeschlagene Tests lassen sich einfach debuggen.

b) JUnit in IntelliJ IDEA

  • Die IDE erkennt Testklassen und Testmethoden automatisch (bei korrektem Aufbau & Annotationen).

  • Du kannst:

    • Einzelne Tests oder alle Tests einer Klasse mit Rechtsklick → "Run" ausführen.
    • Mit dem Shortcut Ctrl+Shift+F10 einen Test starten.
    • Ergebnisse im Test-Runner-Fenster grafisch sehen.
    • Beim Doppelklick auf einen fehlgeschlagenen Test springt die IDE direkt zur Fehlerstelle.

c) JUnit in Eclipse

  • Auch Eclipse erkennt JUnit-Tests automatisch.
  • Ausführung über Kontextmenü oder die Toolbar (Rechtsklick → Run As → JUnit Test).
  • Ergebnisanzeige im JUnit-View mit Baumstruktur und Fehlerdetails.

Hinweis: In beiden IDEs sollte das Maven-Projekt einmal aktualisiert werden, nachdem eine neue Dependency hinzugefügt wurde (Rechtsklick auf Projekt → Maven → Reload project bzw. „Update Project“ in Eclipse).

4. Kurzanleitung: Erster Testlauf

  1. Maven-Dependency für JUnit eintragen
  2. Testklasse in src/test/java erstellen
  3. Testmethode mit @Test schreiben
  4. In der IDE rechtsklicken → Test ausführen
  5. Testergebnis prüfen (grün = OK, rot = Fehler)

5. FAQ & Tipps

  • Fehlermeldung „No tests found“?

    • Prüfe, ob deine Testmethoden mit @Test annotiert sind und ob sie public und void sind.

  • IDE erkennt Tests nicht?

    • Maven-Projekt reloaden, evtl. IDE neu starten.

  • JUnit-Versionen mischen?

    • Achte darauf, in der pom.xml nur JUnit 5-Dependencies zu haben, wenn du die neuen Features nutzen willst.
