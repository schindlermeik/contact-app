Refactoring-Aufgabe: Kontaktverwaltungssystem

Ihre Aufgabe ist es, die bestehende Klasse ContactManager in ein wartbareres und skalierbareres Design umzugestalten. Sie werden dabei so viele Refactoring-Funktionen von IntelliJ IDEA wie möglich verwenden, um die folgenden Ziele zu erreichen:

  1. ContactManager in ContactService umbenennen: Die Klasse ContactManager ist für die Geschäftslogik der Kontaktverwaltung zuständig. Gemäß den Prinzipien der Clean Architecture sollte diese Klasse in ContactService umbenannt werden, um ihre Rolle besser widerzuspiegeln.
  2. Ein Interface (ContactRepository) extrahieren: Der ContactService handhabt derzeit sowohl die Geschäftslogik als auch die Datenpersistenz (Kontakte hinzufügen, finden). Wir müssen diese Zuständigkeiten trennen. Erstellen Sie ein neues Interface namens ContactRepository, das die grundlegenden CRUD-Operationen (Create, Read, Update, Delete) für Contact-Objekte definiert.
  3. Eine Repository-Implementierung erstellen (ContactRepositoryImpl): Implementieren Sie das ContactRepository-Interface in einer neuen Klasse namens ContactRepositoryImpl. Diese Klasse wird die eigentliche Logik zum Speichern und Abrufen von Kontakten enthalten, wie die Verwendung der List<Contact> und die Verwaltung der nextId.
  4. ContactService zur Verwendung des neuen Repositories anpassen: Die Klasse ContactService sollte die List<Contact> nicht mehr direkt verwalten. Stattdessen sollte ihr eine Instanz von ContactRepository injiziert werden (über den Konstruktor), die sie zur Durchführung datenbezogener Operationen verwendet. Die Methoden createContact und findById rufen dann die entsprechenden Methoden des Repositories auf.
  5. Die Main-Methoden-Logik in einen ContactClient auslagern: Die main-Methode in der Klasse Main enthält derzeit die Startlogik der Anwendung und eine einfache Demonstration der Funktionalität des ContactManager. Dieser Code sollte in eine neue Klasse namens ContactClient verschoben werden. Die main-Methode sollte dann einfach eine Instanz des ContactClient erstellen und ausführen. Diese Trennung macht die Hauptklasse zu einem einfachen Einstiegspunkt und den ContactClient zu einer dedizierten Test- oder Demonstrationsklasse.

Anweisungen und Schritte:

Verwenden Sie die Refactoring-Tools von IntelliJ für diese Schritte:

  1. ContactManager umbenennen:

    • Klicken Sie mit der rechten Maustaste auf den Klassennamen ContactManager.
    • Wählen Sie Refactor > Rename... und geben Sie ContactService ein. Dadurch werden alle Verweise in der Klasse Main automatisch aktualisiert.

  2. ContactRepository Interface extrahieren:

    • Klicken Sie mit der rechten Maustaste auf die Klasse ContactService.
    • Wählen Sie Refactor > Extract > Interface....
    • Nennen Sie das Interface ContactRepository.
    • Wählen Sie die Methoden aus, die in das Interface aufgenommen werden sollen: createContact(Contact contact) und findById(Long id). Hinweis: Die Methode createAndAdd ist privat und sollte daher nicht Teil des öffentlichen Interfaces sein.

  3. ContactRepositoryImpl implementieren:

    • Erstellen Sie eine neue Klasse namens ContactRepositoryImpl, die das ContactRepository-Interface implementiert.
    • Verschieben Sie die List<Contact> contacts, Long nextId und die Methode createAndAdd von ContactService nach ContactRepositoryImpl.
    • Die Methode findById aus dem Interface sollte hier implementiert werden, indem die lokale contacts-Liste verwendet wird.

  4. ContactService anpassen:

    • Entfernen Sie die contacts-Liste, nextId und die Methode createAndAdd.
    • Fügen Sie ein privates Feld für das ContactRepository hinzu und erstellen Sie einen Konstruktor, der eine ContactRepository-Instanz akzeptiert.
    • Aktualisieren Sie die Methoden createContact und findById so, dass sie ihre Arbeit an das injizierte Repository delegieren. Zum Beispiel sollte createContact(contact) die Methode contactRepository.createContact(contact) aufrufen.

  5. Main refactoren und ContactClient erstellen:

    • Erstellen Sie eine neue Klasse namens ContactClient.
    • Verschieben Sie den gesamten Inhalt der main-Methode von Main in eine neue Methode in ContactClient (z. B. run()).
    • In der main-Methode der Klasse Main instanziieren Sie ContactRepositoryImpl und ContactService.
    • Erstellen Sie eine Instanz von ContactClient, übergeben Sie bei Bedarf den ContactService und rufen Sie dessen run()-Methode auf.

Die Befolgung dieser Schritte führt zu einer besser organisierten Architektur, die die Zuständigkeiten trennt und die Anwendung leichter testbar und wartbar macht.
