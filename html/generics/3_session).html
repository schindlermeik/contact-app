3. Generics in der Java-Standardbibliothek

Handout: Generics in der Java-Standardbibliothek

1. Verwendung von Generics in Collections

Mit der Einführung von Generics in Java 5 wurden die wichtigsten Datenstrukturen der Standardbibliothek – die sogenannten Collections – vollständig überarbeitet. Seitdem ist es möglich, Listen, Mengen und Maps typsicher zu verwenden. Das erhöht die Sicherheit, Lesbarkeit und Wartbarkeit deines Codes deutlich.

a) Listen (List<T>)

Listen sind geordnete Sammlungen von Elementen. Das bekannteste Beispiel ist ArrayList.

Beispiel:

List<String> namen = new ArrayList<>();
namen.add("Anna");
namen.add("Bernd");
// namen.add(42); // Compilerfehler: 42 ist kein String!

for (String name : namen) {
    System.out.println(name.toUpperCase());
}
  • Ohne Generics: Es wäre möglich, beliebige Typen einzufügen, und ein Fehler würde erst zur Laufzeit auffallen.
  • Mit Generics: Der Compiler verhindert, dass falsche Typen eingefügt werden.

b) Sets (Set<T>)

Sets sind ungeordnete Sammlungen von eindeutigen Elementen.

Beispiel:

Set<Integer> zahlen = new HashSet<>();
zahlen.add(1);
zahlen.add(2);
zahlen.add(2); // Wird ignoriert, weil schon vorhanden
// zahlen.add("Text"); // Compilerfehler!

for (Integer zahl : zahlen) {
    System.out.println(zahl * 2);
}
  • Auch hier sorgt der Compiler dafür, dass nur Integer-Werte eingefügt werden.

c) Maps (Map<K, V>)

Maps sind Sammlungen von Schlüssel-Wert-Paaren.

Beispiel:

Map<Integer, String> idToName = new HashMap<>();
idToName.put(1, "Tina");
idToName.put(2, "Markus");
// idToName.put("Drei", "Julia"); // Compilerfehler: Schlüssel muss Integer sein!

for (Map.Entry<Integer, String> entry : idToName.entrySet()) {
    System.out.println("ID: " + entry.getKey() + ", Name: " + entry.getValue());
}
  • Die Typen für Schlüssel und Werte werden festgelegt und vom Compiler kontrolliert.

2. Vorteile der Typsicherheit am Beispiel von Collections

a) Früherkennung von Fehlern

Mit Generics werden Fehler bereits beim Kompilieren erkannt – nicht erst zur Laufzeit. Beispiel:

List<Integer> zahlen = new ArrayList<>();
zahlen.add(5);
zahlen.add("Text"); // Compilerfehler: "Text" ist kein Integer!

Ohne Generics würde der Fehler hier erst auftreten, wenn das Element später falsch verwendet wird.

b) Keine Casts notwendig

Mit Generics entfällt das explizite Casten beim Zugriff auf Elemente.

Vor Java 5:

List namen = new ArrayList();
namen.add("Lisa");
String name = (String) namen.get(0); // Cast notwendig!

Mit Generics:

List<String> namen = new ArrayList<>();
String name = namen.get(0); // Kein Cast notwendig!

c) Verbesserte Lesbarkeit und Wartbarkeit

Wenn du deklarierst

List<Double> preise = new ArrayList<>();

kann jeder im Team sofort erkennen, welche Art von Daten in der Liste steckt.

d) Kombination verschiedener Collections mit Typkontrolle

Auch verschachtelte Collections sind möglich und sicher:

List<List<String>> wortListen = new ArrayList<>();
List<String> tiere = Arrays.asList("Hund", "Katze");
List<String> farben = Arrays.asList("Rot", "Blau");
wortListen.add(tiere);
wortListen.add(farben);
  • Der Compiler prüft, dass in wortListen wirklich nur Listen von Strings gespeichert werden.

e) Vermeidung von Laufzeitfehlern

Beispiel – ohne Generics:

List namen = new ArrayList();
namen.add("Kai");
namen.add(123); // Wird akzeptiert

for (Object obj : namen) {
    String name = (String) obj; // Beim zweiten Element: ClassCastException!
}

Mit Generics:

List<String> namen = new ArrayList<>();
namen.add("Kai");
// namen.add(123); // Compilerfehler!

Fehler werden frühzeitig entdeckt, und dein Programm ist robuster.

3. Übungen zum Selbststudium

  1. Aufgabe: Lege eine List<Double> an, füge ein paar Werte hinzu und berechne den Durchschnitt aller Werte.

  2. Aufgabe: Erstelle eine Map<String, Integer>, in der als Schlüssel die Namen von Produkten und als Wert deren Lagerbestand gespeichert wird. Gib alle Produkte mit Bestand größer als 10 aus.

  3. Aufgabe: Überlege: Was passiert, wenn du versuchst, einen String in eine List<Integer> einzufügen? Probiere es aus!

4. Merksätze

  • Mit Generics werden alle wichtigen Collections typsicher.
  • Falsche Typen werden schon beim Kompilieren verhindert.
  • Generics machen deinen Code klarer, sicherer und leichter zu warten.
