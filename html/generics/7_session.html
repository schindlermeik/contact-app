Handout: Praktische Übungen und Beispiele zu Generics

1. Schreiben einer eigenen generischen Klasse

Aufgabe 1:

Schreibe eine generische Klasse Pair<T, U>, die zwei Werte beliebigen Typs speichert.

Musterlösung:

public class Pair<T, U> {
    private T first;
    private U second;

    public Pair(T first, U second) {
        this.first = first;
        this.second = second;
    }

    public T getFirst() { return first; }
    public U getSecond() { return second; }
    public void setFirst(T first) { this.first = first; }
    public void setSecond(U second) { this.second = second; }

    @Override
    public String toString() {
        return "(" + first + ", " + second + ")";
    }
}

Beispiel-Verwendung:

Pair<String, Integer> person = new Pair<>("Anna", 27);
System.out.println(person); // Ausgabe: (Anna, 27)

Pair<Double, Double> koordinaten = new Pair<>(12.5, 48.2);

2. Verwendung von Generics in Methoden

Aufgabe 2:

Schreibe eine generische Methode, die das größte Element aus einem Array beliebigen Typs zurückgibt (sofern der Typ vergleichbar ist).

Musterlösung:

public class Utils {
    public static <T extends Comparable<T>> T max(T[] array) {
        if (array == null || array.length == 0) return null;
        T max = array[0];
        for (T element : array) {
            if (element.compareTo(max) > 0) {
                max = element;
            }
        }
        return max;
    }
}

Beispiel-Verwendung:

Integer[] zahlen = {3, 9, 2, 15, 6};
System.out.println(Utils.max(zahlen)); // Ausgabe: 15

String[] namen = {"Lisa", "Bob", "Anna"};
System.out.println(Utils.max(namen)); // Ausgabe: Lisa (alphabetisch)

3. Implementierung und Einsatz von Wildcards

Aufgabe 3:

Schreibe eine Methode, die eine beliebige Liste (egal welcher Typ) ausgibt.

Musterlösung:

public static void printList(List<?> list) {
    for (Object obj : list) {
        System.out.println(obj);
    }
}

Beispiel-Verwendung:

List<Integer> zahlen = Arrays.asList(1, 2, 3);
List<String> worte = Arrays.asList("eins", "zwei", "drei");
printList(zahlen);
printList(worte);

Aufgabe 4:

Schreibe eine Methode, die die Summe aller Zahlen in einer List<? extends Number> berechnet.

Musterlösung:

public static double sumNumbers(List<? extends Number> numbers) {
    double sum = 0.0;
    for (Number n : numbers) {
        sum += n.doubleValue();
    }
    return sum;
}

Beispiel-Verwendung:

List<Integer> ints = Arrays.asList(1, 2, 3, 4);
System.out.println(sumNumbers(ints)); // Ausgabe: 10.0

List<Double> doubles = Arrays.asList(1.5, 2.5, 3.0);
System.out.println(sumNumbers(doubles)); // Ausgabe: 7.0

4. Refactoring: Von „raw types“ zu Generics

Aufgabe 5:

Gegeben ist folgender Code ohne Generics:

List namen = new ArrayList();
namen.add("Anna");
namen.add("Bernd");
namen.add(123); // Fehlerquellen!
for (Object obj : namen) {
    String name = (String) obj;
    System.out.println(name.toUpperCase());
}

Aufgabe:

  • Erkenne die Probleme im Code.
  • Refactore den Code so, dass er Generics nutzt und typsicher ist.

Musterlösung und Erklärung:

Probleme:

  • Unterschiedliche Typen können eingefügt werden.
  • Fehler (z. B. ClassCastException) treten erst zur Laufzeit auf.
  • Ständiges Casten nötig.

Mit Generics:

List<String> namen = new ArrayList<>();
namen.add("Anna");
namen.add("Bernd");
// namen.add(123); // Compilerfehler – Typsicherheit!

for (String name : namen) {
    System.out.println(name.toUpperCase());
}

Ergebnis:

  • Nur Strings können eingefügt werden.
  • Keine Casts mehr nötig.
  • Fehler werden früh (beim Kompilieren) erkannt.

5. Reflexionsfragen (Selbststudium)

  1. Welche Vorteile spürst du beim Umstieg von „raw types“ auf Generics in deinem Code?
  2. Was passiert, wenn du bei deinen eigenen generischen Klassen versuchst, unterschiedliche Typen zu mischen?
  3. Überlege: Wann brauchst du Wildcards (<?> / <? extends T> / <? super T>), wann reichen feste Typ-Parameter?

6. Zusammenfassung

  • Eigene generische Klassen und Methoden machen deinen Code wiederverwendbar und typsicher.
  • Wildcards erhöhen die Flexibilität bei Methoden, die auf verschiedene (aber verwandte) Typen angewendet werden sollen.
  • Der Umstieg von „raw types“ zu Generics ist ein großer Schritt zu sichererem und besser lesbarem Java-Code.
