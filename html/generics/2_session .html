Handout: Verwendung von Generics in eigenen Klassen und Interfaces

1. Generische Klassen

a) Was ist eine generische Klasse?

Eine generische Klasse ist eine Klasse, bei der ein oder mehrere Typen erst bei der Verwendung festgelegt werden. Statt einen festen Typ zu verwenden, setzt man an den Stellen, an denen sonst der Typ stehen würde, einen Typ-Parameter (meist einen Großbuchstaben wie T).

b) Syntax:

public class Box<T> {
    private T content;

    public void setContent(T content) {
        this.content = content;
    }

    public T getContent() {
        return content;
    }
}
  • T ist ein Typ-Parameter (stellvertretend für einen konkreten Datentyp).
  • Beim Erstellen des Objekts wird T festgelegt (z. B. String, Integer, eine eigene Klasse …).

c) Beispiel: Eine generische Box für verschiedene Datentypen

Box<String> stringBox = new Box<>();
stringBox.setContent("Hallo Java");
String wert = stringBox.getContent(); // Kein Cast nötig

Box<Integer> intBox = new Box<>();
intBox.setContent(42);
Integer zahl = intBox.getContent(); // Kein Cast nötig

  • Die Klasse Box<T> kann jetzt für jeden beliebigen Typ verwendet werden.

  • Es können keine „falschen“ Typen hineingelegt werden:

    Box<String> nameBox = new Box<>();
nameBox.setContent("Anna");
// nameBox.setContent(123); // Compilerfehler!

2. Generische Interfaces

a) Was ist ein generisches Interface?

Ein generisches Interface ist ein Interface, das Typ-Parameter verwendet – ähnlich wie bei generischen Klassen. So können Implementierungen für verschiedene Typen geschaffen werden.

b) Beispiel: Das Interface Comparable<T>

public interface Comparable<T> {
    public int compareTo(T o);
}
  • Das Interface Comparable<T> ist in der Java-Standardbibliothek weit verbreitet.
  • Klassen, die Comparable<T> implementieren, müssen die Methode compareTo(T o) für den jeweiligen Typ implementieren.

Beispiel: Eigene Klasse, die Comparable verwendet

public class Student implements Comparable<Student> {
    private String name;
    private int punkte;

    public Student(String name, int punkte) {
        this.name = name;
        this.punkte = punkte;
    }

    @Override
    public int compareTo(Student other) {
        return Integer.compare(this.punkte, other.punkte);
    }
}
  • So kann eine Liste von Student-Objekten problemlos sortiert werden.

3. Instanziierung generischer Klassen

a) Wie „erzeugt“ man eine Instanz einer generischen Klasse?

Beim Erzeugen des Objekts gibst du in spitzen Klammern <...> den konkreten Typ an, für den der Typ-Parameter stehen soll.

Box<String> meineBox = new Box<>();
meineBox.setContent("Java rocks!");

Box<Double> preisBox = new Box<>();
preisBox.setContent(9.99);

  • Ab Java 7 kann beim Erzeugen rechts die Typ-Angabe durch <> abgekürzt werden:

    Box<String> box = new Box<>(); // "Diamond Operator"

b) Wichtige Hinweise

  • Die Typ-Angabe in <...> ist keine echte Variable, sondern existiert nur zur Compile-Zeit.
  • Der Typ-Parameter kann alles sein: primitive Datentypen müssen aber als Wrapper (Integer, Double usw.) verwendet werden, nicht als int/double.

c) Falsche Verwendung wird vom Compiler erkannt:

Box<Integer> box = new Box<>();
// box.setContent("Text"); // Compilerfehler! Nur Integer erlaubt

4. Aufgaben zum Üben

  1. Eigene generische Klasse: Schreibe eine Klasse Pair<T, U>, die zwei Werte von (beliebigen) Typen speichert.

  2. Eigene generische Methode: Erstelle eine Methode, die aus einer Box<T> den Wert zurückgibt, egal welcher Typ.

  3. Implementiere ein eigenes generisches Interface: Schreibe ein Interface Saver<T>, das eine Methode void save(T value) vorgibt. Implementiere eine Klasse StringSaver, die dieses Interface für Strings umsetzt.

5. Zusammenfassung

  • Generische Klassen und Interfaces ermöglichen dir, flexiblen und typsicheren Code zu schreiben.
  • Du musst dich nicht mehr auf einen bestimmten Datentyp festlegen, sondern kannst denselben Code für viele verschiedene Typen wiederverwenden.
  • Der Compiler prüft, dass du keine „falschen“ Typen verwendest.
  • Besonders nützlich ist das in der Java-Standardbibliothek (z. B. bei Listen, Maps, Sets …).

6. Merksätze

  • T steht meist für „Type“; auch gängig: E (Element), K (Key), V (Value).
  • Du kannst mehrere Typ-Parameter nutzen: Pair<T, U>.
  • Generics machen deinen Code robuster und leichter lesbar.
