Handout: Was sind Generics?

1. Was sind Generics?

Generics sind eine Sprachfunktion in Java, mit der du Klassen, Interfaces und Methoden so schreiben kannst, dass sie mit unterschiedlichen Datentypen arbeiten können – und zwar typsicher, d. h. mit Überprüfung schon zur Compile-Zeit.

Beispiel (ohne Generics):

// Eine einfache Box für beliebige Objekte
public class Box {
    private Object content;

    public void setContent(Object content) {
        this.content = content;
    }

    public Object getContent() {
        return content;
    }
}

Nachteile:

  • Beim Herausholen des Inhalts muss man casten:

    Box box = new Box();
box.setContent("Hallo Welt");

String value = (String) box.getContent(); // Cast notwendig!

  • Fehler (z. B. ein Integer statt String) werden erst zur Laufzeit erkannt.

Mit Generics:

public class Box<T> {
    private T content;

    public void setContent(T content) {
        this.content = content;
    }

    public T getContent() {
        return content;
    }
}

  • Die Klasse ist typsicher: Du bestimmst beim Erstellen, welchen Typ die Box enthalten soll.

    Box<String> box = new Box<>();
box.setContent("Hallo Welt");

String value = box.getContent(); // Kein Cast nötig!

2. Motivation: Warum wurden Generics in Java eingeführt?

Vor Java 5 gab es keine Generics. Alle „verallgemeinerten“ Klassen, wie zum Beispiel die Collections-API (List, Map, Set), arbeiteten mit dem Typ Object. Das führte zu folgenden Problemen:

Typische Probleme:

  • Fehlende Typsicherheit: Du konntest jeder Collection beliebige Objekte hinzufügen, aber nicht garantieren, dass alle Elemente denselben Typ haben.
  • Zwang zum Casten: Jedes Element musste beim Herausnehmen aus einer Collection gecastet werden.
  • Laufzeitfehler: Casts konnten fehlschlagen und verursachten dann zur Laufzeit einen ClassCastException.
  • Unübersichtlicher Code: Viele Casts machen Code schwer lesbar und fehleranfällig.

Motivation für Generics:

  • Typsicherheit: Fehler werden bereits beim Kompilieren erkannt.
  • Besser lesbarer und wartbarer Code: Klarheit über verwendete Typen.
  • Vermeidung von Laufzeitfehlern: Potenziell gefährliche Casts entfallen.

3. Vorteile gegenüber der Verwendung von Object

Mit Generics:

  • Du kannst Collections und andere generische Klassen auf einen bestimmten Typ einschränken, z. B. List<String>.
  • Der Compiler prüft, ob nur passende Typen eingefügt werden.
  • Beim Zugriff auf Elemente ist kein Cast mehr nötig.

Beispiel – ohne Generics:

List names = new ArrayList();
names.add("Anna");
names.add(123); // Funktioniert! Aber ist das sinnvoll?

String name = (String) names.get(0); // OK
String another = (String) names.get(1); // Fehler erst zur Laufzeit!

Beispiel – mit Generics:

List<String> names = new ArrayList<>();
names.add("Anna");
names.add(123); // Fehler: Compiler meldet den Fehler sofort!

String name = names.get(0); // Kein Cast nötig

4. Typische Problemstellungen vor Generics (Type Casting, Laufzeitfehler)

a) Type Casting

  • Ohne Generics gibt Java nur ein Object zurück, du musst also casten:

    List list = new ArrayList();
list.add("Hallo");
String s = (String) list.get(0); // Cast nötig

  • Fügst du versehentlich einen anderen Typ ein, passiert der Fehler erst beim Cast:

    list.add(123); // Wird akzeptiert
String t = (String) list.get(1); // ClassCastException!

b) Laufzeitfehler

  • Fehler werden erst zur Laufzeit entdeckt, nicht beim Kompilieren.
  • Ein Programm, das sich kompilieren lässt, kann trotzdem abstürzen, weil der Typ nicht passt.

Konkretes Beispiel:

public static void main(String[] args) {
    List words = new ArrayList();
    words.add("Hund");
    words.add("Katze");
    words.add(42); // Logisch falsch, aber erlaubt!

    for (Object obj : words) {
        String word = (String) obj; // Beim ersten Mal OK, beim dritten Mal ClassCastException!
        System.out.println(word);
    }
}

Ausgabe:

Hund
Katze
Exception in thread "main" java.lang.ClassCastException: java.lang.Integer cannot be cast to java.lang.String

c) Unübersichtlicher und fehleranfälliger Code

  • Viele Casts führen zu weniger verständlichem Code.
  • Fehler sind schwieriger zu finden.

5. Zusammenfassung: Was bringen Generics?

  • Typsicherheit: Fehler werden früh erkannt (bei der Kompilierung).
  • Weniger Casts: Weniger Fehlerquellen und mehr Lesbarkeit.
  • Besser wartbarer Code: Klar definierte Typen.
  • Einheitliche Klassen für verschiedene Typen: Einmal geschrieben, mehrfach verwendbar.

6. Zum Weiterarbeiten

Aufgaben:

  1. Erstelle eine generische Klasse Pair<T, U>, die zwei beliebige Werte speichert.
  2. Schreibe eine Methode, die eine List<Integer> entgegennimmt und die Summe berechnet.
  3. Überlege: Welche Fehler könnten auftreten, wenn du diese Aufgabe ohne Generics löst?

Merke: Generics sind ein Werkzeug, das deinen Code sicherer und klarer macht. Sie helfen dir, Fehler früh zu finden und einmal geschriebenen Code für verschiedene Typen wiederzuverwenden.
