Handout: Funktionale Nutzung von Optional

1. Verkettung mit map() und flatMap()

Mit den Methoden map() und flatMap() lässt sich Optional wie ein kleiner Stream behandeln: Du kannst Werte im Optional transformieren, ohne explizit zu prüfen, ob ein Wert vorhanden ist.

a) .map(Function<T, R>)

  • Wendet eine Funktion auf den Wert an, falls vorhanden.
  • Gibt ein neues Optional zurück – entweder mit dem Ergebnis, oder leer, falls der Ursprungswert leer war.

Beispiel:

Optional<String> name = Optional.of("Anna");
Optional<Integer> laenge = name.map(String::length);
System.out.println(laenge.orElse(0)); // 4

Optional<String> leer = Optional.empty();
Optional<Integer> leerLaenge = leer.map(String::length);
System.out.println(leerLaenge.isPresent()); // false

b) .flatMap(Function<T, Optional<R>>)

  • Wendet eine Funktion an, die selbst ein Optional zurückgibt.
  • Praktisch für verschachtelte Abfragen, bei denen Methoden bereits ein Optional liefern.

Beispiel:

Optional<String> findeNutzername(String userId) {
    // Liefert Optional.empty(), wenn userId unbekannt
}

Optional<String> userId = Optional.of("42");
Optional<String> nutzername = userId.flatMap(Main::findeNutzername);

Unterschied zu map():

  • map() würde in diesem Fall ein Optional<Optional<String>> liefern (Doppel-Optional).
  • flatMap() „entpackt“ das innere Optional automatisch.

2. Bedingtes Weiterarbeiten mit filter()

Mit filter() kann man Werte im Optional nach einer Bedingung „durchlassen“ oder „verwerfen“.

Beispiel:

Optional<String> name = Optional.of("Anna");
Optional<String> nurMitA = name.filter(n -> n.startsWith("A"));
System.out.println(nurMitA.isPresent()); // true

Optional<String> leer = name.filter(n -> n.startsWith("B"));
System.out.println(leer.isPresent()); // false
  • filter() gibt das ursprüngliche Optional zurück, wenn die Bedingung erfüllt ist, sonst ein leeres Optional.

3. Optionals in Kombination mit Streams

Optional und Streams können zusammen verwendet werden, zum Beispiel:

a) Optional in einen Stream umwandeln (stream(), ab Java 9)

  • Damit kann ein Optional in Stream-Operationen eingebaut werden.
  • Ein Optional mit Wert wird zu einem Stream mit einem Element, ein leeres Optional zu einem leeren Stream.

Beispiel:

Optional<String> opt = Optional.of("Hallo");
opt.stream().forEach(System.out::println); // Gibt "Hallo" aus

Optional<String> leer = Optional.empty();
leer.stream().forEach(System.out::println); // Gibt nichts aus

b) Optional als Rückgabewert in Stream-Pipelines

Beispiel:

List<String> namen = Arrays.asList("Anna", "Bob", "Clara");

// Suche den ersten Namen, der mit „C“ beginnt:
Optional<String> gefunden = namen.stream()
    .filter(n -> n.startsWith("C"))
    .findFirst();

Oder: Mehrere Optionals in einen Stream einbinden:

List<Optional<String>> optionals = Arrays.asList(
    Optional.of("a"),
    Optional.empty(),
    Optional.of("b")
);

List<String> werte = optionals.stream()
    .flatMap(Optional::stream)
    .collect(Collectors.toList());
// Ergebnis: ["a", "b"]

4. Praxis-Tipps

  • Nutze map() für einfache Transformationen, flatMap() für verschachtelte Optionals.
  • Mit filter() kannst du Optionals bequem bedingt „weiterreichen“.
  • Durch Kombination von Optionals und Streams wird Code besonders ausdrucksstark und kompakt.

5. Übungen zum Selbststudium

  1. Wandle einen Optional mit map() in einen Optional um, der die Länge des Strings enthält.
  2. Nutze filter(), um aus einem Optional nur Werte zu erhalten, die länger als 5 Zeichen sind.
  3. Wandle eine Liste von Optionals in eine Liste aller enthaltenen Werte (ohne leere), indem du Streams und flatMap() verwendest.

6. Merksätze

  • Mit map und flatMap können Optionals elegant verarbeitet und verschachtelt werden.
  • filter() macht bedingte Wertweitergabe einfach und sicher.
  • Optional und Stream ergänzen sich perfekt, um null-sicheren, modernen Code zu schreiben.
