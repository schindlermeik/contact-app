Handout: Motivation und Problemstellung – NullPointerException (NPE)

1. Was ist eine NullPointerException (NPE)?

Die NullPointerException (NPE) ist eine der häufigsten und berüchtigtsten Laufzeitfehler in Java. Sie tritt auf, wenn versucht wird, eine Methode oder ein Feld auf einer null-Referenz aufzurufen oder zu verwenden.

Beispiel:

String name = null;
int laenge = name.length(); // Hier wird eine NPE ausgelöst!

Hier ist die Variable name nicht auf ein Objekt, sondern auf null gesetzt. Ein Methodenaufruf wie name.length() ist daher nicht möglich und führt zum Programmabbruch mit einer NPE.

2. Typische Ursachen und Beispiele für NPEs in Java

a) Nicht initialisierte Objekte

Person person;
person.getName(); // person ist nicht initialisiert – NPE!

b) Rückgabewert einer Methode ist null

Person person = datenbank.findUserById(42); // Liefert null, wenn User nicht gefunden!
System.out.println(person.getName()); // NPE, wenn person null ist

c) Elemente in Arrays oder Collections sind null

String[] namen = new String[3]; // Standardmäßig: [null, null, null]
System.out.println(namen[0].length()); // NPE!

d) Verschachtelte Zugriffe auf Felder/Methoden

Person person = new Person();
// person.getAdresse() könnte null sein!
String stadt = person.getAdresse().getStadt(); // NPE, wenn Adresse null ist

e) Automatisches Unboxing von null

Integer wert = null;
int x = wert; // Unboxing null → NPE!

3. Warum ist das Thema heute noch relevant?

  • Sehr häufige Fehlerursache: Die NPE ist einer der meistgeworfenen Exceptions in Java-Programmen aller Art.
  • Schwer zu debuggen: Oft fällt der Fehler erst zur Laufzeit und an schwer nachvollziehbarer Stelle auf.
  • Hohe Fehlerkosten: NPEs können Programme abstürzen lassen, zu Datenverlust oder unerwartetem Verhalten führen.
  • Komplexere Codebasis = mehr Risiko: In modernen Anwendungen mit vielen Klassen, Abhängigkeiten, Datenbankzugriffen usw. ist die Gefahr besonders groß.
  • Entwickleralltag: Jeder Java-Entwickler wird mit NPEs konfrontiert und sollte sie frühzeitig erkennen und vermeiden.

5. Merksätze

  • Eine NullPointerException entsteht immer dann, wenn auf eine Referenz zugegriffen wird, die null ist.
  • Sie gehört zu den häufigsten und am schwierigsten zu findenden Fehlern in Java.
  • Das Ziel moderner Java-Programmierung ist es, NullPointerExceptions möglichst zu vermeiden und mit Alternativen wie Optional sicherer zu arbeiten.
