Handout: Einführung in Optional

1. Was ist Optional?

Optional ist eine Klasse, die seit Java 8 im Paket java.util verfügbar ist. Sie dient dazu, die Möglichkeit des Fehlens eines Werts explizit und sicher im Typ auszudrücken, anstatt wie bisher null zu verwenden.

  • Ein Optional<T> kann entweder einen Wert vom Typ T enthalten – oder leer sein (kein Wert).
  • Optional ist ein Container, der entweder „voll“ (Wert vorhanden) oder „leer“ (kein Wert) ist.

Beispiel:

Optional<String> vielleichtName = Optional.of("Anna");
Optional<String> keinName = Optional.empty();

2. Ziele und Vorteile von Optional

  • NullPointerExceptions vermeiden: Mit Optional wird das Fehlen eines Wertes explizit im Typ sichtbar – und kann nicht versehentlich übersehen werden.
  • Explizite Fehlerbehandlung: Der Umgang mit möglicherweise fehlenden Werten wird erzwingbar: Der Entwickler muss sich aktiv entscheiden, wie er mit dem „leeren Fall“ umgeht.
  • Bessere Lesbarkeit: Methoden, die ein Optional zurückgeben, signalisieren klar, dass das Ergebnis fehlen kann – statt versteckter null-Rückgaben.
  • Fördert funktionalen Stil: Viele Methoden auf Optional (wie map, flatMap, filter) erlauben eine dekorative, ausdrucksstarke Verarbeitung von Werten.

Praxisbeispiel: Ohne Optional:

public String findeEmail(String name) {
    if (db.contains(name)) {
        return db.getEmail(name);
    } else {
        return null;
    }
}

Mit Optional:

public Optional<String> findeEmail(String name) {
    if (db.contains(name)) {
        return Optional.of(db.getEmail(name));
    } else {
        return Optional.empty();
    }
}

Jetzt ist klar: Das Ergebnis kann fehlen – und der Aufrufer muss damit umgehen!

3. Vergleich: Null-Referenz vs. Optional

Null-Referenz Optional
Fehlerursache Kann zu NPE führen, wenn Zugriff auf null Keine NPE, aber erzwungene Prüfung
Semantik Fehlen eines Wertes oft nicht sichtbar Fehlen wird explizit signalisiert
Handhabung Vergessen von Null-Checks häufig Muss als Optional behandelt werden
Typischer Umgang if (obj != null) if (optional.isPresent()) oder besser: Methoden von Optional nutzen
Rückgabe von Methoden Oft null Empfohlen: Optional<T>
Lesbarkeit Fehlende Werte schwer zu erkennen Deutlich erkennbar im Methodenkopf

4. Merksätze

  • Optional ist ein sicherer Container für möglicherweise fehlende Werte.
  • Die Verwendung von Optional erzwingt einen bewussten Umgang mit dem „kein Wert“-Fall.
  • Optional erhöht die Lesbarkeit und Sicherheit von APIs und Methoden.
