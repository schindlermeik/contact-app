Handout: Wert aus Optional auslesen

1. get(), isPresent(), isEmpty()

a) .get()

  • Gibt den Wert im Optional zurück – wenn einer vorhanden ist.
  • Achtung: Wenn kein Wert enthalten ist (leeres Optional), wird eine NoSuchElementException geworfen!
  • Empfehlung: Immer nur zusammen mit .isPresent() verwenden oder besser die sichereren Alternativen (orElse, orElseGet, etc.).

Beispiel:

Optional<String> opt = Optional.of("Anna");
System.out.println(opt.get()); // Anna

Optional<String> leer = Optional.empty();
// leer.get(); // → NoSuchElementException!

b) .isPresent() und .isEmpty()

  • .isPresent() gibt true zurück, wenn ein Wert vorhanden ist, sonst false.
  • .isEmpty() gibt true zurück, wenn kein Wert vorhanden ist (ab Java 11).

Beispiel:

if (opt.isPresent()) {
    System.out.println(opt.get());
}
if (opt.isEmpty()) {
    System.out.println("Kein Wert enthalten!");
}

2. Wert mit Default: orElse(), orElseGet()

a) .orElse(T other)

  • Gibt den Wert im Optional zurück – oder den angegebenen Standardwert, falls das Optional leer ist.
Optional<String> opt = Optional.empty();
String name = opt.orElse("Standardwert");
System.out.println(name); // Standardwert

b) .orElseGet(Supplier<? extends T>)

  • Wie .orElse(), aber der Standardwert wird erst berechnet, wenn nötig (also nur, wenn das Optional leer ist). Praktisch für rechenintensive oder komplexe Standardwerte.
String name = opt.orElseGet(() -> "Wert per Supplier");
System.out.println(name); // Wert per Supplier

Unterschied:

  • .orElse() berechnet immer seinen Wert – auch wenn er nicht gebraucht wird.
  • .orElseGet() ruft den Supplier nur auf, wenn das Optional leer ist.

3. Wert bei Fehlen: orElseThrow()

  • Löst eine Exception aus, wenn kein Wert im Optional enthalten ist.
  • Seit Java 10 ohne Argument möglich (wirft dann NoSuchElementException), aber auch mit Supplier für eigene Exceptiontypen nutzbar.
String wert = opt.orElseThrow(); // NoSuchElementException, wenn leer

String wert2 = opt.orElseThrow(() -> new IllegalStateException("Kein Wert da!"));

Nützlich, um Fehler gezielt zu behandeln oder bei Pflichtwerten.

4. Beispielhafte Fehlerquellen bei der Verwendung

  • get() ohne vorherigen Check: Wer unbedacht optional.get() nutzt, riskiert wieder Laufzeitfehler – besser Alternativen verwenden!
  • Zu häufiger Einsatz von .isPresent()/.isEmpty(): Verleitet dazu, Optional wie einen Nullwert zu behandeln – nutze besser die funktionalen Methoden von Optional (ifPresent, map, flatMap, …).
  • Vergessen, mit leerem Optional zu rechnen: Immer davon ausgehen, dass ein Optional auch leer sein kann!
  • Mit null als Default-Wert arbeiten: orElse(null) widerspricht dem Sinn von Optional – besser sinnvolle Werte oder gezielte Fehlerbehandlung nutzen.

5. Praxis-Tipp

  • Für ein Optional niemals null als Standardwert verwenden.
  • Nutze möglichst die Methoden orElse, orElseGet, orElseThrow für sauberen Code.
  • Moderne Streams und Optional-Methoden erlauben es, fast ganz ohne Null- oder Vorhandenheitsprüfungen auszukommen.

6. Übungen zum Selbststudium

  1. Erzeuge ein leeres Optional und rufe darauf get() auf. Was passiert?
  2. Schreibe eine Methode, die ein Optional zurückgibt und bei Fehlen des Werts eine eigene Exception wirft.
  3. Ersetze im eigenen Code alle if (obj != null)-Prüfungen durch Optional und nutze orElse/orElseGet.

7. Merksätze

  • get() nur verwenden, wenn sicher ein Wert vorhanden ist – ansonsten Alternativen nutzen!
  • Mit orElse, orElseGet und orElseThrow werden Standardwerte und Fehlerbehandlung elegant geregelt.
  • Nie mit null als Default arbeiten – das ist gerade das Problem, das Optional löst!
