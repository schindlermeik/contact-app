Handout: Praktische Übungen und Beispiele zu Optional

1. Typische Methoden mit Optional als Rückgabe implementieren

Aufgabe 1

Implementiere eine Methode, die einen Benutzernamen zu einer ID sucht. Der Benutzername kann fehlen. Gib das Ergebnis als Optional<String> zurück.

Lösung:

public Optional<String> findeBenutzernameById(String id) {
    Map<String, String> benutzerDb = Map.of(
        "101", "Anna",
        "102", "Bernd"
    );
    if (benutzerDb.containsKey(id)) {
        return Optional.of(benutzerDb.get(id));
    } else {
        return Optional.empty();
    }
}

// Verwendung:
Optional<String> benutzer = findeBenutzernameById("101");
benutzer.ifPresent(System.out::println); // Gibt "Anna" aus

2. Fehlerbehandlung mit Optional im Vergleich zu klassischen Null-Checks

Aufgabe 2

Vergleiche zwei Varianten einer Methode – einmal mit null, einmal mit Optional – und wie man das Ergebnis verarbeitet.

a) Mit null:

public String findeEmail(String name) {
    if ("Anna".equals(name)) {
        return "anna@example.com";
    } else {
        return null;
    }
}

// Fehleranfällig:
String email = findeEmail("Bernd");
if (email != null) {
    System.out.println(email.toLowerCase());
} else {
    System.out.println("Keine Email gefunden!");
}

b) Mit Optional:

public Optional<String> findeEmailOptional(String name) {
    if ("Anna".equals(name)) {
        return Optional.of("anna@example.com");
    } else {
        return Optional.empty();
    }
}

// Sicherer:
String email = findeEmailOptional("Bernd")
    .map(String::toLowerCase)
    .orElse("Keine Email gefunden!");

System.out.println(email);

Vorteile:

  • Keine Gefahr einer NullPointerException.
  • Der Umgang mit dem „kein Wert“-Fall ist typisiert und klar geregelt.

3. Refactoring: Null-basierte Methode auf Optional umstellen

Aufgabe 3

Nimm eine bestehende Methode, die bisher null zurückgibt, und stelle sie auf Optional um.

Vorher:

public String findeTelefonnummer(String name) {
    if ("Anna".equals(name)) {
        return "01234";
    }
    return null;
}

Nachher:

public Optional<String> findeTelefonnummer(String name) {
    if ("Anna".equals(name)) {
        return Optional.of("01234");
    }
    return Optional.empty();
}

Verwendung:

String nummer = findeTelefonnummer("Bernd")
    .orElse("Nicht gefunden");
System.out.println(nummer); // Gibt "Nicht gefunden" aus

4. Weitere Übungsaufgaben zum Selbststudium

  1. Implementiere eine Methode, die eine Person nach E-Mail sucht und ein Optional zurückgibt.
  2. Nutze map() und filter(), um aus einem Optional nur dann die Länge zurückzugeben, wenn der String länger als 5 Zeichen ist.
  3. Führe ein Refactoring einer bestehenden Methode durch, die bisher mit null arbeitet, und verwende stattdessen Optional.

5. Merksätze

  • Optional als Rückgabetyp macht Code sicherer und klarer – keine NullPointerExceptions mehr!
  • Verwende map(), orElse() und Co., um Optionals elegant auszuwerten.
  • Beim Refactoring von „null“-Methoden auf Optional ändert sich oft wenig Code, aber die Lesbarkeit und Sicherheit steigt deutlich.
