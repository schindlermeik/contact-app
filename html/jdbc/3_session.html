Handout: Arbeiten mit SQL-Befehlen über JDBC

Um aus einer Java-Anwendung heraus mit einer Datenbank zu interagieren, nutzen wir JDBC (Java Database Connectivity). JDBC ist eine API, die eine einheitliche Schnittstelle zu verschiedenen Datenbanksystemen bietet. Im Kern geht es darum, eine Verbindung zur Datenbank herzustellen und dann über spezielle Objekte SQL-Befehle auszuführen.

3.1 Verwenden von Statement und PreparedStatement

In JDBC gibt es mehrere Möglichkeiten, SQL-Befehle an die Datenbank zu senden. Die gängigsten sind Statement und PreparedStatement.

Das Statement-Objekt

Das Statement-Objekt ist die einfachste Form, einen SQL-Befehl auszuführen. Du erstellst es direkt aus dem Connection-Objekt.

// Annahme: 'connection' ist ein aktives Connection-Objekt
Statement statement = connection.createStatement();

Mit einem Statement sendest du den kompletten SQL-Befehl als einen einzigen String an die Datenbank. Das bedeutet, wenn du dynamische Werte (z.B. aus Benutzereingaben) in den Befehl einfügen möchtest, musst du diese direkt in den String einbetten.

Beispiel:

String name = "Max Mustermann";
String sql = "INSERT INTO user (name) VALUES ('" + name + "')";
statement.executeUpdate(sql);

Nachteile von Statement:

  • Sicherheitsrisiko (SQL-Injection): Dies ist der größte Nachteil. Wenn der Wert name von einem Benutzer kommt, könnte dieser bösartigen Code wie ''); DROP TABLE user;-- eingeben. Die Datenbank würde versuchen, diesen Code auszuführen, was zu ernsthaften Sicherheitsproblemen führen kann.
  • Performance: Bei wiederholter Ausführung desselben Befehls mit nur unterschiedlichen Werten muss die Datenbank den Befehl jedes Mal neu parsen und kompilieren. Dies ist ineffizient.

Das PreparedStatement-Objekt

PreparedStatement ist die bevorzugte Methode, um SQL-Befehle auszuführen. Auch dieses Objekt erstellst du aus dem Connection-Objekt, allerdings über einen anderen Mechanismus.

// Der SQL-Befehl enthält Platzhalter (?)
String sql = "INSERT INTO user (name) VALUES (?)";
PreparedStatement preparedStatement = connection.prepareStatement(sql);

Der entscheidende Unterschied ist, dass der SQL-Befehl Platzhalter (?) für die dynamischen Werte enthält. Die Werte werden erst nach der Erstellung des PreparedStatement-Objekts über sogenannte Setter-Methoden (setString, setInt, etc.) eingefügt.

Beispiel:

String name = "Max Mustermann";
String sql = "INSERT INTO user (name) VALUES (?)";
PreparedStatement preparedStatement = connection.prepareStatement(sql);

// Setze den Wert für den ersten Platzhalter
preparedStatement.setString(1, name); 
preparedStatement.executeUpdate();

Vorteile von PreparedStatement:

  • Sicherheit: Die Werte werden erst zur Laufzeit an den Befehl gebunden. Die Datenbank behandelt die Werte als reine Daten und nicht als ausführbaren Code, wodurch das Risiko von SQL-Injections eliminiert wird.
  • Performance: Die Datenbank parst und kompiliert den SQL-Befehl nur einmal. Bei wiederholter Ausführung mit neuen Werten muss nur der Datenaustausch erfolgen, was die Performance deutlich verbessert.
  • Lesbarkeit: Der Code ist klarer, da SQL-Befehl und Daten getrennt voneinander behandelt werden.

3.2 Unterschied zwischen Statement, PreparedStatement und CallableStatement

Feature Statement PreparedStatement CallableStatement
Zweck Einfachste Ausführung statischer SQL-Befehle. Ausführung von SQL-Befehlen mit dynamischen Werten (Platzhalter). Aufruf von gespeicherten Prozeduren (Stored Procedures) in der Datenbank.
Sicherheit Unsicher bei dynamischen Werten (anfällig für SQL-Injection). Sicher gegen SQL-Injection durch Platzhalter. Sicher.
Performance Ineffizient bei wiederholter Ausführung. Sehr effizient bei wiederholter Ausführung, da der Befehl nur einmal kompiliert wird. Effizient, da die Logik in der Datenbank gespeichert und optimiert ist.
Syntax SQL-Befehl als String mit konkatenierten Werten. SQL-Befehl mit Platzhaltern (?). Werte werden mit Setter-Methoden gebunden. Spezielle Syntax zum Aufruf von Prozeduren: {call my_procedure(?, ?)}.

CallableStatement

CallableStatement ist ein spezialisiertes Objekt, das verwendet wird, um gespeicherte Prozeduren und Funktionen in der Datenbank aufzurufen. Gespeicherte Prozeduren sind vorgefertigte Code-Blöcke, die in der Datenbank gespeichert und von dort ausgeführt werden. Sie können Eingabe- (IN), Ausgabe- (OUT) und Ein-/Ausgabeparameter (INOUT) haben.

Beispiel für ein CallableStatement:

// Annahme: Es existiert eine Prozedur `get_user_by_id` in der Datenbank
String sql = "{call get_user_by_id(?, ?)}";
CallableStatement callableStatement = connection.prepareCall(sql);

// Setze den Eingabeparameter
callableStatement.setInt(1, 123);

// Registriere den Ausgabeparameter
callableStatement.registerOutParameter(2, Types.VARCHAR);

// Führe die Prozedur aus
callableStatement.execute();

// Lese den Ausgabewert
String userName = callableStatement.getString(2);

3.3 Ausführen von SQL-Befehlen (executeQuery, executeUpdate)

Nachdem du ein Statement- oder PreparedStatement-Objekt erstellt hast, musst du den SQL-Befehl an die Datenbank senden. Dafür gibt es zwei Hauptmethoden:

executeQuery()

Diese Methode wird verwendet, um SQL-Befehle auszuführen, die Daten von der Datenbank abfragen, also typischerweise SELECT-Befehle.

  • Rückgabewert: Ein ResultSet-Objekt.
  • Was ist ein ResultSet? Ein ResultSet ist ein Objekt, das die Daten der Abfrage zeilenweise speichert. Du kannst mit einem Cursor durch die Zeilen iterieren und die Werte der einzelnen Spalten auslesen.

Beispiel:

String sql = "SELECT id, name FROM user WHERE id = 1";
Statement statement = connection.createStatement();
ResultSet resultSet = statement.executeQuery(sql);

// Durch das ResultSet iterieren
if (resultSet.next()) {
    int id = resultSet.getInt("id");
    String name = resultSet.getString("name");
    System.out.println("ID: " + id + ", Name: " + name);
}

resultSet.close();

Die Methode resultSet.next() bewegt den Cursor zur nächsten Zeile und gibt true zurück, solange noch Zeilen vorhanden sind.

executeUpdate()

Diese Methode wird für SQL-Befehle verwendet, die die Datenbank modifizieren, d.h. INSERT, UPDATE oder DELETE-Befehle.

  • Rückgabewert: Ein int-Wert, der die Anzahl der betroffenen Zeilen angibt.
  • Wann verwenden? Immer dann, wenn du Daten hinzufügen, ändern oder löschen möchtest.

Beispiel:

String sql = "UPDATE user SET name = 'Neue Name' WHERE id = 1";
Statement statement = connection.createStatement();
int affectedRows = statement.executeUpdate(sql);

System.out.println("Es wurden " + affectedRows + " Zeilen aktualisiert.");

Die Methode execute()

Neben den beiden spezifischen Methoden gibt es auch die universelle Methode execute(). Sie kann für jeden SQL-Befehl verwendet werden.

  • Rückgabewert: Ein boolean-Wert.
    • true wird zurückgegeben, wenn das Ergebnis ein ResultSet ist (z.B. bei einem SELECT).
    • false wird zurückgegeben, wenn das Ergebnis ein Update-Count (die Anzahl der betroffenen Zeilen) ist (z.B. bei INSERT, UPDATE, DELETE).
  • Nachteile: Diese Methode ist weniger intuitiv. Es ist in der Regel besser, executeQuery oder executeUpdate zu verwenden, da der Zweck des Befehls dadurch klarer wird.

Best Practices

  • Ressourcen schließen: Stelle sicher, dass du alle JDBC-Ressourcen (ResultSet, Statement, PreparedStatement und Connection) am Ende immer schließt, um Speicherlecks zu vermeiden. Der beste Weg hierfür ist die Verwendung eines try-with-resources-Blocks.
  • PreparedStatement verwenden: Nutze PreparedStatement gegenüber Statement wann immer möglich, um die Sicherheit und Performance deiner Anwendung zu gewährleisten.
  • Verbindungs-Pools: In produktiven Anwendungen solltest du statt einzelner Connection-Objekte einen Verbindungs-Pool verwenden, um die Performance zu optimieren.
