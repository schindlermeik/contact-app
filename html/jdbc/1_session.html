Handout: Einführung in JDBC (Java Database Connectivity)

1.1 Was ist JDBC?

JDBC steht für Java Database Connectivity und ist eine standardisierte Programmierschnittstelle (API), die es Java-Programmen ermöglicht, auf relationale Datenbanken (z. B. PostgreSQL, MySQL, Oracle) zuzugreifen und SQL-Befehle auszuführen.

Ziele von JDBC:

  • Kommunikation zwischen Java-Anwendung und Datenbank
  • Ausführung von SQL-Statements (SELECT, INSERT, UPDATE, DELETE)
  • Verwaltung von Datenbankverbindungen, Transaktionen und Ergebnismengen

Merke: JDBC ist ein API-Standard, keine konkrete Implementierung. Die Anbindung an die jeweilige Datenbank erfolgt über einen passenden JDBC-Treiber.

1.2 Aufbau und Architektur von JDBC

Komponenten der JDBC-Architektur:

  • JDBC API: Java-Bibliothek mit Klassen & Interfaces für den Datenbankzugriff (z. B. Connection, Statement, ResultSet).
  • JDBC Driver Manager: Verbindet die Anwendung mit dem passenden JDBC-Treiber und steuert die Kommunikation.
  • JDBC-Treiber: Treibersoftware, die SQL-Anfragen von Java in das Datenbankprotokoll übersetzt (und umgekehrt).
  • Datenbank: Das eigentliche Datenbanksystem (z. B. PostgreSQL).

Vereinfachte Architektur:

Java-Anwendung
    |
[JDBC API]
    |
[JDBC Driver Manager]
    |
[JDBC-Treiber] <----> [Datenbank (PostgreSQL)]

Wichtige JDBC-Klassen & -Interfaces:

  • DriverManager: Verbindet die Anwendung mit der Datenbank.
  • Connection: Repräsentiert die Verbindung zur Datenbank.
  • Statement / PreparedStatement: Zum Senden von SQL-Befehlen.
  • ResultSet: Repräsentiert das Ergebnis einer Abfrage.

1.3 JDBC-Treiber-Typen und ihre Unterschiede

JDBC-Treiber sind spezielle Programme, die als "Brücke" zwischen Java-Code und Datenbank dienen. Es gibt vier Typen:

Typ 1: JDBC-ODBC Bridge Driver

  • Übersetzt JDBC-Calls in ODBC-Calls, die dann von einem ODBC-Treiber verarbeitet werden.
  • Veraltet, seit Java 8 entfernt!

Typ 2: Native-API Driver

  • Wandelt JDBC-Calls in native API-Aufrufe der Datenbank um.
  • Benötigt Datenbankspezifische Bibliotheken auf dem System.

Typ 3: Network Protocol Driver

  • JDBC-Calls werden in einen netzwerkbasierten Protokollaufruf umgewandelt.
  • Benötigt einen Vermittlungsserver.

Typ 4: Direct-to-Database Pure Java Driver (am gebräuchlichsten!)

  • Komplett in Java geschrieben, kommuniziert direkt mit der Datenbank.
  • Beispiel: Der PostgreSQL-Treiber org.postgresql.Driver
  • Keine zusätzliche Software nötig, Plattform-unabhängig.

In der Praxis verwendest du fast immer einen Typ-4-Treiber.

1.4 Voraussetzungen (JDBC-API, Datenbank, Treiber-JARs)

Um mit JDBC und einer PostgreSQL-Datenbank zu arbeiten, benötigst du:

1. Java Development Kit (JDK 11)

  • Stelle sicher, dass Java 11 installiert ist. Prüfe mit:

    java -version

2. Maven-Projekt (zum Verwalten von Abhängigkeiten)

  • Erstelle ein Maven-Projekt oder verwende ein vorhandenes.
  • Die Abhängigkeiten werden in der pom.xml verwaltet.

3. PostgreSQL-Datenbank

  • Installiere und starte eine PostgreSQL-Instanz.

  • Lege eine Datenbank und ggf. einen Benutzer an.

  • Beispiel (auf der Konsole):

    createdb meine_datenbank
createuser mein_benutzer --pwprompt

4. PostgreSQL JDBC-Treiber in Maven einbinden

  • Füge den Treiber als Dependency in deiner pom.xml hinzu:

    <dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <version>42.7.3</version>
</dependency>

5. Datenbank-Zugangsdaten bereithalten

  • Host (z. B. localhost)
  • Port (Standard: 5432)
  • Datenbankname
  • Benutzername & Passwort

6. Beispiel für Verbindungs-URL (Connection String)

String url = "jdbc:postgresql://localhost:5432/meine_datenbank";
String user = "mein_benutzer";
String password = "mein_passwort";

Exkurs: Erstes kleines Verbindungsbeispiel

So sieht ein einfacher JDBC-Verbindungsaufbau zu einer PostgreSQL-Datenbank aus:

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class JdbcTest {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/meine_datenbank";
        String user = "mein_benutzer";
        String password = "mein_passwort";

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            if (conn != null) {
                System.out.println("Verbindung zur Datenbank erfolgreich!");
            } else {
                System.out.println("Verbindung fehlgeschlagen!");
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

Zusammenfassung – Schritte für den Einstieg

  1. Installiere JDK 11 und richte Maven ein.
  2. Erstelle eine PostgreSQL-Datenbank und Benutzer.
  3. Lege ein Maven-Projekt mit PostgreSQL-Treiber an.
  4. Verwende die passenden Zugangsdaten und baue die Verbindung über JDBC auf.
