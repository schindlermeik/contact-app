Handout: Grundlagen der Verbindung zu einer Datenbank mit JDBC

2.1 Aufbau einer Datenbankverbindung (Connection)

Der Zugriff auf eine relationale Datenbank mit Java erfolgt über die JDBC-API. Der wichtigste Schritt ist das Herstellen einer Verbindung – dies geschieht mit dem Connection-Objekt aus dem Paket java.sql.

Ablauf:

  1. JDBC-Treiber sicherstellen: Der PostgreSQL-Treiber muss im Projekt (über Maven) eingebunden sein.

  2. Datenbankverbindung herstellen: Mit DriverManager.getConnection(...) erzeugst du ein Connection-Objekt, das die Verbindung zur Datenbank kapselt.

Beispielcode:

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class DBConnectionExample {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/meine_datenbank";
        String user = "mein_benutzer";
        String password = "mein_passwort";

        try {
            Connection conn = DriverManager.getConnection(url, user, password);
            System.out.println("Verbindung erfolgreich!");
            // Weiterverarbeitung ...
            conn.close(); // Wichtig: Verbindung wieder schließen!
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

Hinweis: Ab Java 6 (und mit JDBC 4.0-konformen Treibern) ist kein explizites Class.forName("org.postgresql.Driver") mehr nötig.

2.2 Typischer Verbindungsstring (URL, Benutzername, Passwort)

JDBC-Verbindungs-URL für PostgreSQL

Das Format:

jdbc:postgresql://[host]:[port]/[datenbank]
  • host: z. B. localhost (oder IP-Adresse, wenn extern)
  • port: Standardmäßig 5432
  • datenbank: Name deiner Datenbank

Beispiel:

String url = "jdbc:postgresql://localhost:5432/meine_datenbank";
String user = "mein_benutzer";
String password = "mein_passwort";

Weitere Parameter

Du kannst in der URL auch weitere Einstellungen setzen, z. B.:

String url = "jdbc:postgresql://localhost:5432/meine_datenbank?ssl=false&currentSchema=public";

Tipp: Speichere sensible Zugangsdaten nie im Klartext im Quellcode! Nutze ggf. Umgebungsvariablen oder externe Konfigurationsdateien (z. B. .env, application.properties).

2.3 Ressourcenverwaltung (Öffnen und Schließen von Verbindungen)

Warum ist Ressourcenverwaltung wichtig?

  • Verbindungen verbrauchen Ressourcen: Jede offene Verbindung belegt Speicher und Netzwerk-Ressourcen.
  • Leckende Verbindungen: Werden Verbindungen nicht sauber geschlossen, kann die Datenbank keine neuen Verbindungen mehr annehmen (“Connection Leak”).
  • Fehlervermeidung: Unsachgemäße Verwaltung kann zu schwer nachvollziehbaren Fehlern führen.

Verbindungen schließen

Faustregel: Jede geöffnete Verbindung (Connection), jedes Statement (Statement, PreparedStatement) und jedes Ergebnis (ResultSet) muss wieder explizit geschlossen werden!

Ab Java 7: Try-with-Resources

Das try-with-resources-Statement sorgt automatisch dafür, dass alle Ressourcen am Ende des Blocks geschlossen werden – auch bei Fehlern.

Beispiel mit Try-with-Resources:

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;

public class TryWithResourcesExample {
    public static void main(String[] args) {
        String url = "jdbc:postgresql://localhost:5432/meine_datenbank";
        String user = "mein_benutzer";
        String password = "mein_passwort";

        // try-with-resources schließt die Verbindung automatisch
        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            System.out.println("Verbindung hergestellt!");
            // ... Datenbankoperationen ...
        } catch (SQLException e) {
            e.printStackTrace();
        }
        // conn wird hier automatisch geschlossen
    }
}

Wichtig:

  • Das try-with-resources-Konstrukt funktioniert für alle Objekte, die das Interface AutoCloseable implementieren – dazu gehören alle JDBC-Ressourcen (Connection, Statement, ResultSet).
  • Immer möglichst früh und in umgekehrter Reihenfolge zum Öffnen schließen: Erst ResultSet, dann Statement, zuletzt Connection.

Checkliste: Verbindung zu PostgreSQL mit JDBC herstellen

  1. PostgreSQL läuft und akzeptiert Verbindungen.

  2. JDBC-Treiber in der pom.xml eingebunden:

    <dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <version>42.7.3</version>
</dependency>

  3. Connection String, Benutzername, Passwort bereit.

  4. Verbindung mit try-with-resources öffnen.

  5. Nach Benutzung alle Ressourcen schließen (am besten automatisch).

Aufgaben für die Praxis

  1. Stelle eine Verbindung zu deiner PostgreSQL-Datenbank her.

    • Ändere den Connection String so, dass er zu deinem Setup passt.

  2. Probiere aus, was passiert, wenn du absichtlich einen falschen Benutzernamen oder ein falsches Passwort einträgst.

    • Was steht in der Fehlermeldung?

  3. Überlege, wie du Zugangsdaten sicher speichern kannst.

    • Lies dich ggf. zu Umgebungsvariablen oder Properties-Files ein.

Zusammenfassung

  • Verbindungen werden mit DriverManager.getConnection(...) aufgebaut.
  • Connection String, Benutzername, Passwort sind erforderlich.
  • Ressourcen wie Connection müssen immer geschlossen werden – am besten per try-with-resources.
  • Mit Maven kannst du den PostgreSQL-Treiber einfach einbinden.
