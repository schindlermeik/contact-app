Handout: Praktische Übung: Eine einfache Benutzerverwaltung mit JPA

In dieser Übung bauen wir eine Konsolenanwendung, die Benutzer (User) in einer PostgreSQL-Datenbank verwaltet. Du wirst alle gelernten Konzepte anwenden: Entitäten, Konfiguration, CRUD-Operationen und Abfragen.

10.1 Schritt 1: Projekt-Setup mit Maven

Erstelle ein neues Maven-Projekt. In deiner pom.xml fügst du die notwendigen Abhängigkeiten hinzu:

  1. JPA-Provider (Hibernate)
  2. JDBC-Treiber für PostgreSQL
  3. javax.xml.bind-api (wird von neueren Java-Versionen benötigt)

Deine pom.xml sollte so aussehen:

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>jpa-userverwaltung</artifactId>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.hibernate</groupId>
            <artifactId>hibernate-core</artifactId>
            <version>5.6.15.Final</version>
        </dependency>
        <dependency>
            <groupId>org.postgresql</groupId>
            <artifactId>postgresql</artifactId>
            <version>42.5.4</version>
        </dependency>
        <dependency>
            <groupId>javax.xml.bind</groupId>
            <artifactId>jaxb-api</artifactId>
            <version>2.3.1</version>
        </dependency>
    </dependencies>
</project>

10.2 Schritt 2: Entität und Konfiguration erstellen

A. Entitätsklasse User

Erstelle die Klasse User im Verzeichnis src/main/java/com/example/model/User.java. Diese Entität wird die Benutzerdaten speichern. Ein Benutzer hat eine ID, einen Namen und eine E-Mail-Adresse.

package com.example.model;

import javax.persistence.*;

@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "username", nullable = false, unique = true)
    private String name;

    @Column(nullable = false, unique = true)
    private String email;

    // Standard-Konstruktor
    public User() {
    }

    // Konstruktor für die Erstellung
    public User(String name, String email) {
        this.name = name;
        this.email = email;
    }

    // Getters and Setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }

    @Override
    public String toString() {
        return "User{" +
               "id=" + id +
               ", name='" + name + '\'' +
               ", email='" + email + '\'' +
               '}';
    }
}
B. Konfiguration in persistence.xml

Erstelle die Konfigurationsdatei im Verzeichnis src/main/resources/META-INF/persistence.xml. Passe hier deine PostgreSQL-Zugangsdaten an.

<?xml version="1.0" encoding="UTF-8"?>
<persistence xmlns="http://xmlns.jcp.org/xml/ns/persistence"
             xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
             xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/persistence http://xmlns.jcp.org/xml/ns/persistence/persistence_2_1.xsd"
             version="2.1">

    <persistence-unit name="userverwaltung_pu" transaction-type="RESOURCE_LOCAL">
        <provider>org.hibernate.jpa.HibernatePersistenceProvider</provider>
        <class>com.example.model.User</class>
        <properties>
            <property name="javax.persistence.jdbc.url" value="jdbc:postgresql://localhost:5432/jpa_db"/>
            <property name="javax.persistence.jdbc.user" value="dein_benutzer"/>
            <property name="javax.persistence.jdbc.password" value="dein_passwort"/>
            <property name="javax.persistence.jdbc.driver" value="org.postgresql.Driver"/>

            <property name="hibernate.dialect" value="org.hibernate.dialect.PostgreSQLDialect"/>
            <property name="hibernate.hbm2ddl.auto" value="update"/>
            <property name="hibernate.show_sql" value="true"/>
            <property name="hibernate.format_sql" value="true"/>
        </properties>
    </persistence-unit>
</persistence>
  • Wichtig: Achte darauf, dass der name des <persistence-unit>-Tags (hier userverwaltung_pu) exakt dem Namen entspricht, den du später im Code verwendest.

10.3 Schritt 3: CRUD-Operationen in einer Hauptklasse

Jetzt schreibst du eine Hauptklasse, um die JPA-Operationen auszuführen. Erstelle die Klasse MainApp im Verzeichnis src/main/java/com/example/MainApp.java.

package com.example;

import com.example.model.User;
import javax.persistence.EntityManager;
import javax.persistence.EntityManagerFactory;
import javax.persistence.Persistence;
import java.util.List;

public class MainApp {

    public static void main(String[] args) {

        // EntityManagerFactory erstellen
        // Der Name muss mit dem Namen in persistence.xml übereinstimmen
        EntityManagerFactory emf = Persistence.createEntityManagerFactory("userverwaltung_pu");
        EntityManager em = emf.createEntityManager();

        // 1. Benutzer erstellen und speichern (Persistieren)
        System.out.println("--- Benutzer erstellen ---");
        em.getTransaction().begin();
        User user1 = new User("max.mustermann", "max.mustermann@example.com");
        User user2 = new User("erika.mustermann", "erika.mustermann@example.com");
        em.persist(user1);
        em.persist(user2);
        em.getTransaction().commit();

        // 2. Alle Benutzer abfragen (JPQL)
        System.out.println("\n--- Alle Benutzer abfragen ---");
        List<User> users = em.createQuery("SELECT u FROM User u", User.class).getResultList();
        users.forEach(System.out::println);

        // 3. Benutzer aktualisieren
        System.out.println("\n--- Benutzer aktualisieren ---");
        em.getTransaction().begin();
        User userToUpdate = em.find(User.class, user1.getId());
        if (userToUpdate != null) {
            userToUpdate.setEmail("max.neue@adresse.com");
            System.out.println("Aktualisierter Benutzer: " + userToUpdate);
        }
        em.getTransaction().commit();

        // 4. Benutzer löschen
        System.out.println("\n--- Benutzer löschen ---");
        em.getTransaction().begin();
        User userToDelete = em.find(User.class, user2.getId());
        if (userToDelete != null) {
            em.remove(userToDelete);
            System.out.println("Benutzer gelöscht: " + userToDelete.getName());
        }
        em.getTransaction().commit();
        
        // Letzte Abfrage, um zu überprüfen, ob der Benutzer gelöscht wurde
        System.out.println("\n--- Benutzer nach Löschung ---");
        List<User> remainingUsers = em.createQuery("SELECT u FROM User u", User.class).getResultList();
        remainingUsers.forEach(System.out::println);

        // Aufräumen: EntityManager und EntityManagerFactory schließen
        em.close();
        emf.close();
    }
}

10.4 Schritt 4: Ausführen und Testen

  1. Stelle sicher, dass deine PostgreSQL-Datenbank läuft und die Datenbank jpa_db (oder wie immer du sie genannt hast) existiert.
  2. Öffne dein Projekt in einer IDE (z. B. IntelliJ IDEA, Eclipse) und führe die MainApp-Klasse aus.
  3. Beobachte die Konsolenausgabe. Du solltest die SQL-Befehle (CREATE TABLE, INSERT, UPDATE, DELETE) sehen, die Hibernate für dich generiert.
  4. Überprüfe mit einem Datenbank-Tool (z. B. pgAdmin oder der psql-Konsole), ob die Tabelle users erstellt wurde und die Daten korrekt eingefügt, aktualisiert und gelöscht wurden.

Mögliche Erweiterungen der Übung:

  • Beziehung hinzufügen: Erstelle eine UserProfile-Entität und verknüpfe sie mit der User-Entität über eine @OneToOne-Beziehung.
  • Named Query implementieren: Schreibe eine @NamedQuery in der User-Klasse, um Benutzer anhand ihrer E-Mail-Adresse zu finden.
  • Transaktionen absichtlich fehlschlagen lassen: Füge eine ungültige Operation in einen Transaktionsblock ein, um zu sehen, wie rollback funktioniert.

Durch das Durcharbeiten dieser praktischen Übung hast du nicht nur die Theorie verstanden, sondern auch gelernt, wie du JPA-Anwendungen von Grund auf aufbaust.
