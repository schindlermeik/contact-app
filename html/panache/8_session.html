Handout: Praktische Übungen & Mini-Projekt – Panache, Quarkus und PostgreSQL

8.1 Aufbau eines Mini-Microservice mit Panache und PostgreSQL

Ziel: Entwickle einen einfachen Microservice (z. B. User- oder Produktverwaltung) mit Quarkus, Panache und PostgreSQL.

Schritt 1: Neues Quarkus-Projekt anlegen

Am einfachsten per Quarkus Initializer

  • Extensions auswählen:

    • Hibernate ORM with Panache
    • JDBC Driver (PostgreSQL)
    • RESTEasy (für REST-API)
    • (Optional: RESTEasy Jackson für JSON)

Projekt lokal herunterladen und entpacken.

Schritt 2: Datenbank konfigurieren

Bearbeite die Datei src/main/resources/application.properties:

quarkus.datasource.db-kind=postgresql
quarkus.datasource.username=panache_user
quarkus.datasource.password=geheimespasswort
quarkus.datasource.jdbc.url=jdbc:postgresql://localhost:5432/panache_demo
quarkus.hibernate-orm.database.generation=drop-and-create
quarkus.hibernate-orm.log.sql=true

Tipp: Mit drop-and-create wird das Schema bei jedem Start neu gebaut – ideal für Experimente.

Schritt 3: PostgreSQL bereitstellen

  • Lokal: Starte PostgreSQL und lege DB/User an.

  • Oder per Docker:

    docker run --name pg-panache -e POSTGRES_DB=panache_demo \
  -e POSTGRES_USER=panache_user -e POSTGRES_PASSWORD=geheimespasswort \
  -p 5432:5432 -d postgres:16

8.2 Implementieren typischer Anwendungsfälle (z. B. User- oder Produktverwaltung)

Schritt 4: Entity und Repository/Service anlegen

Beispiel: User-Verwaltung mit Active Record Pattern

src/main/java/org/acme/model/User.java

package org.acme.model;

import io.quarkus.hibernate.orm.panache.PanacheEntity;
import javax.persistence.Entity;

@Entity
public class User extends PanacheEntity {
    public String name;
    public String email;
}

Schritt 5: REST-API-Ressource anlegen

src/main/java/org/acme/resource/UserResource.java

package org.acme.resource;

import org.acme.model.User;
import javax.ws.rs.*;
import javax.ws.rs.core.MediaType;
import java.util.List;

@Path("/users")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
public class UserResource {

    @GET
    public List<User> listAll() {
        return User.listAll();
    }

    @POST
    public void addUser(User user) {
        user.persist();
    }

    @GET
    @Path("/{id}")
    public User findById(@PathParam("id") Long id) {
        return User.findById(id);
    }

    @DELETE
    @Path("/{id}")
    public void delete(@PathParam("id") Long id) {
        User.deleteById(id);
    }
}

Hinweis: Das ist eine minimalistische REST-API – ideal für das Verständnis und erste Experimente.

Schritt 6: Anwendung starten & testen

Starte Quarkus im Dev-Modus:

./mvnw quarkus:dev

Teste die REST-API z. B. mit curl, Postman oder httpie:

  • Alle User auflisten: GET http://localhost:8080/users

  • Neuen User anlegen: POST http://localhost:8080/users mit Body:

    {
  "name": "Anna",
  "email": "anna@example.com"
}

  • Einzelnen User abrufen: GET http://localhost:8080/users/1

  • User löschen: DELETE http://localhost:8080/users/1

8.3 Erweiterung: REST-Endpunkte für Panache-basierte Entitäten

Erweitere die API um weitere Endpunkte, z. B.:

  • Suche nach Name:

    @GET
@Path("/search/{name}")
public List<User> searchByName(@PathParam("name") String name) {
    return User.find("name", name).list();
}

  • Paging & Sortierung:

    @GET
@Path("/paged")
public List<User> paged(
    @QueryParam("page") @DefaultValue("0") int page,
    @QueryParam("size") @DefaultValue("10") int size) {
    return User.findAll().page(page, size).list();
}

Baue auch einen einfachen "PUT"-Endpunkt für Updates:

@PUT
@Path("/{id}")
public void update(@PathParam("id") Long id, User updatedUser) {
    User user = User.findById(id);
    if (user != null) {
        user.name = updatedUser.name;
        user.email = updatedUser.email;
        user.persist();
    }
}

8.4 Optional: Containerisierung mit Quarkus und PostgreSQL (Docker)

Quarkus als Docker-Container bauen

  1. Dockerfile im Projekt anlegen (meist wird bei mvn quarkus:create schon eines generiert): Beispiel für JVM-Modus:

    FROM quay.io/quarkus/quarkus-micro-image:2.0
COPY target/*-runner.jar /work/application.jar
CMD ["java", "-jar", "/work/application.jar"]

  2. Build & Run:

    ./mvnw package
docker build -f src/main/docker/Dockerfile.jvm -t panache-demo .
docker run -i --rm -p 8080:8080 --env-file .env panache-demo

    Wichtig: Stelle sicher, dass die Umgebungsvariablen für die DB-Verbindung gesetzt sind! (Entweder in .env oder direkt als Parameter.)

PostgreSQL per Docker bereitstellen (s. o.)

Jetzt kannst du die komplette Lösung (Quarkus + PostgreSQL) lokal oder in der Cloud laufen lassen.

Checkliste zum Üben und Vertiefen

  1. Lege ein neues Quarkus-Projekt mit Panache und PostgreSQL an.
  2. Implementiere eine REST-API mit CRUD-Operationen für eine Entität.
  3. Teste die API per Postman oder Curl.
  4. Erweitere die API um Suchen, Paging, Sortierung oder eigene Geschäftslogik.
  5. Containerisiere die Lösung mit Docker und teste, ob alles läuft.
  6. Optional: Schreibe Integrationstests mit Testcontainers.

Ideen für weitere Übungsaufgaben

  • Baue eine Produktverwaltung (mit Preis, Kategorie, Lagerbestand).
  • Implementiere Beziehungen (z. B. User → Orders).
  • Füge Validierung und Fehlerbehandlung hinzu.
  • Schreibe Integrationstests für die wichtigsten Endpunkte.
  • Optimiere das Setup für den Produktivbetrieb (keine "drop-and-create" Option!).
