Handout: CRUD-Operationen mit Panache

4.1 Einfaches Persistieren, Finden, Aktualisieren und Löschen von Entitäten

Mit Panache kannst du die typischen CRUD-Operationen (Create, Read, Update, Delete) besonders einfach und deklarativ durchführen – und das unabhängig davon, ob du das Active Record Pattern oder das Repository Pattern verwendest.

A) Persistieren (Create)

Active Record Pattern:

User user = new User();
user.name = "Max";
user.email = "max@example.com";
user.persist(); // Speichert das Objekt in der DB

Repository Pattern:

@Inject
UserRepository userRepository;

User user = new User();
user.name = "Max";
user.email = "max@example.com";
userRepository.persist(user); // Persistiert das Objekt

B) Finden (Read)

Finde per ID:

User found = User.findById(1L); // Active Record
// oder:
User found = userRepository.findById(1L); // Repository

Alle Entities auflisten:

List<User> all = User.listAll(); // Active Record
// oder:
List<User> all = userRepository.listAll(); // Repository

Suche mit Bedingungen (z. B. nach Name):

List<User> usersNamedMax = User.find("name", "Max").list(); // Active Record
// oder:
List<User> usersNamedMax = userRepository.find("name", "Max").list(); // Repository

C) Aktualisieren (Update)

Einfach die Felder ändern und persistieren:

User user = User.findById(1L);
user.email = "max.neu@example.com";
user.persist(); // Änderungen werden automatisch übernommen (managed Entity)

Tipp: In Panache wird die Änderung beim laufenden EntityManager automatisch übernommen. Bei Detach oder in neuen Transaktionen ggf. explizit persist() oder update() aufrufen.

D) Löschen (Delete)

User user = User.findById(1L);
user.delete(); // Active Record

// oder direkt per statischer Methode:
User.deleteById(1L);

// Oder via Repository:
userRepository.delete(user);
userRepository.deleteById(1L);

4.2 Komfortable Methoden wie find(), listAll(), delete(), etc.

Panache bringt viele nützliche Methoden bereits mit, z. B.:

Methode Beschreibung Beispiel
persist() Entity speichern (Insert) user.persist();
findById() Entity anhand der ID finden User.findById(1L);
listAll() Alle Entities als Liste User.listAll();
count() Anzahl der Entities User.count();
find(query, params...) Suche mit JPQL-ähnlicher Syntax User.find("name", "Max").list();
delete() Entity löschen user.delete();
deleteById() Entity anhand der ID löschen User.deleteById(1L);
findAll() Gibt einen Stream aller Entities (Paging/Slicing möglich) User.findAll().page(0, 10).list();

Praktische Zusatzfunktionen:

  • Paging:

    List<User> firstTen = User.findAll().page(Page.ofSize(10)).list();

  • Sortierung:

    List<User> sorted = User.list("ORDER BY name DESC");

4.3 Dynamische und statische Queries (Query DSL mit Panache)

Mit Panache kannst du Abfragen sehr flexibel gestalten:

A) Dynamische Queries

Syntax: find("Feld = ?1", wert).list();

  • Platzhalter mit ?1, ?2 usw.
  • Parameter werden als Argumente übergeben.

Beispiel:

List<User> users = User.find("name = ?1 AND email LIKE ?2", "Max", "%.com").list();

B) Statische Queries

Du kannst auch direkt nach Feldern suchen (Query by Example):

User.find("name", "Max").list();

Hier wird automatisch nach dem Feld name gesucht.

C) Native Queries

Du kannst auch nativen SQL-Code nutzen:

User.find("SELECT * FROM user WHERE email LIKE ?1", Sort.by("name"), "%@example.com").list();

Oder mit dem Repository:

userRepository.find("email LIKE ?1", "%@example.com").list();

D) Query mit Sortierung und Paging

Sortierung:

List<User> sorted = User.find("name = ?1", Sort.by("email").descending(), "Max").list();

Paging:

List<User> paged = User.findAll().page(Page.ofSize(10)).list();

Zusammenfassung – CRUD mit Panache

  • Mit Panache erledigst du die wichtigsten Datenbankoperationen extrem einfach – egal, ob du Active Record oder Repository Pattern nutzt.
  • Standardoperationen (persist, find, delete, listAll) sind direkt verfügbar.
  • Abfragen (Queries) werden mit klarer, deklarativer Syntax formuliert – inkl. Platzhalter, Sortierung, Paging und nativen Queries.

Praxis-Checkliste:

  1. Entity anlegen, von PanacheEntity oder PanacheEntityBase ableiten.
  2. CRUD-Operationen mit Methoden wie persist(), findById(), delete() direkt nutzen.
  3. Abfragen mit .find() und Parametern schreiben.
  4. Bei komplexeren Szenarien Sortierung und Paging nutzen.
  5. Repository-Pattern einsetzen, wenn du saubere Trennung oder Testbarkeit brauchst.
