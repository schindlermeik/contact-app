Handout: Voraussetzungen & Projekt-Setup – Panache mit Quarkus und PostgreSQL

2.1 Integration von Panache in ein Maven-Projekt

Panache ist eine Erweiterung von Quarkus und setzt auf Hibernate ORM auf. Die Einbindung erfolgt komfortabel über Maven – das verbreitetste Build- und Dependency-Management-System für Java.

Vorgehensweise

  1. Quarkus-Projekt erstellen: Entweder manuell oder mit dem Quarkus Initializer.

  2. Benötigte Extensions im Maven-Projekt eintragen: Abhängigkeiten werden im Abschnitt <dependencies> in der pom.xml gepflegt.

Projektstruktur (Auszug)

mein-panache-projekt/
├── src/
│   ├── main/
│   │   ├── java/           // Quellcode
│   │   └── resources/
│   │        └── application.properties
└── pom.xml                 // Maven-Konfigurationsdatei

2.2 Notwendige Abhängigkeiten für Panache

Um mit Panache, Hibernate ORM und PostgreSQL arbeiten zu können, werden folgende Maven-Dependencies benötigt:

Beispiel-Abschnitt für die pom.xml:

<dependencies>
    <!-- Quarkus BOM für die Versionsverwaltung -->
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>io.quarkus</groupId>
                <artifactId>quarkus-bom</artifactId>
                <version>3.8.3</version> <!-- Aktuelle Version prüfen! -->
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>
    
    <!-- Quarkus Hibernate ORM mit Panache -->
    <dependency>
        <groupId>io.quarkus</groupId>
        <artifactId>quarkus-hibernate-orm-panache</artifactId>
    </dependency>
    
    <!-- Quarkus JDBC-Treiber für PostgreSQL -->
    <dependency>
        <groupId>io.quarkus</groupId>
        <artifactId>quarkus-jdbc-postgresql</artifactId>
    </dependency>
    
    <!-- Quarkus RESTEasy für REST-APIs (optional, aber meist gewünscht) -->
    <dependency>
        <groupId>io.quarkus</groupId>
        <artifactId>quarkus-resteasy</artifactId>
    </dependency>
</dependencies>

Hinweise:

  • Versionsnummer: Am besten nutzt du das Quarkus-BOM (quarkus-bom) im <dependencyManagement>, dann entfallen explizite Versionsangaben bei den einzelnen Quarkus-Abhängigkeiten.
  • Weitere Extensions: Je nach Projekt können weitere Extensions hinzugefügt werden (z. B. quarkus-resteasy-jackson für JSON-Support).
  • Dependency-Updates: Prüfe aktuelle Versionen und kompatible Erweiterungen im Quarkus Extension Catalog.

2.3 Datenbankanbindung und Konfiguration (Beispiel: PostgreSQL)

Datenbank vorbereiten

  • Installiere PostgreSQL lokal (z. B. per offizieller Downloadseite) oder nutze einen bestehenden Server.

  • Lege eine Datenbank und einen Benutzer mit Passwort an:

    CREATE DATABASE panache_demo;
CREATE USER panache_user WITH ENCRYPTED PASSWORD 'geheimespasswort';
GRANT ALL PRIVILEGES ON DATABASE panache_demo TO panache_user;

Konfiguration in Quarkus

Die zentrale Konfigurationsdatei bei Quarkus ist die src/main/resources/application.properties.

Beispielkonfiguration für PostgreSQL mit Panache:

# JDBC-URL zur Datenbank (Host, Port, DB-Name)
quarkus.datasource.db-kind=postgresql
quarkus.datasource.username=panache_user
quarkus.datasource.password=geheimespasswort
quarkus.datasource.jdbc.url=jdbc:postgresql://localhost:5432/panache_demo

# Hibernate DDL-Strategie (z. B. drop-and-create, update, validate)
quarkus.hibernate-orm.database.generation=drop-and-create

# Panache-Spezifisch: Optional Logging für SQL-Befehle
quarkus.hibernate-orm.log.sql=true

Hinweise:

  • Die Einstellungen sollten nicht für Produktionsdatenbanken verwendet werden (drop-and-create löscht die Tabellen bei jedem Start!).
  • Für die Entwicklung ist das automatische Schema-Management praktisch, kann aber zu Datenverlust führen, wenn nicht vorsichtig eingesetzt.
  • Die Zugangsdaten können auch über Umgebungsvariablen oder externe Konfigurationsmechanismen gesetzt werden (siehe Quarkus Configuration Reference).

(Optional) Docker-Setup für PostgreSQL

Du kannst eine lokale PostgreSQL-Datenbank auch schnell mit Docker starten:

docker run --name postgres-panache -e POSTGRES_DB=panache_demo \
    -e POSTGRES_USER=panache_user -e POSTGRES_PASSWORD=geheimespasswort \
    -p 5432:5432 -d postgres:16

Damit hast du sofort eine lauffähige Datenbank für Quarkus & Panache zur Verfügung.

Kurze Schritt-für-Schritt-Checkliste

  1. Neues Maven-Projekt anlegen (z. B. per Quarkus Initializer).
  2. Notwendige Dependencies (quarkus-hibernate-orm-panache, quarkus-jdbc-postgresql) ergänzen.
  3. PostgreSQL-Datenbank & Benutzer vorbereiten (lokal oder Docker).
  4. application.properties passend konfigurieren.
  5. Projekt bauen und ersten Test starten (./mvnw quarkus:dev).
