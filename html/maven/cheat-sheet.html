Maven Build-Lifecycle Cheat Sheet

Standard-Lifecycles

Lifecycle Zweck
default Kompilieren, Testen, Paketieren und Installieren von Software
clean Aufräumen (löscht Build-Ergebnisse, z. B. target)
site Erzeugt Projekt-Dokumentation und Berichte

Phasen im Default-Lifecycle (wichtigste)

Phase Zweck
validate Prüft, ob das Projekt korrekt ist (alle nötigen Infos da)
compile Kompiliert den Quellcode (src/main/java → target/classes)
test Führt Tests aus (src/test/java), ohne Paket zu bauen
package Erstellt ein Paket (z. B. JAR, WAR) im target-Verzeichnis
verify Zusätzliche Prüfungen (z. B. Integrationstests, Checks)
install Installiert das Artefakt ins lokale Maven-Repository
deploy Veröffentlicht das Artefakt im entfernten Repository

Wichtige Befehle (CLI)

mvn clean           # Löscht Build-Ergebnisse
mvn validate        # Prüft Projektstruktur
mvn compile         # Kompiliert Quellcode
mvn test            # Führt Unit-Tests aus
mvn package         # Erstellt z. B. eine JAR-Datei
mvn verify          # Fährt zusätzliche Prüfungen (wenn konfiguriert)
mvn install         # Installiert Artefakt lokal (~/.m2/repository)
mvn deploy          # Lädt Artefakt auf zentrales Repo (falls konfiguriert)

Was passiert, wenn du eine Phase aufrufst?

  • Alle vorhergehenden Phasen werden automatisch mit ausgeführt!
  • Beispiel: mvn package → läuft durch validate, compile, test, und package.

Unterschied: Phase vs. Goal

  • Phase: Logischer Schritt im Build-Prozess (compile, test, …).
  • Goal: Konkrete Aufgabe (z. B. compiler:compile, surefire:test), meist durch Plugins ausgeführt.

Praktische Optionen

  • Tests überspringen: mvn package -DskipTests
  • Build im Debug-Modus (mehr Logs): mvn compile -X
  • Nur ein bestimmtes Plugin ausführen: mvn compiler:compile (führt das Compile-Goal des compiler-Plugins aus, unabhängig von den Phasen)

Weitere Tipps

  • Phasen und Plugins anzeigen: mvn help:describe -Dcmd=phase
  • Plugin-Goals nachlesen: mvn help:describe -Dplugin=PLUGINNAME

Merksatz: „Maven baut von validate bis deploy – alles dazwischen läuft automatisch!“
