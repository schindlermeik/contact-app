Handout: Maven Build-Lifecycle – Der Build-Prozess

5.1 Was ist ein Lifecycle?

Maven steuert den kompletten Build-Prozess eines Projekts mit dem Konzept des „Build-Lifecycle“ (Lebenszyklus). Ein Lifecycle ist eine vordefinierte Abfolge von Schritten, die beim Bauen eines Projekts automatisch in einer bestimmten Reihenfolge durchlaufen werden.

Maven kennt drei Standard-Lifecycles:

  • default (auch: build): Der Haupt-Lifecycle für das Erzeugen von Artefakten (z. B. JAR, WAR).
  • clean: Für das Aufräumen des Projekts (z. B. Löschen des target-Ordners).
  • site: Für das Erstellen von Webseiten und Berichten zum Projekt.

Merke: Wenn du einen bestimmten Befehl wie mvn package oder mvn test eingibst, startet Maven den entsprechenden Lifecycle und durchläuft automatisch alle nötigen Phasen bis zu diesem Schritt!

5.2 Die wichtigsten Phasen im Standard-Lifecycle

Ein Lifecycle besteht aus Phasen. Die wichtigsten Phasen des „default“-Lifecycles sind:

Phase Bedeutung
validate Prüft, ob das Projekt korrekt aufgebaut und vollständig ist
compile Übersetzt den Quellcode (src/main/java) in Bytecode (target/classes)
test Führt automatisierte Tests im Testverzeichnis (src/test/java) aus
package Packt das kompilierte Projekt in ein Artefakt (z. B. JAR, WAR)
verify Führt ggf. zusätzliche Prüfungen durch (z. B. Integrations-Tests)
install Installiert das Artefakt im lokalen Maven-Repository (für andere Projekte auf deinem Rechner verfügbar)
deploy Lädt das fertige Artefakt in ein zentrales Repository (z. B. Nexus/Artifactory)

Weitere Lifecycles/Phasen:

  • clean:

    • pre-clean
    • clean (löscht target)
    • post-clean

  • site:

    • pre-site
    • site (erstellt Projektseite/Reports)
    • post-site
    • site-deploy (veröffentlicht die Seite)

5.3 Unterschied: Phase vs. Goal

Was ist eine Phase?

  • Ein logischer Schritt im Build-Prozess, z. B. compile oder test.

Was ist ein Goal?

  • Ein Kommando oder eine Aufgabe, die von einem Maven-Plugin ausgeführt wird, z. B. compiler:compile oder surefire:test.
  • Jedes Goal kann einer Phase zugeordnet sein.

Beispiel:

  • Die Phase test ruft das Goal surefire:test (vom Surefire-Plugin) auf.

Was passiert, wenn du mvn test oder mvn package eingibst?

Beispiel 1: mvn test

  • Maven führt alle Phasen von Anfang bis einschließlich test aus:

    • validate → compile → test

Beispiel 2: mvn package

  • Maven führt alle Phasen von Anfang bis einschließlich package aus:

    • validate → compile → test → package

Wichtig:

  • Alle vorangehenden Phasen werden immer mitausgeführt!
  • Wenn du z. B. mvn package aufrufst, wird immer auch kompiliert und getestet (außer du schaltest Tests explizit aus).

Zusammenfassung

  • Der Maven Build-Lifecycle sorgt für einen klaren, reproduzierbaren Ablauf beim Bauen eines Projekts.
  • Phasen und Goals sind die zentralen Bausteine.
  • Mit einfachen Befehlen steuerst du, bis zu welchem Schritt Maven arbeiten soll – alle nötigen Schritte davor werden automatisch erledigt.
  • Mit gezielten Konsolenbefehlen lernst du, die Abläufe besser zu verstehen und Probleme gezielt zu erkennen.

Tipp: Wenn du dir den Ablauf grafisch darstellen lassen möchtest, probiere z. B. den Befehl

mvn help:describe -Dcmd=package

aus – oder schau in die offizielle Maven-Lifecycle-Doku.
