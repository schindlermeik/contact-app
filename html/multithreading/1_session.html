Handout: Einführung in Nebenläufigkeit in Java

1.1 Was bedeutet Nebenläufigkeit in Java?

Nebenläufigkeit (engl. Concurrency) bezeichnet die Fähigkeit eines Programms, mehrere Aufgaben (Threads) so auszuführen, dass sie sich überschneiden oder gleichzeitig erscheinen – auch wenn nicht unbedingt echte Parallelität vorliegt.

In Java bedeutet Nebenläufigkeit:

  • Mehrere Aufgaben (Threads) können innerhalb eines Programms unabhängig voneinander laufen.
  • Das Java-Runtime-System (JVM) verwaltet die Ausführung und den Wechsel zwischen den Threads (sog. Scheduling).
  • Nebenläufigkeit kann auf einer einzelnen CPU (durch schnelles Umschalten – Multitasking) oder auf mehreren CPUs/Kernen (echte Parallelität) stattfinden.

Beispiel: Ein Webserver in Java kann mehrere Anfragen von Benutzern gleichzeitig bearbeiten. Für jede Anfrage wird ein eigener Thread erzeugt.

1.2 Abgrenzung: Nebenläufigkeit (Concurrency) vs. Parallelität (Parallelism) in Java

Nebenläufigkeit (Concurrency)

  • Bedeutet, dass mehrere Aufgaben logisch „gleichzeitig“ ablaufen.
  • Die JVM teilt der CPU mit, wann welcher Thread an der Reihe ist (häufig durch schnelles Umschalten, sog. Interleaving).
  • Kann auf einem Einzelkern-Prozessor realisiert werden.
  • Beispiel: Mehrere Nutzer bearbeiten in einer Java-Desktopanwendung gleichzeitig unterschiedliche Dokumente. Der Wechsel zwischen Aufgaben ist so schnell, dass alle Nutzer das Gefühl haben, ihre Aufgabe läuft „gleichzeitig“.

Parallelität (Parallelism)

  • Mehrere Aufgaben werden tatsächlich zur gleichen Zeit auf verschiedenen Prozessorkernen ausgeführt.
  • Seit Java 1.5 unterstützt die JVM echte Parallelverarbeitung, wenn das Betriebssystem und die Hardware mehrere Kerne bereitstellen.
  • Beispiel: In einer Datenanalyse-Anwendung werden große Datenmengen auf verschiedene Kerne verteilt und parallel berechnet.

Kurz gesagt:

  • Nebenläufigkeit = Gleichzeitigkeit in der Programmstruktur, nicht zwingend in der Hardware.
  • Parallelität = Tatsächliche Gleichzeitigkeit durch parallele Ausführung auf mehreren Kernen.

1.3 Typische Anwendungsfälle für Multithreading in Java

a) Benutzeroberflächen (GUI)

  • Reaktionsfähigkeit sichern: Während ein Java-Programm (z. B. mit Swing oder JavaFX) rechnet, soll die Benutzeroberfläche nicht „einfrieren“.
  • Lösung: Hintergrund-Threads für lange Berechnungen.

b) Serveranwendungen

  • Ein Server (z. B. Webserver, Chatserver) muss viele Anfragen gleichzeitig verarbeiten.
  • Lösung: Für jede eingehende Verbindung wird ein neuer Thread (oder ein Thread aus einem Pool) verwendet.

c) Datenverarbeitung und Berechnungen

  • Große Datenmengen oder komplexe Berechnungen können auf mehrere Threads aufgeteilt und parallel ausgeführt werden (z. B. Bildverarbeitung, Simulationen).

d) Asynchrone Aufgaben

  • Netzwerkkommunikation, Dateioperationen, zeitgesteuerte Abläufe – Threads ermöglichen, dass das Hauptprogramm weiterläuft, während andere Aufgaben im Hintergrund erledigt werden.

1.4 Herausforderungen: Race Conditions, Deadlocks, Visibility-Probleme in Java

a) Race Conditions (Wettlaufsituationen)

  • Tritt auf, wenn zwei oder mehr Threads gleichzeitig auf dieselbe Ressource zugreifen und mindestens einer sie verändert.

  • Kann zu unerwarteten und schwer zu findenden Fehlern führen.

  • Beispiel:

    // Zwei Threads inkrementieren denselben Wert:
class Counter {
    private int count = 0;
    public void increment() {
        count = count + 1; // Nicht thread-sicher!
    }
}

  • Lösung: Synchronisation (späteres Thema).

b) Deadlocks (Verklemmungen)

  • Entstehen, wenn zwei oder mehr Threads sich gegenseitig blockieren, indem sie jeweils auf eine Ressource warten, die vom anderen gehalten wird.
  • Beispiel: Thread A hält Lock 1 und wartet auf Lock 2, Thread B hält Lock 2 und wartet auf Lock 1 – keiner kann weiterarbeiten.
  • Sehr schwer zu debuggen und zu vermeiden ohne gute Architektur.

c) Visibility-Probleme (Sichtbarkeitsprobleme)

  • Änderungen eines Threads an gemeinsamen Variablen werden von anderen Threads eventuell nicht oder verspätet gesehen.
  • Ursache: Caches, CPU-Optimierungen oder JVM können verhindern, dass Werte sofort für alle Threads sichtbar sind.
  • Lösung: Verwendung von volatile, Synchronisation oder speziellen Datenstrukturen (AtomicInteger, ConcurrentHashMap etc.).

Zusammenfassung

  • Nebenläufigkeit ist ein zentraler Bestandteil moderner Java-Anwendungen.
  • Sie ermöglicht reaktionsfähige Programme und effiziente Ausnutzung von Mehrkern-Prozessoren.
  • Sie bringt jedoch neue Herausforderungen, wie Race Conditions, Deadlocks und Visibility-Probleme, die eine gute Planung und den Einsatz geeigneter Java-Techniken erfordern.
