Handout: Hands-On – Übungen und Beispiele in Java (Multithreading)

8.1 Schreibe einen einfachen Multi-Threaded Counter mit Synchronisation

Ziel: Mehrere Threads sollen denselben Zähler inkrementieren, ohne dass Daten verloren gehen.

Schritt 1: Unsicherer Counter (ohne Synchronisation)

class Counter {
    private int count = 0;
    public void increment() {
        count++;
    }
    public int getCount() {
        return count;
    }
}

public class CounterTest {
    public static void main(String[] args) throws InterruptedException {
        Counter counter = new Counter();

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);
        t1.start(); t2.start();
        t1.join(); t2.join();

        System.out.println("Endstand (ohne Synchronisation): " + counter.getCount());
    }
}

Ergebnis: Meistens weniger als 2000 → Race Condition!

Schritt 2: Sicherer Counter (mit Synchronisation)

class SafeCounter {
    private int count = 0;
    public synchronized void increment() {
        count++;
    }
    public int getCount() {
        return count;
    }
}

// Ersetze Counter durch SafeCounter in obigem Test.

Jetzt: Immer genau 2000.

8.2 Simuliere ein Race Condition-Problem und behebe es durch Synchronisation

Beispielproblem: Zwei Threads heben gleichzeitig von einem gemeinsamen Bankkonto ab.

class BankAccount {
    private int balance = 1000;
    public void withdraw(int amount) {
        if (balance >= amount) {
            balance -= amount;
        }
    }
    public int getBalance() {
        return balance;
    }
}

public class BankTest {
    public static void main(String[] args) throws InterruptedException {
        BankAccount account = new BankAccount();

        Runnable task = () -> {
            for (int i = 0; i < 1000; i++) {
                account.withdraw(1);
            }
        };

        Thread t1 = new Thread(task);
        Thread t2 = new Thread(task);
        t1.start(); t2.start();
        t1.join(); t2.join();

        System.out.println("Endstand (ohne Synchronisation): " + account.getBalance());
    }
}

Ergebnis: Oft negativer Kontostand oder mehr als null übrig → Race Condition.

Lösung:

class SafeBankAccount {
    private int balance = 1000;
    public synchronized void withdraw(int amount) {
        if (balance >= amount) {
            balance -= amount;
        }
    }
    public int getBalance() {
        return balance;
    }
}

Jetzt: Niemals negativer Kontostand.

8.3 Setze einen Producer-Consumer mit wait()/notify() um

Problem: Ein Producer erzeugt Daten und legt sie in einen Puffer, ein Consumer entnimmt sie – beide laufen in eigenen Threads.

class Buffer {
    private int value;
    private boolean available = false;

    public synchronized void put(int v) throws InterruptedException {
        while (available) wait();
        value = v;
        available = true;
        notifyAll();
    }

    public synchronized int get() throws InterruptedException {
        while (!available) wait();
        available = false;
        notifyAll();
        return value;
    }
}

public class ProducerConsumerDemo {
    public static void main(String[] args) {
        Buffer buffer = new Buffer();

        // Producer
        Thread producer = new Thread(() -> {
            try {
                for (int i = 0; i < 5; i++) {
                    buffer.put(i);
                    System.out.println("Produced: " + i);
                }
            } catch (InterruptedException ignored) {}
        });

        // Consumer
        Thread consumer = new Thread(() -> {
            try {
                for (int i = 0; i < 5; i++) {
                    int val = buffer.get();
                    System.out.println("Consumed: " + val);
                }
            } catch (InterruptedException ignored) {}
        });

        producer.start();
        consumer.start();
    }
}
  • Producer und Consumer greifen sicher synchronisiert auf den Puffer zu.

8.4 Verwende einen Thread Pool (ExecutorService) für parallele Aufgabenbearbeitung

Beispiel: 10 Aufgaben werden parallel in einem Thread Pool bearbeitet

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadPoolExample {
    public static void main(String[] args) {
        ExecutorService pool = Executors.newFixedThreadPool(4); // 4 Threads

        for (int i = 0; i < 10; i++) {
            int taskId = i;
            pool.submit(() -> {
                System.out.println("Task " + taskId + " läuft im Thread " + Thread.currentThread().getName());
                try { Thread.sleep(500); } catch (InterruptedException ignored) {}
            });
        }

        pool.shutdown(); // Nach Aufgabenende sauber beenden
    }
}
  • Aufgaben werden automatisch auf Threads im Pool verteilt.

8.5 Optional: Implementiere einen Deadlock und löse ihn

Beispiel:

class DeadlockDemo {
    private final Object lock1 = new Object();
    private final Object lock2 = new Object();

    public void methodA() {
        synchronized (lock1) {
            try { Thread.sleep(50); } catch (InterruptedException ignored) {}
            synchronized (lock2) {
                System.out.println("Method A fertig");
            }
        }
    }

    public void methodB() {
        synchronized (lock2) {
            try { Thread.sleep(50); } catch (InterruptedException ignored) {}
            synchronized (lock1) {
                System.out.println("Method B fertig");
            }
        }
    }

    public static void main(String[] args) {
        DeadlockDemo demo = new DeadlockDemo();
        Thread t1 = new Thread(demo::methodA);
        Thread t2 = new Thread(demo::methodB);
        t1.start(); t2.start();
    }
}

Ergebnis:

  • Beide Threads blockieren sich gegenseitig – das Programm hängt.

Lösung:

  • Immer gleiche Reihenfolge der Locks erzwingen (z.B. beide Methoden: erst lock1, dann lock2).
