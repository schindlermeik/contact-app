Handout: Threads in Java – Grundlagen

2.1 Was ist ein Thread? Unterschied Prozess vs. Thread in Java

Prozess:

  • Ein Prozess ist ein eigenständiges, laufendes Programm mit eigenem Speicherbereich, Ressourcen und Betriebssystem-Kontext.
  • Mehrere Prozesse können unabhängig voneinander laufen (z.B. dein Browser und dein Texteditor gleichzeitig).

Thread:

  • Ein Thread (dt. „Faden“) ist ein „leichter“ Ausführungsstrang innerhalb eines Prozesses.
  • Alle Threads eines Prozesses teilen sich Speicher, aber jeder Thread hat einen eigenen Ausführungskontext (Callstack, Register).

In Java:

  • Eine Java-Anwendung ist mindestens ein Prozess (die JVM).
  • In einer JVM kann es viele Threads geben, z.B. den „main“-Thread, Garbage Collector Thread und beliebig viele selbst erstellte Threads.
  • Threads ermöglichen parallele/gleichzeitige Bearbeitung mehrerer Aufgaben innerhalb eines Programms.

Vorteile von Threads:

  • Bessere Ausnutzung von Mehrkern-CPUs.
  • Reaktionsfähige Benutzeroberflächen (UI friert nicht ein).
  • Gleichzeitige Bearbeitung von Aufgaben (z.B. Netzwerk, Dateioperationen).

2.2 Threads erzeugen

2.2.1 Mit der Klasse Thread (Vererbung)

  • Man erstellt eine Klasse, die von Thread erbt, und überschreibt die run()-Methode:
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread läuft!");
    }
}

// Verwendung:
MyThread t = new MyThread();
t.start(); // nie run() direkt aufrufen!

Hinweis: start() erzeugt einen neuen Thread, der dann run() ausführt. Wird nur run() aufgerufen, läuft der Code nicht in einem neuen Thread!

2.2.2 Mit dem Interface Runnable (Implementierung)

  • Man erstellt eine Klasse, die das Interface Runnable implementiert, und übergibt ein Objekt davon an einen neuen Thread.
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable läuft!");
    }
}

// Verwendung:
Thread t = new Thread(new MyRunnable());
t.start();

Vorteil:

  • Die Klasse kann von einer anderen Klasse erben, da Java nur Einfachvererbung unterstützt.

2.2.3 Mit Lambda-Ausdrücken (ab Java 8)

  • Da Runnable ein „Functional Interface“ ist, kann man ab Java 8 Lambdas verwenden:
Thread t = new Thread(() -> {
    System.out.println("Thread per Lambda läuft!");
});
t.start();
  • Praktisch für kurze Aufgaben, ohne extra Klasse zu schreiben.

2.3 Lebenszyklus eines Threads in Java

Ein Thread in Java durchläuft verschiedene Zustände, die im Enum Thread.State definiert sind:

  1. NEW:

    • Der Thread wurde erzeugt (Objekt ist vorhanden), aber start() wurde noch nicht aufgerufen.

  2. RUNNABLE:

    • Der Thread ist bereit zur Ausführung oder läuft gerade (JVM und Betriebssystem entscheiden, wann er CPU-Zeit bekommt).

  3. BLOCKED:

    • Der Thread wartet auf einen Monitor-Lock, um einen synchronisierten Block/Methoden betreten zu können.

  4. WAITING:

    • Der Thread wartet auf eine andere Aktion (z. B. Object.wait(), Thread.join()).

  5. TIMED_WAITING:

    • Wie WAITING, aber mit Zeitlimit (z. B. Thread.sleep(1000), wait(1000)).

  6. TERMINATED:

    • Der Thread hat seine run()-Methode beendet.

NEW → RUNNABLE ↔ BLOCKED/WAITING/TIMED_WAITING → TERMINATED

2.4 Wichtige Thread-Methoden

  • start() Startet einen neuen Thread und ruft im neuen Thread die run()-Methode auf.

  • run() Enthält den Code, der im Thread ausgeführt wird. (Nicht direkt aufrufen, sondern immer mit start()!)

  • sleep(long millis) Lässt den aktuellen Thread für die angegebene Zeit pausieren. Beispiel:

    Thread.sleep(1000); // schläft 1 Sekunde

    Kann eine InterruptedException werfen.

  • join() Wartet darauf, dass ein anderer Thread beendet ist. Beispiel:

    t.join(); // Der aktuelle Thread wartet, bis t fertig ist

  • interrupt() Setzt das „Interrupt-Flag“ eines Threads. Kann genutzt werden, um einen Thread freundlich zu bitten, sich selbst zu beenden.

  • isAlive() Gibt true zurück, solange der Thread noch läuft (noch nicht TERMINATED ist).

Zusammenfassung

  • In Java sind Threads leichte Prozesse innerhalb einer JVM, die gleichzeitig Aufgaben ausführen können.
  • Threads können durch Vererbung von Thread, Implementieren von Runnable oder durch Lambdas (ab Java 8) erstellt werden.
  • Jeder Thread durchläuft mehrere Lebenszyklus-Zustände, die für korrekte Synchronisation und Steuerung wichtig sind.
  • Methoden wie start(), join(), sleep() und interrupt() sind essenziell für das Thread-Management.
