Handout: Thread-Sicherheit und sichere Datenstrukturen in Java

5.1 Was ist Thread-Sicherheit?

Definition: Ein Codeabschnitt, eine Methode oder eine Datenstruktur ist thread-sicher, wenn sie auch bei Zugriffen durch mehrere Threads gleichzeitig stets konsistente, korrekte Ergebnisse liefert und keine Race Conditions auftreten.

Wichtig:

  • Thread-Sicherheit bezieht sich darauf, dass keine Daten verloren gehen, keine inkonsistenten Zustände entstehen und keine unvorhersehbaren Ausnahmen auftreten, selbst wenn viele Threads auf dieselbe Ressource zugreifen.

Merkmale thread-sicheren Codes:

  • Schutz vor Race Conditions (durch Synchronisation, Locks oder atomare Operationen)
  • Korrekte Sichtbarkeit von Änderungen zwischen Threads (Visibility)
  • Keine unvorhergesehenen Ausnahmen aufgrund konkurrierenden Zugriffs

5.2 Nicht-thread-sichere vs. thread-sichere Implementierungen

Nicht-thread-sichere Datenstrukturen

  • ArrayList, HashMap, LinkedList, usw.
  • Diese Klassen sind NICHT für gleichzeitigen Zugriff durch mehrere Threads gedacht.

Beispielproblem:

List<Integer> list = new ArrayList<>();
// Zwei Threads fügen gleichzeitig Elemente hinzu → Inkonsistenzen!
  • Mögliche Folgen: Datenverluste, ConcurrentModificationException, beschädigte interne Zustände.

Thread-sichere Datenstrukturen

  • Historisch:

    • Vector und Hashtable – synchronisierte Methoden, aber langsam, heute meist nicht mehr empfohlen.

  • Moderne Varianten:

    • CopyOnWriteArrayList, ConcurrentHashMap, ConcurrentLinkedQueue, etc. aus dem Paket java.util.concurrent.

Beispiel:

List<Integer> safeList = new CopyOnWriteArrayList<>();
Map<String, String> safeMap = new ConcurrentHashMap<>();
  • Diese Klassen sind für gleichzeitigen Zugriff optimiert.

5.3 Verwendung von synchronisierten Collections

a) Synchronisierte Wrapper

  • Man kann eine nicht-thread-sichere Collection mit einem synchronisierten Wrapper versehen:
List<Integer> syncList = Collections.synchronizedList(new ArrayList<>());
Map<String, String> syncMap = Collections.synchronizedMap(new HashMap<>());
  • Jeder Zugriff auf die Collection wird dann durch einen gemeinsamen Lock (Monitor) geschützt.

ACHTUNG:

  • Iteratoren auf synchronisierten Collections sind nicht automatisch thread-sicher! Beim Iterieren muss man selbst synchronisieren:
synchronized(syncList) {
    for (Integer i : syncList) {
        // ...
    }
}

b) Moderne, spezialisierte Datenstrukturen

  • ConcurrentHashMap – Für hohe Performance bei vielen Lese-/Schreibzugriffen von mehreren Threads.
  • CopyOnWriteArrayList – Für Listen mit vielen Lesezugriffen und seltenen Änderungen.
  • BlockingQueue – Für Producer/Consumer-Patterns.

Vorteil:

  • Keine manuelle Synchronisation nötig, viele interne Optimierungen.

5.4 Immutable Objects als Strategie für Nebenläufigkeit

Was sind Immutable Objects?

  • Objekte, deren Zustand nach der Erstellung nicht mehr geändert werden kann (z.B. String, eigene Klassen mit final-Feldern und ohne Setter).
  • Alle Felder sind final und werden nur im Konstruktor gesetzt.

Vorteil für Thread-Sicherheit:

  • Immutable Objects können gefahrlos von beliebig vielen Threads gleichzeitig gelesen und verwendet werden – keine Synchronisation nötig!

Beispiel:

public final class Point {
    private final int x, y;
    public Point(int x, int y) {
        this.x = x; this.y = y;
    }
    public int getX() { return x; }
    public int getY() { return y; }
}
  • Auch Sammlungen können (z.B. mit List.of(...) ab Java 9) als immutable erstellt werden.

5.5 Praktische Beispiele für Thread-sicheren Code

a) Verwendung synchronisierter Collections

List<String> list = Collections.synchronizedList(new ArrayList<>());
list.add("A");
list.add("B");
// In mehreren Threads: kein Datenverlust, kein Absturz

b) Verwendung von ConcurrentHashMap

Map<String, Integer> map = new ConcurrentHashMap<>();
map.put("Apfel", 3);
map.put("Birne", 5);
// Viele Threads können gleichzeitig lesen und schreiben

c) Immutable Object

public final class Configuration {
    private final String server;
    private final int port;
    public Configuration(String server, int port) {
        this.server = server; this.port = port;
    }
    public String getServer() { return server; }
    public int getPort() { return port; }
}

d) CopyOnWriteArrayList für häufiges Lesen

List<String> userList = new CopyOnWriteArrayList<>();
// Optimal, wenn viele Threads lesen und nur selten geschrieben wird

e) Mit Synchronisation

class SafeCounter {
    private int count = 0;
    public synchronized void increment() { count++; }
    public synchronized int getCount() { return count; }
}

Zusammenfassung

  • Thread-Sicherheit ist die Basis für stabile und korrekte Multithread-Programme in Java.
  • Nutze moderne, thread-sichere Datenstrukturen wie ConcurrentHashMap und CopyOnWriteArrayList wann immer möglich.
  • Immutable Objects eliminieren viele Nebenläufigkeitsprobleme auf elegante Weise.
  • Vermeide manuelle Synchronisation, wenn es spezialisierte Datenstrukturen gibt – sie sind meist schneller und weniger fehleranfällig.
