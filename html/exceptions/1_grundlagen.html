2.1 Grundlagen: Was sind Exceptions?

2.1.1 Motivation und Problemstellung

Was passiert, wenn beim Programmieren Fehler auftreten?

In jedem Programm können Fehler oder unerwartete Situationen auftreten. Zum Beispiel:

  • Eine Datei, auf die du zugreifen möchtest, existiert nicht.
  • Der Nutzer gibt falsche Eingaben ein.
  • Eine Netzwerkverbindung bricht ab.

Bisherige (alte) Fehlerbehandlung: Fehlercodes

Vor der Einführung von Exceptions mussten Fehler oft mit speziellen Rückgabewerten oder Fehlercodes behandelt werden:

int result = openFile("data.txt");
if (result == -1) {
    // Fehlerbehandlung
}

Probleme dieser Methode:

  • Fehler werden leicht übersehen, wenn der Rückgabewert nicht geprüft wird.
  • Der eigentliche Programmfluss wird durch ständige Fehlerüberprüfungen unübersichtlich.
  • Es ist schwer, verschiedene Fehlerarten zu unterscheiden.

Warum reicht das nicht aus?

  • Fehlerbehandlung wird „versteckt“ oder vergessen.
  • Rückgabewerte sagen nichts über die Ursache des Fehlers aus.
  • Wartung und Erweiterung des Codes wird schwierig.

2.1.2 Exception als Lösung

Was ist eine Exception?

Eine Exception (deutsch: Ausnahme) ist ein objektorientiertes Konzept in Java (und vielen anderen Sprachen), um Fehler oder unerwartete Situationen klar und strukturiert darzustellen.

  • Exceptions sind Objekte, die Informationen über einen Fehler enthalten (z.B. Art des Fehlers, Ursache, zusätzliche Details).
  • Exceptions unterbrechen den normalen Programmfluss: Wenn im Programm ein Fehler auftritt, „springt“ die Ausführung zu einer Fehlerbehandlung – sie läuft also nicht einfach weiter, als sei nichts passiert.

Vorteile von Exceptions:

  • Zentrale Fehlerbehandlung: Fehler werden an einer Stelle im Code gesammelt behandelt, nicht überall verstreut.
  • Typisierung: Verschiedene Fehlerarten können durch unterschiedliche Exception-Klassen dargestellt werden (z.B. IOException, NullPointerException).
  • Sauberer Code: Der Haupt-Programmfluss wird nicht durch ständige Fehlerüberprüfungen unterbrochen.
  • Automatische Weitergabe: Exceptions können automatisch an die nächste Ebene (z.B. eine höhere Methode) weitergegeben werden.

Beispiel: Datei öffnen mit Exception

try {
    FileReader reader = new FileReader("data.txt");
    // Datei wird gelesen
} catch (FileNotFoundException e) {
    System.out.println("Datei nicht gefunden: " + e.getMessage());
}

2.1.3 Syntax & Ablauf: try-catch-finally

Das Grundprinzip

In Java werden Fehler (Exceptions) mit dem sogenannten try-catch-finally-Block behandelt:

try {
    // Code, der eine Exception auslösen kann
} catch (ExceptionTyp e) {
    // Fehlerbehandlung, wenn eine Exception auftritt
} finally {
    // Code, der immer ausgeführt wird, egal ob ein Fehler aufgetreten ist oder nicht
}

Die Bestandteile im Detail

1. try-Block

  • Enthält den Code, bei dem Fehler auftreten könnten.
  • Sobald eine Exception im try-Block ausgelöst wird, wird der Rest des Blocks übersprungen und die Ausführung springt in den passenden catch-Block.

2. catch-Block

  • Hier wird festgelegt, wie auf eine bestimmte Exception-Art reagiert werden soll.
  • Du kannst mehrere catch-Blöcke für unterschiedliche Exception-Typen verwenden.
  • Im catch-Block steht ein Parameter (z. B. Exception e), über den du auf Details der Exception zugreifen kannst.

3. finally-Block (optional)

  • Wird immer ausgeführt – egal, ob eine Exception aufgetreten ist oder nicht.
  • Typische Anwendung: Ressourcen aufräumen, z. B. Datei schließen, Verbindung beenden.
  • Der finally-Block ist optional; viele Anwendungen kommen auch ohne aus.

Beispiel: Datei einlesen und Ressourcen aufräumen

FileReader reader = null;
try {
    reader = new FileReader("data.txt");
    // Datei verarbeiten
} catch (FileNotFoundException e) {
    System.out.println("Datei nicht gefunden: " + e.getMessage());
} finally {
    // Wird immer ausgeführt
    if (reader != null) {
        try {
            reader.close();
        } catch (IOException e) {
            System.out.println("Fehler beim Schließen der Datei: " + e.getMessage());
        }
    }
}

Mehrere catch-Blöcke – Beispiel

try {
    // Code, der verschiedene Fehler verursachen kann
} catch (FileNotFoundException e) {
    System.out.println("Datei nicht gefunden!");
} catch (IOException e) {
    System.out.println("Allgemeiner Ein-/Ausgabefehler!");
}

Ablauf bei Exceptions (Was passiert genau?)

  1. Code im try-Block wird ausgeführt.
  2. Tritt eine Exception auf, wird der passende catch-Block gesucht.
  3. catch-Block führt die Fehlerbehandlung aus.
  4. finally-Block wird immer ausgeführt (falls vorhanden).
  5. Falls keine Exception auftritt, wird der try-Block vollständig durchlaufen, dann folgt der finally-Block.

Zusammenfassung: Die wichtigsten Punkte

  • Fehler können überall auftreten – man sollte sie nicht ignorieren!
  • Exceptions sind Objekte, die Fehler beschreiben und den Ablauf des Programms steuern.
  • Mit try-catch-finally werden Fehler strukturiert behandelt.
  • Exceptions ermöglichen es, den Programmfluss sauber zu trennen: normale Verarbeitung & Fehlerbehandlung.
  • Gute Fehlerbehandlung erhöht die Zuverlässigkeit und Wartbarkeit von Programmen.

Tipp für die Praxis: Übe das Schreiben von try-catch-finally-Blöcken mit kleinen Beispielen (Dateizugriff, Benutzereingaben, Division durch Null), um den Mechanismus zu verinnerlichen!
